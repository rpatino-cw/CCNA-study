3.5 First-Hop Redundancy Protocols — Read This Before Anything Else

Cisco 200-301 §3.5 — DESCRIBE level (concepts, not config). HSRP (Cisco) · VRRP (open) · GLBP (Cisco). All solve same problem: default-gateway redundancy. Memorize virtual MAC formats + priority defaults + preempt behavior.

1. The 30-Second Answer read this twice

FHRP = First-Hop Redundancy Protocol. Solves: a host\'s default gateway is a single point of failure. Solution: two routers share a virtual IP + virtual MAC. One is active, one is standby. If active dies, standby takes over without hosts noticing. Three flavors: HSRP (Cisco) · VRRP (open standard) · GLBP (Cisco, load-balances).
  • HSRP — Cisco. Active/Standby. Virtual MAC 0000.0c07.acXX. Priority 100 default. Preempt OFF default.
  • VRRP — open standard (RFC 5798). Master/Backup. Virtual MAC 0000.5e00.01XX. Priority 100 default. Preempt ON default.
  • GLBP — Cisco. AVG + multiple AVFs (load-balances across all routers, not just one active).

Memorize: Virtual IP = host\'s default gateway · Virtual MAC = ARP reply target

2. The Problem FHRP Solves why it exists

Hosts use ONE default gateway IP. If that gateway dies, hosts can\'t reach anything outside their subnet. ARP cache holds the gateway\'s MAC for ~4 hours. Even if you have a SECOND router, hosts won\'t know to use it without manual reconfig.

Without FHRP:

  • Active gateway dies → hosts black-hole all off-subnet traffic
  • Manual fix: change every host\'s default gateway → unmanageable
  • DHCP can push new gateway, but lease renewal takes minutes/hours

With FHRP:

  • Two routers share a virtual IP (the gateway hosts know)
  • And a virtual MAC (in hosts\' ARP cache)
  • Active router answers ARP + forwards traffic
  • If active dies, standby seamlessly takes over the virtual IP + MAC
  • Hosts never notice — same IP, same MAC, just a different physical router behind the scenes
FHRP = high availability for default gateways. Hosts unaware. Failover sub-second to seconds.

3. FHRP Anatomy at a Glance visual

Both routers share the virtual IP + virtual MAC. ARP for the virtual IP returns the virtual MAC. The ACTIVE router responds and forwards. STANDBY listens silently. On active failure, standby takes over the same vIP/vMAC — host\'s ARP cache stays valid, no reconvergence needed on the host side.

4. HSRP — Hot Standby Router Protocol Cisco proprietary

PropertyValue
StandardizationCisco proprietary (RFC 2281 informational)
RolesActive / Standby + Listen states for others
Virtual MAC format0000.0c07.acXX (XX = group number in hex)
Group number range0-255 (HSRPv1) or 0-4095 (HSRPv2)
Multicast (v1)224.0.0.2 — UDP port 1985
Multicast (v2)224.0.0.102 — UDP port 1985
Default priority100 (range 0-255). Higher wins active election.
PreemptOFF by default — must enable: standby 1 preempt
Hello / Hold timer3s / 10s default
StatesInit · Learn · Listen · Speak · Standby · Active
R1(config-if)# standby 1 ip 10.10.10.254 R1(config-if)# standby 1 priority 110 R1(config-if)# standby 1 preempt

Group number = HSRP "instance" on that interface. You can run multiple groups on one interface (different VIPs). Group X uses vMAC 0000.0c07.acXX (XX is the group number in hex).

Trap to recognize
HSRP preempt is OFF by default. So even if you raise R1\'s priority above current active, R1 won\'t take over until current active fails. You must explicitly enable standby X preempt.

5. HSRP States 6-stage

  1. Init — initial state, before HSRP starts on the interface.
  2. Learn — router doesn\'t know the virtual IP yet (waiting to learn from active or be told by config).
  3. Listen — knows vIP, but not active or standby. Listening to Hellos.
  4. Speak — actively sending Hellos, participating in election.
  5. Standby — currently the standby router. Will take over if active fails.
  6. Active — currently forwarding for the virtual IP. Replies to ARP. Sends Hellos.

Election: highest standby X priority wins. Tiebreaker = highest interface IP.

Preempt: if enabled, a higher-priority router that joins LATER will take over from current active. Without preempt, active stays active until it fails — even if a more-deserving router is available.

Failover time: roughly 10 seconds with default Hello/Hold timers (3s/10s). Tunable lower via standby 1 timers msec 200 msec 750 for sub-second failover.

6. VRRP — Virtual Router Redundancy Protocol RFC 5798 — open standard

PropertyValue
StandardizationOpen (RFC 3768/5798). Multi-vendor.
RolesMaster / Backup
Virtual MAC format0000.5e00.01XX (XX = VRID/group in hex)
Multicast224.0.0.18 — IP protocol 112 (no L4)
Default priority100 (range 1-254). Higher wins. Priority 0 = withdrawal. 255 reserved for IP owner.
PreemptON by default
Hello (advertisement) interval1 second default
Master-down interval~3 seconds (3 × hello)
R1(config-if)# vrrp 1 ip 10.10.10.254 R1(config-if)# vrrp 1 priority 110

"IP address owner" twist: if the virtual IP is also a real IP on one of the routers, that router gets priority 255 automatically and becomes Master. Cisco/CCNA usually configures vIP as a separate non-physical IP to avoid this corner case.

VRRP is functionally similar to HSRP but with key differences: open standard, faster default timers (1s vs 3s), preempt on by default, different vMAC range.

7. GLBP — Gateway Load Balancing Protocol Cisco · multi-active

Unlike HSRP/VRRP (one active, others idle), GLBP uses ALL routers simultaneously for load-balancing.

PropertyValue
StandardizationCisco proprietary
Active Virtual Gateway (AVG)One router elected. Hands out different vMACs to different hosts via ARP replies.
Active Virtual Forwarders (AVFs)Multiple. Each owns a different vMAC and forwards traffic from hosts mapped to it.
Virtual MAC format0007.b400.XXYY (per-AVF MAC)
Multicast224.0.0.102 — UDP port 3222
Default priority100
Hello / Hold timer3s / 10s default
Load-balance algorithmsRound-robin (default) · Weighted · Host-dependent

How it works: AVG receives ARP requests for the virtual IP. It responds with DIFFERENT vMACs (one per AVF) on a round-robin basis. Result: hosts unknowingly get spread across multiple physical routers. Each AVF independently forwards its share of traffic.

Failure handling: if an AVF dies, AVG reassigns its vMAC to another router. Hosts unaware. Same end-effect as HSRP/VRRP failover but without idle hardware.

11. Cisco's Favorite Traps spot the wrong answer

Trap #1 — "HSRP preempt is on by default"
Wrong. HSRP preempt = OFF by default. VRRP preempt = ON by default. Most-tested HSRP-vs-VRRP difference.
Trap #2 — "HSRP active and VRRP master are different concepts"
Wrong. Same role, different name. HSRP "Active" = VRRP "Master" = the router currently forwarding for the virtual IP.
Trap #3 — "GLBP only uses one router at a time"
Wrong. GLBP uses ALL routers simultaneously via multiple AVFs. AVG hands out different vMACs to different hosts via ARP, splitting traffic. That\'s the whole point.
Trap #4 — "VRRP is Cisco proprietary"
Wrong. VRRP is open (RFC 3768/5798). HSRP + GLBP are Cisco proprietary. Use VRRP for mixed-vendor environments.
Trap #5 — "Higher HSRP priority always wins, automatically taking over active role"
Wrong (without preempt). Without preempt enabled, a higher-priority router that joins later does NOT take over. Active stays active until it dies. Enable preempt explicitly.
Trap #6 — "Hosts know which physical router is active"
Wrong. Hosts only know virtual IP + virtual MAC. The whole point is host abstraction. Hosts don\'t care which physical router is forwarding. Failover is invisible to them.
Trap #7 — "Virtual MAC for HSRP group 5 is 0000.0c07.ac05"
CORRECT. Group number XX is hex. Group 5 → 05. Group 16 → 10 (hex). Group 255 (max v1) → ff. HSRPv2 uses different format: 0000.0C9F.FXXX where XXX = group (3 hex digits, 0-FFF).

13. Self-Check no peeking

Q1. R1 (HSRP priority 110) and R2 (priority 100) are HSRP peers. R2 was up first and is currently Active. R1 boots up. Does R1 take over?

Reveal

NO — without preempt enabled. HSRP preempt OFF by default. R2 stays active until it dies. To make R1 take over now: standby 1 preempt on R1. Refer back: §4, §11 trap #5.

Q2. HSRP virtual MAC for group 1?

Reveal

0000.0c07.ac01. XX = group number in hex. Group 1 → 01. Group 10 → 0a. Refer back: §4.

Q3. Which FHRP is open standard?

Reveal

VRRP — RFC 3768/5798. Use for multi-vendor environments. HSRP + GLBP are Cisco proprietary. Refer back: §6.

Q4. Which FHRP load-balances across multiple routers simultaneously?

Reveal

GLBP. AVG + multiple AVFs. ARP replies hand different vMACs to different hosts. HSRP/VRRP keep standby idle. Refer back: §7.

Q5. List HSRP states in order.

Reveal

Init · Learn · Listen · Speak · Standby · Active. "I L L S S A". Refer back: §5.
