Before you subnet anything, know which addresses are private, public, and special. Cisco gives away free points to anyone who memorizes these.

Special Addresses — Recognize these instantly
RangeNameWhat it is
127.0.0.0/8LoopbackTests local TCP/IP stack. 127.0.0.1 = "localhost." Never leaves the device.
169.254.0.0/16APIPA / Link-LocalAuto-assigned when DHCP fails. Cannot be routed. Means "DHCP is broken."

224.0.0.0/4Multicast (Class D)224.0.0.0 – 239.255.255.255. One-to-many. Used by OSPF, EIGRP, etc.
240.0.0.0/4Reserved (Class E)240.0.0.0 – 255.255.255.254. Experimental. Not assignable.
Exam Shortcut — "Is this IP private?"

Starts with 10. → private. Starts with 172.16-31. → private. Starts with 192.168. → private. Everything else in 1-223 range → public (unless special). It's that simple.

The Only Formula

Usable Hosts = 2(32 - prefix) - 2

Subtract 2 for: network address (all 0s in host portion) + broadcast (all 1s in host portion)

4. The Long Way — Binary AND Method

Textbook method. Slow but essential to understand. Click through each step.

Example: Find the subnet for 192.168.10.75/26
1

Convert the IP to binary

192 = 11000000   168 = 10101000   10 = 00001010   75 = 01001011
2

Write the /26 mask in binary

11111111.11111111.11111111.11000000

26 ones, then 6 zeros. Only the last octet is "interesting" (not all 1s or all 0s).

3

AND the last octet (the interesting one)

IP:   01001011 (75)
Mask: 11000000 (192)
AND:  01000000 (64)
4

Results

Network: .64 First host: .65 Last host: .126 Broadcast: .127

62 usable hosts. This took 2-3 minutes. The magic number method does it in 10 seconds.

8. Route Summarization — Combine Routes Into One

Summarization (supernetting) is the reverse of subnetting: combine multiple specific routes into one advertisement. Find the common prefix bits.

Live Summarizer — Enter networks to combine

Comma-separated. Example: 192.168.16.0/24, 192.168.17.0/24, 192.168.18.0/24, 192.168.19.0/24

The Method
  1. Convert all network addresses to binary
  2. Line them up and compare bit-by-bit from the left
  3. Count how many bits are identical — that's your summary prefix length
  4. The summary route is the common bits + all zeros for the rest

11. Exam Traps — Click to Reveal

Cisco writes wrong answers that look right. Click each trap to see how they trick you.

Trap #1 — Forgetting to subtract 2▼
WRONG: /26 = 64 hosts
RIGHT: /26 = 64 - 2 = 62 usable hosts
Network address and broadcast address are NOT assignable to hosts. Always subtract 2.
Trap #2 — IP is the network or broadcast address▼
"Can 192.168.1.64 be assigned to a host on 192.168.1.64/26?"
NO — .64 IS the network address. First usable = .65. Same with broadcast: .127 is NOT usable.
Trap #3 — Two hosts on different subnets▼
"Can 10.0.0.62 and 10.0.0.65 communicate on a /26?"
.62 is in subnet .0-.63  |  .65 is in subnet .64-.127
Different subnets — need a router. Use the magic number to quickly find which subnet each IP belongs to.
Trap #4 — /31 and /32 edge cases▼
/31 = 2 hosts, both usable (RFC 3021, point-to-point only, no broadcast)
/32 = single host route (routing tables and loopbacks, NOT subnetting)
Trap #5 — Mask doesn't fit what you expect▼
"Subnet mask for 200 hosts?"
/24 = 254 hosts (enough!)  |  /25 = only 126 (NOT enough!)
Answer: 255.255.255.0 (/24) — Always pick the smallest prefix that fits.
