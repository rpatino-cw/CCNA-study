1. The 30-Second Answer read this twice

A static route is a manually-configured "to reach X, send via Y." Four flavors: default (0.0.0.0/0), network (a subnet), host (/32), floating (raised AD = backup). AD = 1 by default. IPv4 syntax: ip route NETWORK MASK NEXT-HOP [AD]. IPv6 syntax: ipv6 route PREFIX/LEN NEXT-HOP [AD].
  • Default route (3.3.a): ip route 0.0.0.0 0.0.0.0 203.0.113.1 → catches everything not matched by a longer prefix.
  • Network route (3.3.b): ip route 10.10.20.0 255.255.255.0 10.10.10.2 → reaches a subnet.
  • Host route (3.3.c): ip route 10.10.20.50 255.255.255.255 10.10.10.5 → reaches ONE specific IP.
  • Floating (3.3.d): ip route 10.10.20.0 255.255.255.0 10.10.10.99 130 → AD 130 makes it backup behind RIP/OSPF.

Memorize: ip route NETWORK MASK NEXT-HOP [AD]

2. Static Route Syntax memorize

R1(config)# ip route DESTINATION-NETWORK MASK NEXT-HOP-IP-or-EXIT-INTERFACE [ADMIN-DISTANCE]
DESTINATION-NETWORKThe remote subnet you want to reach (e.g. 10.10.20.0). MASKSubnet mask in dotted decimal (e.g. 255.255.255.0 = /24). NOT prefix length, NOT wildcard. NEXT-HOP-IPThe IP of the next router (e.g. 10.10.10.2). Most common form. Recursive lookup. EXIT-INTERFACEOr just an interface (e.g. GigabitEthernet0/0). Avoid on multi-access (Ethernet) — forces ARP for every dest. [ADMIN-DISTANCE]Optional. Default = 1. Raise to make floating static (backup).

You can combine next-hop AND exit interface: ip route 10.10.20.0 255.255.255.0 GigabitEthernet0/0 10.10.10.2. Most explicit form. Skips recursive lookup.

Trap to recognize
Cisco loves to hand you a wildcard mask in a stem (e.g. 0.0.0.255) instead of a subnet mask. Static route uses SUBNET MASK (255.255.255.0), NOT wildcard. ACL uses wildcard. Don't confuse.

3. Default Route — 3.3.a 0.0.0.0/0

R1(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1

The default route matches ANY destination not covered by a longer prefix. It becomes the "gateway of last resort."

  • Network = 0.0.0.0, mask = 0.0.0.0 (zero bits matched = matches everything).
  • In show ip route, marked as S* — static + candidate default.
  • "Gateway of last resort" line appears at top of show ip route once configured.
  • Common at branch routers pointing toward HQ or ISP. Common at HQ routers pointing toward ISP for internet egress.
R1# show ip route Gateway of last resort is 203.0.113.1 to network 0.0.0.0 S* 0.0.0.0/0 [1/0] via 203.0.113.1

4. Network Route — 3.3.b subnet

R1(config)# ip route 10.10.20.0 255.255.255.0 10.10.10.2

Reaches a specific subnet (a /N where N < 32). Most-used static type for connecting site-to-site or pointing at branch networks.

  • Mask determines subnet size: /24 = 255.255.255.0, /16 = 255.255.0.0, /30 = 255.255.255.252.
  • Code in routing table: S.
  • AD default = 1 (very trusted).

Hub-and-spoke pattern: HQ router gets a static for each branch subnet, all pointing at the WAN gateway. Simple but doesn't scale → use OSPF/EIGRP at scale.

5. Host Route — 3.3.c /32

R1(config)# ip route 10.10.20.50 255.255.255.255 10.10.10.5

Reaches ONE specific IP. Mask = 255.255.255.255 (/32) = all 32 bits matched. Beats every supernet via LPM.

  • Use cases: pin one host to a different next-hop than the rest of its subnet, route to a loopback for management, override a /24 for one critical destination.
  • Code in table: S with /32.
  • Beats every shorter-prefix route for that one IP via LPM (rule #1).

6. Floating Static — 3.3.d backup

R1(config)# ip route 10.10.20.0 255.255.255.0 10.10.10.99 130

A regular static route with the AD manually raised above the primary protocol's AD. Sits dormant; only activates if the primary route disappears.

Pick AD ABOVE primary protocol:

Primary protocolDefault ADFloating-static AD to use
OSPF110≥111 (typical: 130)
RIP120≥121 (typical: 130 or 200)

HA pattern: primary path = dynamic protocol. Backup = floating static via secondary uplink. If OSPF withdraws the route, floating static takes over. Failover = sub-second to seconds.

Static AD 1 + raised value = floating. Default trust → backup trust. Single number changes the role.

7. IPv6 Static Routes parallel syntax

R1(config)# ipv6 unicast-routing R1(config)# ipv6 route 2001:db8:20::/64 2001:db8:10::2 R1(config)# ipv6 route ::/0 2001:db8:10::1 ! default route R1(config)# ipv6 route 2001:db8:20::5/128 2001:db8:10::5 ! host route R1(config)# ipv6 route 2001:db8:20::/64 2001:db8:10::99 200 ! floating
  • Must enable ipv6 unicast-routing first — disabled by default.
  • Syntax: ipv6 route PREFIX/LENGTH NEXT-HOP [AD]. Prefix-length notation (no separate mask).
  • Default route: ::/0 (zero-prefix-length).
  • Host route: /128 (all 128 bits matched).
  • Verify: show ipv6 route static.
  • AD values + LPM rules same as IPv4.
Trap to recognize
If you forget ipv6 unicast-routing, IPv6 routes don't actually forward traffic. Router still accepts the config silently. Easy production gotcha.

9. Show-Output Decoder read fluently

show ip route static

R1# show ip route static S* 0.0.0.0/0 [1/0] via 203.0.113.1 S 10.10.20.0/24 [1/0] via 10.10.10.2 S 10.10.20.50/32 [1/0] via 10.10.10.5 S 10.10.99.0/24 [130/0] via 10.10.10.99 ! floating (AD 130)
SStatic. Plain network route. S*Static + candidate default = gateway of last resort. [1/0]AD=1 (default static), metric=0. [130/0]Raised AD = floating static. Backup; only active if primary protocol fails. /32Host route. Reaches a single IP.

10. Cisco's Favorite Traps spot the wrong answer

Trap #1 — "Static route uses wildcard mask"
Wrong. Static uses SUBNET MASK (255.255.255.0). ACL uses wildcard mask (0.0.0.255). Don't swap.
Trap #2 — "Default static AD is 0"
Wrong. Static default AD = 1. Connected = 0. One step less trusted than directly connected.
Trap #3 — "Floating static is a /32 route"
Wrong. Floating = static with raised AD. Any prefix length. The "floating" describes its trust level (above primary protocol AD), not its mask.
Trap #4 — "Default route is automatic"
Wrong. You must configure it. ip route 0.0.0.0 0.0.0.0 X. Without it, "Gateway of last resort is not set" + unknown destinations get dropped.
Trap #5 — "ipv6 route works without ipv6 unicast-routing"
Wrong. You MUST enable ipv6 unicast-routing globally first. Without it, IPv6 routes go in the table but the router won't actually forward IPv6 packets between interfaces.
Trap #6 — "Static route to exit-interface on Ethernet is fine"
Wrong (mostly). Exit-interface-only static on Ethernet (multi-access) forces the router to ARP for every distinct destination IP on that segment — performance hit + potentially broken. Always use next-hop IP on Ethernet.
Trap #7 — "Floating static AD must be exactly 130"
Wrong. Just must exceed the primary protocol's AD. 130 is a common convention but anything above OSPF (110) or RIP (120) works. Pick a value that\'s higher than ALL primary protocols you want it to back up.

Memorize until automatic
  1. Static syntax: ip route NETWORK MASK NEXT-HOP [AD]. Default AD = 1.
  2. 4 flavors: default (0.0.0.0/0) · network (subnet) · host (/32) · floating (raised AD).

12. Self-Check no peeking

Q1. Configure a default route via 192.0.2.1.

Reveal

ip route 0.0.0.0 0.0.0.0 192.0.2.1. Will show as S* (gateway of last resort). Refer back: §3.

Q2. You want a backup static for 10.10.20.0/24 via 10.10.10.99 that only activates if OSPF (AD 110) fails. Command?

Reveal

ip route 10.10.20.0 255.255.255.0 10.10.10.99 130. AD 130 > OSPF\'s 110 → floating. Refer back: §6.

Q3. Configure a host route to send traffic for 10.10.5.5 specifically via 10.10.10.7.

Reveal

ip route 10.10.5.5 255.255.255.255 10.10.10.7. /32 mask = exact host. Will beat any /24 covering 10.10.5.0 via LPM. Refer back: §5.

Q4. Configure an IPv6 default route via 2001:db8:1::1.

Reveal

First: ipv6 unicast-routing globally. Then: ipv6 route ::/0 2001:db8:1::1. Refer back: §7.

Q5. Why is exit-interface-only static dangerous on Ethernet?

Reveal

Forces ARP for every distinct destination IP traversing that segment — burns CPU + memory + may break forwarding. Use next-hop IP on multi-access. Exit-int OK on point-to-point links. Refer back: §10 trap #6.
