TFTP vs FTP File Transfer Sim

Two file-transfer protocols, both decades old, both still in production. TFTP is dumb and fast — UDP, no auth, ~256-byte blocks, perfect for booting a switch off the network or shoving an IOS image onto a router. FTP is heavyweight — two TCP channels, login, full directory ops. Use TFTP behind locked doors. Use FTP everywhere else (or SFTP/SCP if you can).

Side-by-side

TFTP

  • Transport: UDP port 69 (request) + ephemeral data ports
  • Auth: none
  • Encryption: none
  • Block size: 512 bytes (or up to 65464 with options)
  • Stop-and-wait ACK per block
  • Use case: IOS image upload, ROMmon recovery, IP phone firmware, PXE boot
  • Verdict: simple, fast, dangerous on the open internet

FTP

  • Transport: TCP — control on 21, data on 20 (active) or ephemeral (passive)
  • Auth: username + password (plaintext — sniffable)
  • Encryption: none (use FTPS or SFTP for that)
  • Two channels: control sends commands, data sends bytes
  • Active mode: server connects back from port 20 — breaks behind NAT
  • Passive mode: client opens both connections — NAT-friendly, default modern
  • Use case: legacy file servers, large transfers needing resume, scripted directory ops
