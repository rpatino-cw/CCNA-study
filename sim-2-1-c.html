The VLAN You Forgot You Were Ondrill

Out of the box every Cisco Catalyst port lives in VLAN 1 — the default. It's enabled, it spans every trunk, you can't delete it, and it carries control-plane chatter (CDP, VTP, DTP, PAgP). Best practice: move user ports off it, change the native VLAN on trunks to something unused, and never put management on VLAN 1. Click any port below to relocate it and watch show vlan brief follow.

Out-of-box switch — every port in VLAN 1

VLAN 1 (default) VLAN 10 (DATA) VLAN 20 (VOICE) VLAN 99 (MGMT)

What's special about VLAN 1

Built-in behavior

  • Cannot be deleted — reserved by IOS
  • Default for every access port
  • Default native VLAN on every trunk
  • Carries CDP, VTP, DTP, PAgP, STP control frames

Why it's a security risk

  • VLAN 1 spans every trunk by default — any port leaks to every switch
  • Attacker on a forgotten VLAN-1 port shares L2 broadcast with management
  • VLAN-hopping (double-tag) targets default native VLAN 1
  • NSA / CIS hardening guides explicitly call out moving off VLAN 1

Worked example — harden a fresh switch

! before — every port still on VLAN 1 SW1# show vlan brief VLAN Name Status Ports ---- -------------------------------- --------- ----------------------- 1 default active Fa0/1, Fa0/2, Fa0/3, Fa0/4 ... ! commands SW1(config)# vlan 10 SW1(config-vlan)# name DATA SW1(config)# interface range fa0/1 - 8 SW1(config-if-range)# switchport mode access SW1(config-if-range)# switchport access vlan 10 ! after — first 8 ports off VLAN 1 SW1# show vlan brief VLAN Name Status Ports ---- -------------------------------- --------- ----------------------- 1 default active Fa0/9, Fa0/10, Fa0/11 ... 10 DATA active Fa0/1, Fa0/2, Fa0/3 ... Fa0/8

Faded prompt — your turn

Audit team flagged: "VLAN 1 active on user-facing ports." Your move:

  1. Create a non-default user VLAN (e.g. vlan 10).
  2. Reassign every access port to that VLAN — switchport access vlan 10.
  3. On every trunk, change the native VLAN to an unused, user-free VLAN (e.g. 999) — switchport trunk native vlan 999.
  4. Shut unused ports — interface range + shutdown.

VLAN 1 still exists after this — you just don't use it. That's the right answer.
