Bit Drill

Octet conversion both directions. Decimal → binary (192 → 11000000), binary → decimal (11010110 → 214), bit-position values (128, 64, 32, 16, 8, 4, 2, 1). Every subnet mask is just this. Once it's reflex, masks read themselves.

Why bits? An IPv4 octet is 8 bits. Each bit position has a fixed value: 128 64 32 16 8 4 2 1. Add the on-bits = decimal value. Mask 255.255.255.192 means the last octet is 11000000 = 128+64. That tells you where the network bits end and host bits begin.

Walkthrough
How to convert in your head
  1. Memorize bit-positions: 128, 64, 32, 16, 8, 4, 2, 1. Left to right. Each is half the previous.
  2. Decimal → binary: walk the positions. Does 128 fit in your number? Yes → write 1, subtract. Does 64 fit in remainder? Repeat. Eight passes done.
  3. Binary → decimal: for every 1-bit, add that position's value.
  4. Mask shortcut: all-1s from the left, then all-0s. Mask octets are only ever: 0, 128, 192, 224, 240, 248, 252, 254, 255. Memorize.
  5. Network bits: count of on-bits in mask. /24 = 24 ones. /26 = 24 ones + 2 more in 4th octet (192).
