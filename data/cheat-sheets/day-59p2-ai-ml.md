# Day 59 (part 2) — AI & Machine Learning

## 🎯 What this video covers

Maps to v1.1 blueprint **6.4 Describe AI and machine learning in network operations** — a NEW v1.1 topic. Definitions, the four ML types, predictive vs generative AI, and Cisco Catalyst Center AI features.

## 🧠 Core Concept

**AI = computers simulating intelligence; ML is a subset of AI where models learn from data; deep learning is a subset of ML using multi-layered neural networks.**

## 🔑 Must-Know Table

| Term | One-line function | Key trait | Exam giveaway phrase |
|---|---|---|---|
| **AI** | Computers simulating intelligence | Wide field, includes ML | "pattern recognition, decision making" |
| **Machine Learning (ML)** | Learn from data without explicit programming | Subset of AI | "learn from data" |
| **Supervised learning** | Train on **labeled** data | Highly accurate, expensive labeling | "labeled dataset, classification" |
| **Unsupervised learning** | Train on **unlabeled** data; find clusters | No labels needed; hidden patterns | "unlabeled, clustering" |
| **Reinforcement learning** | Learn via reward/penalty in environment | Game AI, self-driving | "reward and penalty, agent" |
| **Deep learning** | ML using multi-layer **neural networks** | Mimics human brain; "black box" | "neural networks, hidden layers" |
| **Predictive AI** | Forecast future from historical data | Anomaly detection, traffic forecast | "predicts outcomes" |
| **Generative AI** | Creates new content | LLMs, image/video gen | "ChatGPT, generates text/images" |

## 🧩 Concept-pair clarifications

| Pair | Key contrast |
|---|---|
| **Supervised vs unsupervised** | Supervised = labeled data, accurate. Unsupervised = unlabeled, finds clusters/patterns |
| **Reinforcement vs supervised** | Reinforcement = trial+error in env. Supervised = pre-labeled examples |
| **Deep learning vs other ML** | Deep = multi-layer neural networks for complex/unstructured data. Other ML may use simpler models |
| **Predictive vs generative AI** | Predictive = forecasts. Generative = creates new content |

## 🔗 How it all connects

```
[AI]→[ML subset]→[Deep Learning subset]→[Powers Predictive + Generative AI]
```

## 🚨 Exam Traps

- **ML is NOT all of AI** — AI includes rule-based and other non-ML techniques
- **Deep learning is NOT a separate top-level category** — it's a subset of ML using neural networks
- **Supervised learning is NOT good for finding hidden patterns** — that's unsupervised's job
- **Generative AI does NOT always require deep learning** — though most powerful ones do
- **Reinforcement learning does NOT use labeled data** — it uses rewards/penalties
- **Cisco Catalyst Center is NOT the same as DNA Center** — it's the renamed product (same thing)

## 🧪 Quick-Answer Map

| Stem | Answer |
|---|---|
| ML type that mimics the human brain | **Deep learning** |
| ML type using labeled data | **Supervised** |
| ML type using clusters of unlabeled data | **Unsupervised** |
| ML type for game AI / self-driving | **Reinforcement** |
| AI that forecasts traffic congestion | **Predictive AI** |
| AI that writes emails or code | **Generative AI** |
| Catalyst Center feature for root-cause analysis | **MRE (Machine Reasoning Engine)** |
| Catalyst Center feature using ML to baseline normal behavior | **AI Network Analytics** |
| Catalyst Center feature for endpoint classification | **AI Endpoint Analytics** |
| Catalyst Center feature optimizing Wi-Fi | **AI-enhanced RRM (Radio Resource Management)** |
| Mixing labeled + unlabeled data | **Semi-supervised learning** |

## ⚡ One-Line Master Recall

**AI → ML (supervised, unsupervised, reinforcement, deep) → predictive & generative applications → in networking: traffic forecasting, anomaly detection, config generation, troubleshooting, and Catalyst Center features (AI Net Analytics, MRE, Endpoint Analytics, RRM).**

## ➕ EXTRA — not in video, but on the exam

- **AI hallucinations** — generative AI making up facts. Jeremy mentions; v1.1 may ask why outputs need verification.
- **Bias in training data** — closely related to "garbage in, garbage out"; common framing in security/ethics questions.
- **Network automation pairing** — AI + Ansible/Terraform = predictive ops; pairs with Day 63 topics.

## 🧾 Recap

- **AI ⊃ ML ⊃ Deep Learning.** Predictive and generative AI are applications, not separate ML types.
- **Four ML types**: supervised (labeled), unsupervised (unlabeled), reinforcement (reward/penalty), deep (neural nets).
- **Catalyst Center AI features**: AI Network Analytics, MRE, AI Endpoint Analytics, AI-enhanced RRM.
- **Predictive AI** = forecast; **Generative AI** = create new content (ChatGPT, DALL-E).
- If you can match the four ML types to one example each AND name 3 Catalyst Center AI features, move on.
