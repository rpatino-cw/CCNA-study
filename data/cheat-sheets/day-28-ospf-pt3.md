# Day 28 — OSPF Part 3 (Network Types, Neighbor Requirements, LSA Types)

## 🎯 What this video covers

Maps to v1.1 **3.4 OSPF configure and verify** — network types (broadcast vs point-to-point), DR/BDR election, neighbor adjacency requirements, and the three LSA types you'll see on the exam.

## 🧠 Core Concept

**OSPF behavior depends on network type. Broadcast elects DR/BDR; point-to-point doesn't. Neighbors must match on 7 things; LSA types describe what's in the LSDB.**

## 🔑 Must-Know Table

| OSPF Network Type | Default on | DR/BDR? | Hello / Dead | Notes |
|---|---|---|---|---|
| **Broadcast** | Ethernet, FDDI | **Yes** | 10 / 40 sec | Most common (Ethernet) |
| **Point-to-point** | Serial PPP/HDLC | **No** | 10 / 40 sec | Two-router serial |
| **Non-broadcast** | Frame Relay, X.25 | Yes (manual neighbors) | 30 / 120 sec | Legacy |

| Term | Function | Selection |
|---|---|---|
| **DR (Designated Router)** | Receives LSAs from segment, distributes | **Highest interface priority** → highest router ID |
| **BDR (Backup DR)** | Promotes if DR fails | Second place |
| **DROther** | Anything that isn't DR/BDR | Stuck in 2-way with other DROthers |
| **Default priority** | 1 | Range 0–255 (0 = never DR/BDR) |
| **Loopback as router ID** | Always-up virtual interface | Picked over physical IPs |

| LSA Type | Name | Generated by | Carries |
|---|---|---|---|
| **Type 1** | Router LSA | **Every** OSPF router | Router ID + connected networks |
| **Type 2** | Network LSA | **DR** of multi-access net | List of routers on segment |
| **Type 5** | AS-External LSA | **ASBR** | Routes outside the OSPF domain |

## 🧩 Concept-pair clarifications

| Pair | Key contrast |
|---|---|
| **Neighbor vs Adjacency** | Neighbor = hello-exchanged. **Full adjacency** = LSDB synced (only DR↔all + BDR↔all in broadcast) |
| **DROther↔DROther vs DROther↔DR** | DROthers stop at **2-way**; only fully adjacent with DR/BDR |
| **Multicast 224.0.0.5 vs 224.0.0.6** | All OSPF routers vs **DR/BDR only** |
| **Broadcast vs Point-to-point** | Broadcast elects DR/BDR. P2P doesn't (only 2 routers) |
| **Election preemption** | Non-preemptive — new high-priority router does NOT take over existing DR |
| **HDLC vs PPP** | Both serial L2 encapsulations. HDLC = Cisco default. PPP = open standard |

## 🔗 How it all connects

```
Hello → 2-way → DR/BDR election → Exstart → Exchange → Loading → Full (only with DR/BDR)
```

## 🚨 Exam Traps

- **DR election is NOT** preemptive — a higher-priority router added later does NOT take over
- **Priority 0 does NOT** participate in DR/BDR election ever
- **DROther state with another DROther is NOT** Full — it's **2-way** (stable, expected)
- **Point-to-point does NOT** have a DR or BDR — `show ip ospf neighbor` shows a dash
- **OSPF process IDs do NOT** need to match between routers — area numbers do
- **MTU mismatch is NOT** caught at hello — neighbors form, but stuck in **EXSTART**, no LSDB sync
- **Network type mismatch is NOT** flagged — neighbors show Full but routes don't appear

## 🧪 Quick-Answer Map

| Stem | Answer |
|---|---|
| Default network type on Ethernet | **Broadcast** |
| Default network type on serial PPP/HDLC | **Point-to-point** |
| DR election tie-breaker | **Highest router ID** |
| Default OSPF interface priority | **1** |
| Priority that bars DR/BDR election | **0** |
| OSPF "all routers" multicast | **224.0.0.5** |
| OSPF "DR/BDR only" multicast | **224.0.0.6** |
| Default Hello / Dead on broadcast | **10 / 40 seconds** |
| LSA type generated by every router | **Type 1 (Router LSA)** |
| LSA type generated by DR | **Type 2 (Network LSA)** |
| LSA type for external routes | **Type 5 (AS-External)** |
| 7 neighbor requirements | **Area, subnet, process up, unique RID, Hello/Dead, auth, MTU, network type** |
| Stuck-in-EXSTART symptom | **MTU mismatch** |

## ⚡ One-Line Master Recall

**Broadcast elects DR/BDR (highest priority → highest RID, non-preemptive); point-to-point doesn't; LSA types 1/2/5 = router/network/external; mismatch in MTU or net-type silently breaks LSDB sync.**

## ➕ EXTRA — not in video, but on the exam

- **Type 3 Summary LSA** — Generated by ABR; advertises inter-area routes. Common distractor.
- **`router-id X.X.X.X`** — Manual override beats loopback selection. Requires `clear ip ospf process` to apply.
- **`ip ospf network point-to-point`** — Common Ethernet tweak when only 2 routers share a link (skips DR election, faster).
- **Passive interface** — Interface still advertised but no Hellos sent (security on user-facing ports).

## 🧾 Recap

- **Network type drives DR/BDR behavior — broadcast elects, point-to-point doesn't.**
- **DR election: highest priority → highest router ID, non-preemptive.**
- **DROthers stay in 2-way with each other; Full only with DR/BDR.**
- **LSA types: 1 = router (every), 2 = network (DR), 5 = external (ASBR).**
- **7 neighbor reqs — area, subnet, process up, unique RID, timers, auth, MTU, net-type.**
- If you can troubleshoot a stuck-EXSTART or stuck-2-way scenario by name, move to Day 29.
