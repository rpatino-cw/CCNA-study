Next let me introduce the concept of DHCP Relay.&nbsp;&nbsp;
Some network engineers might choose to&nbsp;
configure each router to act as the DHCP server&nbsp;&nbsp;
for its connected LANs. However, large enterprises&nbsp;
often choose to use a centralized DHCP server,&nbsp;&nbsp;
which will assign IP addresses to DHCP clients&nbsp;
in all subnets throughout the enterprise network.&nbsp;&nbsp;
If the server is centralized, it won’t receive&nbsp;
the DHCP clients’ broadcast DHCP messages.&nbsp;&nbsp;
Remember, broadcast messages don’t leave the local&nbsp;
subnet, routers don’t forward them. However to&nbsp;&nbsp;
fix this you can configure a router to act as a&nbsp;
‘DHCP relay agent’. If you do that, the router&nbsp;&nbsp;
will forward the clients’ broadcast DHCP messages&nbsp;
to the remote DHCP server as unicast messages.&nbsp;&nbsp;
Let me use this network to demonstrate. PC1&nbsp;
is a DHCP client, so it will broadcast a DHCP&nbsp;&nbsp;
Discover message to ask DHCP servers on the&nbsp;
local network to assign it an IP address.&nbsp;&nbsp;
However, in this case R1 isn’t a DHCP server.&nbsp;
SRV1 is the central DHCP server for this network,&nbsp;&nbsp;
so R1 will need to forward any&nbsp;
DHCP messages from PC1 to SRV1.
Let me demonstrate that visually. SRV1 is a&nbsp;
DHCP server, and R1 is a DHCP relay agent.&nbsp;&nbsp;
PC1 broadcasts a DHCP Discover message to get an&nbsp;
IP address. R1, a DHCP relay agent, relays the&nbsp;&nbsp;
message to SRV1. Notice that the source address&nbsp;
changes to the address of R1’s G0/1 interface,&nbsp;&nbsp;
and the destination is SRV1’s IP&nbsp;
address. So, it is a unicast message.&nbsp;&nbsp;
Then SRV1 replies with the DHCP Offer, sending&nbsp;
it to the address of R1’s G0/1 interface,&nbsp;&nbsp;
192.168.1.1. R1 then forwards it to&nbsp;
PC1. The source is R1’s IP address,&nbsp;&nbsp;
and the message is either sent unicast to PC1 or&nbsp;
broadcast. Then PC1 broadcasts a request message,&nbsp;&nbsp;
and R1 relays it to SRV1. Finally SRV1 replies&nbsp;
with a DHCP Ack, and R1 forwards that to PC1,&nbsp;&nbsp;
which configures the IP address it was&nbsp;
assigned, for example 192.168.1.100.&nbsp;&nbsp;
