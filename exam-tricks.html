How the CCNA Tricks You

The exam doesn't just test knowledge — it tests your ability to read carefully. Here's how Cisco words questions to trip you up, and how to beat every trick.

Section 1

Question Format Types

The CCNA uses five distinct question formats. Each has a specific trap designed to cost you points even when you know the material.

Type 1

"Choose TWO" / "Choose THREE"

The Trap

Students pick only one answer and move on, losing half the points.

Example

Which TWO statements describe OSPF? (Choose two.)

A. Uses hop count as its metric
B. Is a link-state routing protocol
C. Has an administrative distance of 120
D. Uses Dijkstra's algorithm
E. Is a distance-vector protocol
Correct: B, D
Strategy

Read the question stem FIRST. If it says "choose two," mentally circle that. Count your selections before submitting. The exam interface shows how many you need — match that number exactly.

Type 2

"NOT" / "EXCEPT" Questions

The Trap

You find the right answer instead of the wrong one. Your brain defaults to "which is correct?" and you pick the opposite of what they asked.

Example

Which is NOT a characteristic of TCP?

A. Reliable delivery
B. Connection-oriented
C. Connectionless
D. Uses acknowledgments
Correct: C (that's UDP)
Strategy

Cover the "NOT" mentally. Find the 3 correct statements — the remaining one is your answer. Mark true/false next to each option. The odd one out wins.

Type 3

Scenario-Based (The Paragraph)

The Trap

Information overload. You panic reading 4-5 sentences of context and miss the actual question buried at the end.

Example

A network engineer is configuring a new branch office. The branch has 45 users who need access to the main office VLANs. The engineer plans to use a router-on-a-stick design. Which interface configuration would correctly enable inter-VLAN routing?

A. Configure subinterfaces with 802.1Q encapsulation
B. Configure each VLAN on a separate physical interface
C. Enable dynamic trunking on the router interface
D. Configure a Layer 3 switch with SVIs
Correct: A
Strategy

Read the LAST sentence first — that's the actual question. Then scan the paragraph for relevant details (45 users, router-on-a-stick). Ignore the noise. The scenario is set dressing; the question is what matters.

Type 4

"Best" Answer (Two Seem Right)

The Trap

Two options are technically correct, but one is "more correct" per Cisco's textbook definition. You pick the practical one; Cisco wanted the theoretical one.

Example

What is the BEST reason to implement VLANs?

A. Improve security
B. Reduce broadcast domains
C. Increase bandwidth
D. Simplify cabling
Correct: B (Cisco's official primary reason)
Strategy

Think "what would Cisco say?" Always pick the textbook/RFC-aligned answer over the practical real-world one. Cisco wrote the exam and the textbook — they want their answer, not yours.

Type 5

Drag-and-Drop

The Trap

You can't go back and change it once you've submitted. One wrong placement can cascade into multiple wrong answers.

Example

Drag the correct cable type to each connection:

Router-to-Switch → Straight-through
Switch-to-Switch → Crossover
PC-to-Router → Crossover
Match each pair to the right cable type
Strategy

Do the ones you're 100% sure about first. If unsure, use elimination — place the definite matches, then the remaining items narrow down. These are often ordering or matching, so process of elimination is powerful.

Section 2

10 Wording Traps

Cisco uses specific words to redirect your thinking. Learn to spot these triggers and you'll stop giving away free points.

  1. "All that apply"

    Means MORE than one is correct. Never pick just one. If you only see one right answer, you're missing something — re-read the options.

  2. "Which statement is TRUE"

    Only one is fully correct. The other three have one wrong word each — a wrong protocol, a wrong number, a wrong direction. Read every word in every option.

  3. "Default" behavior

    Cisco asks about defaults, not what you'd configure. Know: default VLAN (1), default AD values (OSPF=110, EIGRP=90, RIP=120), default hello timer (10s), default STP priority (32768).

  4. "Most efficient"

    Means fewest addresses wasted. Always pick the tightest subnet that fits the host count. /28 for 14 hosts beats /24 for the same — Cisco wants minimal waste.

  5. "Automatically"

    Means without manual configuration. Traps people who confuse auto vs manual features. CDP discovers neighbors automatically; static routes don't. DTP negotiates trunks automatically; manual trunk config doesn't.

  6. "Immediately"

    STP takes 30-50 seconds to converge. OSPF hellos are every 10 seconds. Nothing is truly immediate in networking. If an answer says "immediately," it's almost always wrong unless it's about RSTP or a direct command.

  7. "Ensure" vs "Enable"

    "Ensure" = guarantee it works end-to-end. "Enable" = just turn it on. A question asking you to "ensure SSH access" requires hostname + domain name + crypto keys + VTY config. "Enable SSH" might just be the transport input command.

  8. "By default"

    The configuration BEFORE you touch it. Not what you'd set it to. All ports are in VLAN 1 by default. STP is enabled by default. DTP is on by default. Know the factory state.

  9. "Required" vs "Recommended"

    Cisco loves this distinction. SSH requires a hostname + domain name + RSA keys. That's required, not optional. OSPF requires matching hello/dead timers, subnet masks, and area IDs between neighbors.

  10. Double negatives

    "Which is NOT an incorrect statement?" = "Which IS correct?" Simplify the grammar before answering. Write it in plain English on your scratch paper.

Section 3

Domain-Specific Traps

Each exam domain has its own favorite tricks. Here are the ones that catch the most people.

Subnetting Traps

  • Forgetting to subtract 2 — network address + broadcast address are not usable hosts. 2n - 2 for host count.
  • Confusing /31 with /30 — /31 is for point-to-point links (both addresses usable per RFC 3021). /30 gives 2 usable hosts with the traditional subtract-2 rule.
  • "How many SUBNETS" vs "how many HOSTS" — subnets = 2borrowed bits. Hosts = 2host bits - 2. Different formulas, same mask — read which one they asked.

OSPF Traps

  • DR/BDR election order — uses ROUTER PRIORITY first (highest wins), then ROUTER ID as tiebreaker. Not interface IP. Default priority is 1; priority 0 means "never DR."
  • Metric confusion — OSPF cost is based on BANDWIDTH (reference BW / interface BW), not hop count. Hop count is RIP. Don't mix them up.
  • Network type differences — broadcast (default on Ethernet) elects DR/BDR. Point-to-point (default on serial) does not. NBMA requires manual neighbor config. The question will test whether you know the default network type.

Security Traps

  • Standard ACLs: place CLOSE TO DESTINATION — because standard ACLs only match source IP. Placing near source would block traffic to all destinations, not just the intended one.
  • Extended ACLs: place CLOSE TO SOURCE — because extended ACLs match source, destination, and port. Placing near source prevents unwanted traffic from crossing the network.
  • Implicit "deny any" — every ACL ends with an invisible deny. If you only write permit statements, everything else is denied. If you only write deny statements, nothing gets through (no implicit permit).

Wireless Traps

  • WPA2-Personal = PSK. WPA2-Enterprise = 802.1X/RADIUS. They ALWAYS test this distinction. "Pre-shared key" is the giveaway for Personal mode.
  • CAPWAP tunnels — the control plane always goes to the WLC. Data plane can be locally switched (FlexConnect) or tunneled to the WLC. The question will specify which mode.
Section 4

Time Management Strategy

You have 120 minutes for roughly 100-110 questions. That's tight. Here's how to budget it.

~65s
Per Question
120 minutes / ~110 questions. This is your default pace. If a question takes more than 45 seconds, flag it and move on.
3–5 min
Per Simulation
Testlet and simulation questions take longer. Budget 15-20 minutes total for these. Don't rush them — they're worth more.
15 min
Review Buffer
Aim to finish with 15 minutes left. Use that time for flagged questions. Fresh eyes catch mistakes tired eyes missed.
0 penalty
For Guessing
Never leave a question blank. There is no penalty for wrong answers. A guess has a 25% chance; a blank has 0%.
Section 5

The "Cisco Way" — How They Think

Cisco wrote the exam AND the textbook. When in doubt, think like Cisco, not like a real-world engineer.

  • Standards over practice. Cisco always prefers the standards-based (RFC/IEEE) answer over the practical real-world one. "What does the RFC say?" beats "what would I actually do?"

  • Security = authentication AND encryption. If a question asks about security best practices, the answer almost always involves both. SSH over Telnet. HTTPS over HTTP. WPA2 over WPA.

  • The L2 security trio. Layer 2 security questions almost always involve port security + DHCP snooping + Dynamic ARP Inspection (DAI). Know all three and how they work together.

  • Troubleshoot bottom-up. Follow the OSI model: physical → data link → network → transport. Check cables before checking routes. Check Layer 2 before Layer 3. Cisco expects this methodology.

  • "show" commands verify. "How would you verify?" = which show command. show ip route, show ip interface brief, show vlan brief, show spanning-tree. Know the right show command for each topic.

Section 6

Practice: Read Like Cisco

Identify the trick before you answer. Click to reveal what Cisco is really testing.

Question 1
Which TWO protocols operate at the transport layer of the OSI model? (Choose two.)
A. HTTP
B. TCP
C. IP
D. UDP
E. ARP
The Trick

"Choose two" is easy to miss. HTTP (application), IP (network), and ARP (data link) are all real protocols but wrong layers. Students who pick only one or confuse layers lose points.

Correct: B, D — TCP and UDP both operate at Layer 4 (transport). Count your selections: you need exactly two.

Question 2
Which statement about OSPF is NOT correct?
A. OSPF uses cost as its metric
B. OSPF has an administrative distance of 110
C. OSPF uses hop count to determine the best path
D. OSPF is a link-state routing protocol
The Trick

The "NOT" reversal. Three statements are true. Your brain wants to pick a true statement. C sounds plausible if you confuse OSPF with RIP — RIP uses hop count, OSPF uses cost based on bandwidth.

Correct: C — OSPF uses cost (reference bandwidth / interface bandwidth), not hop count. That's RIP. A, B, and D are all true statements about OSPF.

Question 3
A company has a /24 network and needs to create subnets that support at least 30 hosts each. What is the most efficient subnet mask?
A. /26 (62 hosts)
B. /27 (30 hosts)
C. /28 (14 hosts)
D. /25 (126 hosts)
The Trick

"Most efficient" means tightest fit. /26 works (62 hosts) but wastes 32 addresses. /27 gives exactly 30 usable hosts (2^5 - 2 = 30) — perfect fit. /28 is too small. /25 is way too big.

Correct: B — /27 provides exactly 30 usable hosts, which is the minimum needed. No waste. "Most efficient" always means smallest subnet that fits.

Question 4
An administrator needs to ensure that only authorized devices can connect to switch ports. By default, how many MAC addresses does port security allow?
A. 0
B. 1
C. 2
D. Unlimited
The Trick

"By default" is the key phrase. The scenario about authorized devices is just context — the question is purely about the default setting. Students who overthink the scenario might pick "unlimited" (no port security) or guess a higher number.

Correct: B — When port security is enabled, the default maximum number of MAC addresses is 1. Not 0, not unlimited — exactly 1. Know your defaults.

Question 5
A network technician is troubleshooting connectivity between two directly connected routers. The physical layer has been verified as operational. Which step should the technician perform NEXT?
A. Verify the routing table entries
B. Check the data link layer (encapsulation, framing)
C. Test end-to-end application connectivity
D. Verify the DNS configuration
The Trick

"NEXT" combined with the OSI model. Physical is verified — so the NEXT step is Layer 2 (data link), not Layer 3 (routing table) or Layer 7 (application/DNS). Cisco expects bottom-up troubleshooting methodology.

Correct: B — After verifying Layer 1 (physical), the next OSI layer is Layer 2 (data link). Check encapsulation type, framing, and Layer 2 addressing. Never skip layers.

Final Advice

The exam tests your ability to read carefully, not just your networking knowledge. Slow down. Read every word. Count your selections. Check for NOT/EXCEPT. And when two answers seem right, think: what would the Cisco textbook say?
