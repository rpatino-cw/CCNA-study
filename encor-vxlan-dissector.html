VXLAN Packet Dissector

VXLAN wraps a full Ethernet frame inside UDP, giving you 16.7 million virtual segments (24-bit VNI) across a routed IP underlay. The VTEP device performs all encapsulation and decapsulation. Step through the animation to see every byte added.

Walk-away facts (exam-accurate)

  • VXLAN = MAC-in-UDP (RFC 7348)
  • VTEP: Virtual Tunnel Endpoint. Encaps and decaps on both sides.
  • UDP dst port 4789 (IANA). Src port = hash of inner frame for ECMP load balancing.
  • VXLAN header is exactly 8 bytes. Contains the 24-bit VNI.
  • 24-bit VNI gives 16,777,216 segments vs 4,096 for 802.1Q VLAN.
  • Total outer overhead: 14 (Eth) + 20 (IP) + 8 (UDP) + 8 (VXLAN) = 50 bytes.
  • Enables L2 adjacency between VMs across routed (L3) underlay networks.
  • Used in VXLAN-EVPN (BGP control plane) and flood-and-learn data-plane variants.
