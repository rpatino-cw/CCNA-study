Phase 1: Place GPU Nodes

The spine switch and two leaf switches are pre-installed in the rack. Click a palette item, then click the canvas to place your two DGX H100 compute nodes. The rail-optimized topology requires exactly 2 GPU nodes with 2 NICs each.

Select a device from the palette, then click to place it

Phase 2: Wire the Rails

Click a NIC port (small circle on a GPU node) then click a switch port to wire them. Rail-optimized means: mlx5_0 on EVERY node connects to leaf-0, mlx5_1 on EVERY node connects to leaf-1. Both leaves connect to the spine. A naive wiring (all NICs from node-0 to leaf-0, all NICs from node-1 to leaf-1) forces same-rank traffic up to the spine and is what you should avoid.

Phase 3: Configure the Fabric

Use the terminals below to bring up the subnet manager and set the storage partition. Start on the SM host (dgx-h100-01), then verify and set the PKey.

admin@dgx-h100-01 -- IB fabric bring-up
admin@dgx-h100-01:~$

Configure Commands

  • opensm -p 15 Start SM, priority 15 (wins master election)
  • sminfo Verify SM state (expect SMINFO_MASTER)
  • saquery --pkey List partition keys in the fabric
  • saquery -p 0x8001 Set storage PKey full member
  • ibdiagnet Full fabric health scan
  • help Full command list

Phase 4: Validate the Fabric

Run the fabric health scan and bandwidth test to confirm the bring-up is complete. A clean ibdiagnet plus a passing ib_write_bw result confirm the fabric is ready for jobs.

admin@dgx-h100-01 -- validation
admin@dgx-h100-01:~$

Validate Commands

  • ibdiagnet Full fabric scan: no errors = pass
  • iblinkinfo Fabric link map, all HEALTHY
  • sminfo Confirm SMINFO_MASTER still elected
  • ib_write_bw Bandwidth test (pass = 380+ Gb/s)
  • all_reduce_perf NCCL collective busbw test
