1. The 5 Meta-Rules

Universal multiple-choice elimination, CCNA-flavored. Burn these in first. [know] = from cheatsheet · [inferred] = reasoning

Rule 1 · [know]Routing decisions go: prefix, then AD, then metric.

If a question asks which route wins, longest prefix match wins first, always. Admin distance only breaks ties between equal prefixes. Metric only inside the same protocol.

Rule 2 · [inferred]If two options say the same thing, both are wrong.

Cisco never leaves two correct answers. Twins cancel. Pick from the other two.

Rule 3 · [inferred]Absolute words (always, never, all, only) usually wrong.

One real exception to keep: OSPF priority 0 means never DR. That "never" is true.

Rule 4 · [inferred]Unsure? Pick the secure, modern Cisco default.

SSH over Telnet. WPA3 over WPA2 over WEP. AES over TKIP. HTTPS over HTTP. TACACS+ for device admin.

Rule 5 · [inferred]Match the verb in the stem to the layer.

"Switch, VLAN, MAC, collision" means Layer 2. "Route, subnet, IP, broadcast" means Layer 3. Wrong-layer answers are free eliminations.

4. Trigger Phrases → Answer

Stem keyword almost always points to one answer. Read for these first. [know]

  1. "source IP only / 1-99" → Standard ACL
  2. "destination + protocol + port / 100-199" → Extended ACL
  3. "place close to the source" → Extended ACL
  4. "place close to the destination" → Standard ACL
  5. "dropped, nothing explicitly denied" → implicit deny any
  6. "connect 30 PCs in one room" → Switch
  7. "join two different networks" → Router
  8. "APs managed centrally via CAPWAP" → WLC (lightweight)
  9. "no DR/BDR election" → OSPF point-to-point
  10. "highest loopback becomes this" → OSPF Router-ID
  11. "neighbors won't form, no error" → area / subnet / timer mismatch
  12. "auto IP, DORA" → DHCP
  13. "PCs on another subnet get no IP" → ip helper-address
  14. "late collisions + slow link" → duplex mismatch
  15. "backup route, higher AD" → floating static
  16. "secure management / no clear-text password" → SSH
  17. "agentless, push, YAML" → Ansible
  18. "declarative, infrastructure as code" → Terraform

6. Subnet Speed Method

Magic-number / block-size. Any subnet question in under 30 seconds. [know]

  1. Find the interesting octet (where the mask is not 255 or 0).
  2. Magic number = 256 minus that mask value. That is your block size.
  3. Networks count up by the block: 0, block, 2x block, ...
  4. Your network = the multiple just below your IP.
  5. Broadcast = next network minus 1. Usable = network+1 to broadcast-1.
192.168.1.100 /26
/26 → mask 255.255.255.192, interesting octet = 4th.
Magic = 256 - 192 = 64. Blocks: 0, 64, 128, 192.
.100 falls in the .64 block → network .64.
Broadcast = 128 - 1 = .127. Usable .65 to .126.
Hosts shortcut: 2^(32-26) - 2 = 62.

7. Exam-Day Execution

Order of operations under the clock. [inferred]

Order of ops

  • No going back. Once you submit a question it is locked. Commit, do not dither.
  • Sims and drag-drops are worth more. Do not rush, but cap them at 8-10 minutes.
  • First pass: answer everything you know cold. Numbers, triggers, pairs.
  • Feels like a rabbit hole? Pick your best elimination and move. You cannot return anyway.

Time + guessing

  • About 60 seconds per multiple choice. Hard cap sims at 8-10 minutes.
  • 90 seconds on an MCQ with no answer → eliminate 2, guess, go.
  • Never blank. Blank and wrong score the same. A guess after eliminating 2 is better than a coin flip.
  • Timer warning hits with items left? Stop reading, fill every blank. A guessed B beats an empty.
