Subnetting Drill

Random subnetting questions. The exam expects you to subnet in your head — no calculator.

What You Need to Know

Memorize this and you can answer every subnetting question on the Network+ exam.

Host Formula
2(32 - prefix) - 2 = usable hosts
Always subtract 2 — network address + broadcast are never assignable.
Magic Number (Fast Method)
256 - mask octet = block size
Count by the block size. The block your IP falls in = your subnet.
Private Ranges (RFC 1918)
10.x   172.16-31.x   192.168.x
Everything else 1-223 = public. 127 = loopback. 169.254 = APIPA.
Key Addresses
Network = first IP in block
Broadcast = last IP in block
First usable = network + 1. Last usable = broadcast - 1.
Masks to Memorize
/24.0 — 254h
/25.128 — 126h
/26.192 — 62h
/27.224 — 30h
/28.240 — 14h
/29.248 — 6h
/30.252 — 2h
Common Traps
  • Always subtract 2 — /26 = 64 addresses but only 62 usable
  • Network addr is NOT a host — .64 on a .64/26 subnet is the network, not assignable
  • Close IPs can be different subnets — .62 and .65 on /26 need a router between them

Worked Example: 192.168.10.75/26

Step 1 — Find the mask. /26 means the last octet mask = 256 - 2(32-26) = 256 - 64 = 192. Full mask: 255.255.255.192
Step 2 — Magic number. 256 - 192 = 64. That's your block size.
Step 3 — Find the block. Count by 64: 0, 64, 128, 192… The IP 75 falls in the 64–127 block.
Step 4 — Read the answers.
Network: 192.168.10.64  |  Broadcast: 192.168.10.127
First host: .65  |  Last host: .126  |  Usable: 62
