Translation walkthrough
How NAT translation works (recipe)
  1. Inside-local = the host's real IP on the inside network. Inside-global = how the inside host APPEARS to the outside (post-NAT).
  2. Static NAT: 1-to-1 fixed map. ip nat inside source static 10.0.0.5 200.1.1.5 → 10.0.0.5 always becomes 200.1.1.5.
  3. Dynamic NAT pool: 1-to-1, pool of inside-globals. First inside host gets first free pool entry. When the pool runs dry, new translations fail. No port-sharing.
  4. PAT / overload: N-to-1. All inside hosts share ONE outside IP (often the router's outgoing interface). Unique source ports keep flows separate.
  5. Direction matters. Outbound: source rewritten (inside-local → inside-global). Inbound (return traffic): destination rewritten (inside-global → inside-local).
                  Inside              Outside
                       │                 │
       inside-local ───┤ NAT box ├─── inside-global   (source after NAT)
       outside-local ──┤         ├─── outside-global  (= the destination, usually unchanged)
