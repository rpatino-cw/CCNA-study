ARP — Turning IPs Into MACsdrill

A host has a destination IP but L2 needs the dest MAC to build a frame. ARP is the question "who has 10.1.1.5? Tell 10.1.1.3" sent as a broadcast. Only the owner replies (unicast) with its MAC. Both ends cache the result. Without ARP, no LAN packet leaves a host.

  1. 1. Source checks its ARP cache — miss.
  2. 2. Source builds ARP REQUEST (target IP=X). Broadcasts to ff:ff:ff:ff:ff:ff.
  3. 3. Switch floods request out every other port.
  4. 4. Each host inspects: is this MY IP? All ignore except the owner.
  5. 5. Owner sends ARP REPLY as unicast back to source MAC, includes its MAC.
  6. 6. Source caches IP→MAC. Now it can build the data frame.
Hit Resolve. Watch the broadcast hit every host (only the target replies). Cache populates on both ends.
