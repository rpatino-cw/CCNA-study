1. The 30-Second Answer read this twice

A routing table line tells the router HOW to forward an IP packet. Format: [Code] [Prefix/Mask] [AD/Metric] via [Next-hop], [Exit-interface]. Six fields. Memorize each.
O 10.20.0.0/24 [110/65] via 10.0.0.2, 00:14:33, GigabitEthernet0/1
  • O = code (OSPF in this example)
  • 10.20.0.0/24 = prefix + mask
  • 110 = administrative distance (AD)
  • 65 = metric (OSPF cost)
  • 10.0.0.2 = next-hop IP
  • GigabitEthernet0/1 = exit interface

Memorize: Code В· Prefix/Mask В· [AD/Metric] В· via Next-hop В· Exit-interface

2. Anatomy of a Route Line visual

Some routes show only next-hop (e.g. recursive lookups), some only exit-interface (point-to-point links), some both. Connected routes (C, L) have NO AD/metric brackets вЂ” they're always [0/0] implicitly.

Metric only compares routes from the SAME protocol. OSPF cost 10 vs RIP hop count 10 are not comparable вЂ” AD picks first (OSPF AD 110 vs RIP AD 120 в†’ OSPF wins).

6. Next Hop + Exit Interface вЂ” 3.1.d recursive lookup

Next-hop IP

  • via 10.0.0.2
  • Router does a recursive lookup to find HOW to reach 10.0.0.2 (which subnet в†’ which interface)
  • Common in dynamic protocols + most static routes

Exit interface

  • via GigabitEthernet0/1
  • Tells router to send out a specific interface вЂ” no recursive lookup
  • Common on point-to-point links (one neighbor per interface)

Both (most common)

  • via 10.0.0.2, GigabitEthernet0/1
  • Router knows BOTH the L3 next-hop AND the L2 egress interface
  • Most efficient вЂ” skips ARP recursion

Recursive lookup = if route says "via X.X.X.X" with no exit interface, the router searches its table again to find the route to X.X.X.X. Continues until it finds a connected interface to send out.

7. Gateway of Last Resort + Default Route вЂ” 3.1.g 0.0.0.0/0

The default route (0.0.0.0/0) matches ANY destination not covered by a more specific route. It's the catch-all вЂ” used when nothing else matches.

The gateway of last resort (GoLR) = the next-hop currently used for the default route. Shown at the top of show ip route:

R1# show ip route Gateway of last resort is 203.0.113.1 to network 0.0.0.0 ... S* 0.0.0.0/0 [1/0] via 203.0.113.1

Configured via:

  • Static: ip route 0.0.0.0 0.0.0.0 203.0.113.1 в†’ shown as S*
  • OSPF: default-information originate в†’ propagated as O*E2
  • EIGRP: candidate default flag в†’ shown as D*
  • BGP: a 0.0.0.0/0 received from peer в†’ B*
Trap to recognize
"Gateway of last resort is not set" = NO default route exists. Packets to unknown destinations are dropped + ICMP "destination unreachable" sent back.

Route line вЂ” 6 fields, left to right

Route selection вЂ” 3 tiebreakers in order
1) Longest prefix В· 2) Lowest AD В· 3) Lowest metric

Longest-Prefix Match wins FIRST (more specific = better). Then AD compares routes for the same prefix from different protocols. Then metric within one protocol.

Default route вЂ” 4 ways
S* (static) В· O*E2 (OSPF) В· D* (EIGRP) В· B* (BGP)

The asterisk on S* = "candidate default" вЂ” this is the gateway of last resort.

C vs L вЂ” common confusion
C = whole subnet В· L = router's own /32

10. Cisco's Favorite Traps spot the wrong answer

Trap #1 вЂ” "Lowest metric always wins"
Wrong. Longest prefix wins FIRST. Then AD. Metric only tiebreaks WITHIN one protocol. OSPF metric 10 vs RIP metric 5 в†’ OSPF still wins (lower AD: 110 vs 120). Metric doesn't even compare across protocols.
Trap #2 вЂ” "OSPF AD is 100"
Wrong. OSPF AD = 110. AD 100 = legacy IGRP. Easy to confuse. Memorize: 110 OSPF, 115 IS-IS, 120 RIP вЂ” they're 5 apart.
Trap #3 вЂ” "C and L are the same"
Wrong. C = the connected subnet. L = the /32 of the router's interface IP itself. Two different table entries from one interface.
Trap #4 вЂ” "Default route is automatic"
Wrong. You must configure it. Static (ip route 0.0.0.0 0.0.0.0 X) or learn it dynamically. Without one, "Gateway of last resort is not set" appears + unknown destinations get dropped.
Trap #5 вЂ” "eBGP AD is 200"
Wrong. eBGP AD = 20 (very trusted). iBGP AD = 200 (very untrusted). Backwards from intuition. Reason: eBGP routes are from explicit BGP peering and assumed correct; iBGP routes lack guaranteed loop prevention so they're trusted last.
Trap #6 вЂ” "If two routes have same AD, both go in routing table"
Half-right. Same AD + same metric + same prefix = ECMP (Equal-Cost Multi-Path), both installed. Same AD but different metric в†’ only the better metric wins. Different AD = only lower-AD wins.
Trap #7 вЂ” "Static AD is 0"
Wrong. Static AD = 1. Connected = 0. Static is one step less trusted than the router's own connected subnets.

11. Top 10 Must-Know if-only-this

Memorize until automatic
  1. Route line format: Code В· Prefix/Mask В· [AD/Metric] В· via Next-hop В· Exit-interface
  2. Codes: C=connected, L=local /32, S=static, S*=static default, O=OSPF, D=EIGRP, R=RIP, B=BGP.
  3. AD ladder: 0/1/20/90/110/115/120/170/200/255 (Connected/Static/eBGP/EIGRP/OSPF/ISIS/RIP/EIGRP-ext/iBGP/Unreachable).
  4. Route selection order: 1) Longest prefix В· 2) Lowest AD В· 3) Lowest metric. NEVER swap these.
  5. OSPF metric = cost (10вЃё / bandwidth). RIP = hop count (max 15). EIGRP = composite (BW + delay).
  6. Default route = 0.0.0.0/0. Marked with * (S*, O*E2, D*, B*). "Gateway of last resort" line at top of show ip route shows current default's next-hop.
  7. Recursive lookup: "via X.X.X.X" without exit-int means router re-looks-up X.X.X.X to find how to reach it.
  8. C and L appear together for every up/up interface. C = subnet (/24), L = router's own IP (/32).
  9. eBGP AD = 20 (low, very trusted). iBGP AD = 200 (high, least trusted). Counterintuitive вЂ” memorize cold.
  10. Connected (C/L) and Static (S) have NO metric shown (or [0/0]) вЂ” metric only matters for dynamic protocols.

12. Self-Check no peeking

Q1. O 10.20.0.0/24 [110/65] via 10.0.0.2, GigabitEthernet0/1 вЂ” what does the 110 mean?

Reveal

Administrative distance. 110 = OSPF default AD. The 65 after the slash is the metric (OSPF cost). Refer back: В§1, В§4.

Q2. AD of a directly connected subnet?

Reveal

0. Most trusted вЂ” the router can SEE the link directly. Static = 1, eBGP = 20, EIGRP = 90, OSPF = 110. Refer back: В§4.

Q3. Two routes вЂ” OSPF cost 100 vs RIP cost 5 вЂ” both for 10.10.10.0/24. Which gets installed in the routing table?

Reveal

OSPF. AD compared first: OSPF=110 vs RIP=120 в†’ OSPF wins. Metric is irrelevant across protocols. Refer back: В§4, В§10 trap #1.

Q4. What does S* 0.0.0.0/0 [1/0] via 203.0.113.1 mean?

Reveal

Static default route. S* = static + candidate default. AD=1 (static), metric=0. All packets without a more specific match go to 203.0.113.1. This IS the gateway of last resort. Refer back: В§7.

Q5. C 192.168.1.0/24 is directly connected, FastEthernet0/0 вЂ” where does this entry come from?

Reveal

Automatically when an interface is configured with an IP and is up/up. No protocol needed. Removing the interface IP or shutting the interface removes the entry. Refer back: В§3.
