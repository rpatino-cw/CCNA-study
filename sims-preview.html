CDP vs LLDP

D2.3 · Layer 2 discovery protocols
comparison

CDP

  • Vendor: Cisco proprietary
  • Standard: none
  • Multicast: 01:00:0c:cc:cc:cc
  • Default timer: 60 s hello, 180 s hold
  • Enable: cdp run (global) · cdp enable (interface)
  • Show: show cdp neighbors detail

LLDP

  • Vendor: open / IEEE 802.1AB
  • Standard: 802.1AB
  • Multicast: 01:80:c2:00:00:0e
  • Default timer: 30 s hello, 120 s hold
  • Enable: lldp run · lldp transmit/receive
  • Show: show lldp neighbors detail
What you learnCDP = Cisco gear talking to Cisco gear. LLDP = vendor-neutral, required for mixed environments. Both are L2 — frames don't cross routers. Both can leak topology to attackers — disable on edge ports.

Router Forwarding Decision

D3.2 · How a router forwards by default
decision tree

Tiebreakers stack in order. Router walks down until one rule decides.

1. Longest-prefix match
↓ if multiple match same length
2. Lowest Administrative Distance
↓ if tied (same protocol)
3. Lowest metric (cost / hops / BW)
↓ if still tied
4. Equal-cost load-balance (CEF)
What you learnRouters prefer specific over trusted over fast. A /32 static beats a /24 OSPF route every time, because longest-prefix wins before AD is even checked. Then AD picks protocol; metric breaks ties within one protocol.

TFTP vs FTP

D4.9 · TFTP/FTP capabilities
comparison

TFTP

  • Transport: UDP 69
  • Reliability: app-layer retry (lockstep)
  • Auth: none
  • Encryption: none
  • Use: device IOS image push, config backup over LAN
  • Cisco: copy running-config tftp:

FTP

  • Transport: TCP 20 (data) / 21 (control)
  • Reliability: TCP
  • Auth: username/password (plaintext)
  • Encryption: none (use SFTP/SCP)
  • Use: bulk file transfer, larger size, auth required
  • Cisco: copy ftp: flash:
What you learnTFTP is fast + simple but unauthenticated and unreliable for big files. FTP supports auth but plaintext. For secure file moves use SCP (TCP 22) or SFTP (TCP 22). Internal lab: TFTP. Anything crossing untrusted networks: SCP.

Wireless Security Evolution

D5.9 · WPA, WPA2, WPA3
timeline
WEP (1997) — broken

RC4 + 40/104-bit keys. Crackable in minutes. Never use.

WPA (2003) — interim

TKIP keying on top of WEP hardware. Patched WEP flaws but still weak.

WPA2 (2004) — current baseline

AES/CCMP encryption + 4-way handshake. PSK or Enterprise (802.1X). Vulnerable to KRACK (2017, patched).

WPA3 (2018) — modern

SAE replaces 4-way handshake (forward secrecy). Mandatory PMF. WPA3-Enterprise: 192-bit suite.

What you learnWPA2 = AES/CCMP + 4-way. WPA3 = SAE (Simultaneous Authentication of Equals) — resists offline dictionary attacks. PSK (pre-shared key) for home; Enterprise (802.1X + RADIUS) for corp.

Manual vs Automated Network Mgmt

D6.1 · Automation impact
before/after

Manual

  • SSH into each device individually
  • Copy/paste configs
  • Errors propagate from typos
  • No drift detection
  • Slow, doesn't scale past ~50 devices
  • Audit trail = paper / memory

Automated

What you learnAutomation reduces human-error, enforces standards, and turns "configure" into "deploy." Source of truth lives in Git → CI validates → tool (Ansible/Terraform/IaC) pushes → telemetry confirms. CCNA expects you to recognize the shift, not write code.
