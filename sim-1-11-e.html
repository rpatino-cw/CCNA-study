SSID — The Wi-Fi Network's Namedrill

An SSID is a 1–32 octet network name carried in the beacon's SSID information element. It's not security, it's a label. One AP can broadcast multiple SSIDs (each its own BSSID, each pinned to a VLAN). Hiding an SSID just zeroes the field in the beacon — probe requests still leak the name in plaintext. Build SSIDs below and watch what a client scan sees.

SSID vs BSSID vs ESSID

SSID

The human-readable name (1–32 octets, case-sensitive). One per WLAN. Examples: Corp-WiFi, Guest, IoT-2.4.

BSSID

The radio's MAC address. One BSSID per SSID per radio. A multi-SSID AP broadcasts multiple BSSIDs (often virtualized by flipping the locally-administered bit).

ESSID = Extended Service Set ID — a single SSID broadcast by many APs. Lets clients roam across coverage as if it's one big Wi-Fi.

Faded prompt — your turn

  1. Add a 4th SSID called Voice on VLAN 40, broadcast on, security WPA2-Enterprise.
  2. Confirm the AP now broadcasts 4 BSSIDs (one per SSID per radio).
  3. Hide the Corp-WiFi SSID. Note: client scan now lists (hidden). Question: did this make the network safer?

Answer: No. Hidden SSIDs disable beacon broadcast but every probe request from a known client still includes the SSID in plaintext. Real security comes from WPA2/WPA3 + strong PSK or 802.1X — not from hiding the name.
