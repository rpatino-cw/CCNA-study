The Pipeline — | filters any show output
The pipe takes the full output of a show command and runs it through a filter, exactly like a Unix pipe. Memorize these four.
show run | section ospf→only the OSPF config block (and its sub-lines)
show run | include ip route→only lines containing that text (grep)
show ip int brief | exclude unassigned→drop matching lines (hide the noise)
show run | begin interface→start output at the first match, show rest
