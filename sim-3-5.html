FHRP Sim — HSRP / VRRP / GLBP

Two routers share a virtual IP as the default gateway. Step through HSRP states. Trigger active failure → watch standby promote in <1s. Compare HSRP vs VRRP vs GLBP MACs and behavior.
WHY this matters Every host on a subnet has one default gateway IP burned in. If that gateway dies, every host loses internet. FHRPs let two (or more) routers share a virtual IP — and a virtual MAC — so hosts never know which physical box answered. HSRP, VRRP, and GLBP all solve this; they differ in vendor, election rules, and load-sharing behavior. This is the only protocol family where sub-second failover is the entire product.
