DHCP cheatsheet
  • DORA: Discover (broadcast) → Offer (server unicast/broadcast) → Request (broadcast) → Ack (server). Then client owns the lease.
  • Ports: server uses UDP 67, client uses UDP 68. Discover: src 0.0.0.0:68 → dst 255.255.255.255:67.
  • Relay agent: ip helper-address X.X.X.X on the inbound interface forwards client broadcasts as unicast to server X. Without it, DHCP doesn\'t cross routers.
  • Renewal: at T1 (50% of lease) client tries to renew with the same server (unicast). At T2 (87.5%) it broadcasts to any server. At lease expiry, it must DORA again.
  • Conflict: server pings the offered IP before issuing. Client also ARPs after Ack — if reply, sends DHCP DECLINE and starts over.
