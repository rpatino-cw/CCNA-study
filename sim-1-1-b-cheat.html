1. The 30-Second Answer read this twice

A switch forwards Ethernet frames inside a single broadcast domain (or VLAN). L2 switches decide on destination MAC. L3 switches do that PLUS route between VLANs at line rate in hardware.
  • L2 switch = Layer 2. Reads dst MAC, looks in CAM table, forwards out one port. If unknown → flood. If broadcast → flood. Doesn't care about IP.
  • L3 switch = L2 + L3. Has SVIs (Switched Virtual Interfaces) per VLAN, builds an IP routing table, runs OSPF/EIGRP, routes inter-VLAN in ASIC at wire speed.
  • One switch = one collision domain per port (full-duplex). One VLAN = one broadcast domain. Add VLANs to subdivide.

Memorize: L2 = MAC + flood. L3 = also IP + route.

2. Anatomy at a Glance visual

L2 switching (CAM table). Optional L3 routing (SVIs + RIB) for multilayer. VLANs partition broadcast domains. PoE supplies powered devices. STP prevents loops. CDP/LLDP discover neighbors.

3. Frame Flow Through an L2 Switch memorize

1
Receive frame on a port. Read source MAC + destination MAC.
2
Learn: add (or refresh) source MAC → port mapping in CAM table. Aging timer = 300s default (5 min).
3
Decide on destination MAC:
  • Known unicast (in CAM) → forward out THAT port only
  • Unknown unicast (not in CAM) → flood out every port except the inbound
  • Broadcast (FF:FF:FF:FF:FF:FF) → always flood
  • Multicast → flood (unless IGMP snooping is configured)
4
Forward unchanged. No L3 inspection. No TTL change. No L2 header rewrite (unless dot1q tag added/stripped at trunk boundary).
L2 switch never touches IP / TTL. Frame travels intact except for VLAN tag at trunk transitions.

Trap to recognize
Unknown unicast = flood (NOT drop). Cisco loves: "Switch receives a frame for an unknown destination MAC. What happens?" → Flood out all ports except inbound (within VLAN).

5. L3 Switch — What's Different SVI = key concept

Pure L2 switch

  • Forwards by MAC only
  • NO routing table
  • Single mgmt IP (one SVI for VLAN 1 typically)
  • Inter-VLAN requires external router
  • Cisco fit: Cat 1000, older 2960

L3 (multilayer) switch

  • Forwards by MAC AND routes by IP
  • Routing table (RIB) like a router
  • SVIs per VLAN — each VLAN gets its own L3 IP gateway
  • Inter-VLAN in ASIC at line rate
  • Runs OSPF/EIGRP/BGP (license dependent)
  • Cisco fit: Cat 9300, 9400, 9500

SVI = Switched Virtual Interface. interface vlan 10 + ip address 10.10.10.1 255.255.255.0 = the default gateway for hosts in VLAN 10. L3 switch routes between SVIs in hardware.

6. Mnemonics memorize

L2 switch decision tree (3 paths)
KNOWN → forward · UNKNOWN → flood · BROADCAST → flood

"Known one port. Unknown all ports. Broadcast always."

CAM table magic numbers
300s aging · per-VLAN entries · learned from src MAC
L3 vs L2 quick test
Has SVI + RIB → L3. Just CAM → L2.
Switching modes (decreasing latency, increasing risk)
Store-and-forward → Fragment-free → Cut-through

Store-and-forward = full frame check (catches errors, modern default). Cut-through = forwards after reading dst MAC (lowest latency, may forward errors). Fragment-free = waits for first 64 bytes (catches collision fragments).

show mac address-table

SW1# show mac address-table Mac Address Table Vlan Mac Address Type Ports ---- ----------- ---- ----- 10 aabb.cc00.0001 DYNAMIC Fa0/1 10 aabb.cc00.0002 DYNAMIC Fa0/2 20 ddee.ff00.0010 DYNAMIC Fa0/13 Total Mac Addresses for this criterion: 3
VlanPer-VLAN — same MAC could appear in two VLANs (rare but possible). TypeDYNAMIC = learned from frame source. STATIC = manually configured. SECURE = port-security pinned. PortsEgress port for that MAC. Port-channels show as Po1, Po2, etc.

show vlan brief

SW1# show vlan brief VLAN Name Status Ports ---- ------------- ------ -------------------------- 1 default active Fa0/24 10 DATA active Fa0/1, Fa0/2, Fa0/3 20 VOICE active Fa0/13, Fa0/14 99 MGMT active Fa0/23
Default VLANVLAN 1 = factory default. Best practice: don't use for user/voice/mgmt traffic. PortsOnly access ports show here. Trunk ports do NOT appear in `show vlan brief` (they carry all VLANs and show in `show interfaces trunk`).

show interfaces status

SW1# show interfaces status Port Name Status Vlan Duplex Speed Type Fa0/1 connected 10 a-full a-100 10/100BaseTX Fa0/2 connected 10 a-full a-100 10/100BaseTX Fa0/24 connected trunk a-full a-1000 10/100/1000BaseTX Fa0/23 notconnect 99 auto auto 10/100BaseTX
Statusconnected = link up, both sides good. notconnect = no signal (no cable / no peer). err-disabled = port-security/BPDU guard tripped — recovery needed. show ip route on an L3 switch

SW1(L3)# show ip route connected C 10.10.10.0/24 is directly connected, Vlan10 L 10.10.10.1/32 is directly connected, Vlan10 C 10.10.20.0/24 is directly connected, Vlan20 L 10.10.20.1/32 is directly connected, Vlan20

Connected routes via Vlan10 / Vlan20 = SVI = L3 switch routing inter-VLAN. Pure L2 switch wouldn't have any of this output.

9. Cisco's Favorite Traps spot the wrong answer

Trap #1 — "Switches decrement TTL"
Wrong. L2 switches don't touch L3 headers. TTL is decremented by routers (and L3 switches when they ROUTE between VLANs). Pure L2 forwarding leaves the IP header alone.
Trap #2 — "Unknown unicast = drop"
Wrong. Unknown unicast = flood out all ports in that VLAN except the inbound. Drop only happens with explicit security (port-security, ACL).
Trap #3 — "Pure L2 switch can route inter-VLAN"
Wrong. Pure L2 switch cannot route. You need an L3 switch (with SVIs) or an external router (router-on-a-stick).
Trap #4 — "Switch builds an ARP cache"
Mostly wrong. L2 switches don't ARP for forwarding (they read existing dst MAC, no IP-to-MAC resolution needed). L3 switches DO have ARP (for SVI routing). If question says "pure L2 switch builds ARP cache" → wrong.
Trap #5 — "MAC table same as ARP cache"
Wrong. MAC table = (VLAN, MAC) → port. Lives on switches. ARP cache = IP → MAC. Lives on routers + L3 switches + endpoints. Different jobs at different layers.
Trap #6 — "All switches block broadcast traffic"
Wrong. Switches FLOOD broadcast traffic within the VLAN. Routers block broadcasts (don't forward across L3 boundaries by default). VLANs limit the broadcast domain.
Trap #7 — "Cut-through is always faster + better"
Faster, yes. But cut-through forwards before verifying the FCS — corrupted frames may propagate. Modern campus switches default to store-and-forward for safety.

10. Top 10 Must-Know if-only-this

Memorize until automatic
  1. L2 switch decision = destination MAC, looked up in CAM table per VLAN.
  2. Unknown unicast or broadcast = flood out all ports in VLAN except inbound.
  3. L2 switches do NOT change TTL or any L3 header. They forward at L2 only.
  4. MAC table aging timer = 300 seconds (5 min) default.
  5. L3 switch = L2 + L3 routing in hardware via SVIs. interface vlan 10 + IP = SVI.
  6. One VLAN = one broadcast domain. Switches break collision domains (per port), VLANs break broadcast domains.
  7. show mac address-table = L2 switch core command. show vlan brief = VLAN/port assignments. show interfaces status = port speed/duplex/access-trunk.
  8. Trunk ports do NOT appear in `show vlan brief` (they carry all VLANs).
  9. Catalyst 9300 = modern access. 9500 = core/dist. 1000/2960 = legacy/SMB. Nexus = DC.
  10. Switching modes: store-and-forward (default) > fragment-free > cut-through. Modern default = store-and-forward.

11. Self-Check no peeking

Q1. A switch receives a frame with a destination MAC it doesn't have in the CAM table. What happens?

Reveal

Flood out all ports in the same VLAN except the inbound port. Unknown unicast = flood. Drop only happens with port-security or ACL. Refer back: §3, §9 trap #2.

Q2. Default MAC address table aging timer on Cisco IOS?

Reveal

300 seconds (5 minutes). Refresh on every frame seen with that source MAC. Refer back: §4.

Q3. A VLAN-10 access port and a VLAN-20 access port. Hosts on each. Without a router or L3 switch, can they communicate?

Reveal

No. Different VLANs = different broadcast domains. Pure L2 switch cannot route between them. Need L3 switch (SVI) or external router. Refer back: §5.

Q4. An L2 switch decrements TTL by 1 on every forwarded frame. True or false?

Reveal

False. L2 switches don't touch L3 headers. Routers + L3 switches (when routing) decrement TTL. Refer back: §9 trap #1.

Q5. Which command shows VLAN-to-port assignments?

Reveal

show vlan brief. Note: only ACCESS ports appear here. Trunk ports show via `show interfaces trunk`. Refer back: §7.
