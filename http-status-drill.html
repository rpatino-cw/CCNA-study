HTTP Status Code Drill

Status codes a REST API returns: 1xx informational, 2xx success, 3xx redirect, 4xx client error, 5xx server error. The trap on the exam: 301 vs 302 (permanent vs temporary), 401 vs 403 (no creds vs creds-but-denied), 500 vs 503 (any server fault vs overloaded/down).

HTTP status code cheatsheet
1xx info 2xx success 3xx redirect 4xx client error 5xx server error
  • 200 OK — request succeeded, body has the data (default for GET).
  • 201 Created — POST/PUT made a new resource. Location header points to it.
  • 204 No Content — success but no body (typical for DELETE).
  • 301 Moved Permanently — resource lives at a new URL forever. Update bookmarks.
  • 302 Found — temporary redirect. Keep using the original URL.
  • 304 Not Modified — cached copy still valid (response to If-Modified-Since).
  • 400 Bad Request — malformed syntax, invalid JSON, missing required field.
  • 401 Unauthorized — missing or invalid auth (token / API key / creds).
  • 403 Forbidden — auth was valid but you're not allowed to do this.
  • 404 Not Found — URI doesn't match any resource.
  • 405 Method Not Allowed — verb wrong for that resource (e.g., DELETE on read-only).
  • 429 Too Many Requests — rate-limited. Retry after the Retry-After header.
  • 500 Internal Server Error — generic "something blew up server-side" catch-all.
  • 502 Bad Gateway — upstream server returned junk to the gateway/proxy.
  • 503 Service Unavailable — server is overloaded or down for maintenance.
