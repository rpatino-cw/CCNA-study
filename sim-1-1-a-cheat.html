1.1.a Routers — Read This Before Anything Else

1. The 30-Second Answer read this twice

A router is a Layer-3 device that forwards IP packets between different IP networks by reading the destination IP, looking it up in its routing table, and sending the packet out the right interface toward the next hop.

Three things make a router a router:

  • Layer 3 — operates on IP, not MAC. Switches forward by MAC inside one network; routers forward by IP between networks.
  • Routing table (RIB) — the brain. Built from connected routes, static routes, and dynamic protocols (OSPF/EIGRP/BGP/RIP).
  • One interface = one IP subnet = one broadcast domain. Every interface is its own network. Routers BREAK broadcast domains; switches EXTEND them.

If you remember nothing else: router = L3 + routing table + per-interface subnet.

2. Anatomy at a Glance visual

Two interfaces (LAN + WAN — could be many more). Two tables (RIB built by control plane, FIB built FROM RIB and used by data plane). Three feature engines (ARP/ACL/NAT). Two planes (CPU does decisions, ASIC does forwarding).

3. Packet Flow Through a Router memorize the 6 steps

A packet arrives at Gi0/0. Walk every step:

1
Receive frame — physical bits become an Ethernet frame. Router reads the dst MAC. If it's the router's own MAC, accept; if not, drop.
2
Strip L2 header — discard the Ethernet header. Hold the IP packet payload.
3
Look up dst IP in FIB — longest-prefix match wins. Get the egress interface + next-hop IP. Decrement TTL by 1. If TTL=0, drop and send ICMP Time Exceeded.
4
ARP for next-hop MAC — check ARP cache. If not cached, send ARP request, wait for reply.
5
Build new L2 header — src MAC = router's egress interface MAC, dst MAC = next-hop's MAC. Recompute IP checksum (TTL changed).
6
Forward out egress interface — frame goes on the wire toward the next hop. Repeat at every router until destination reached or TTL=0.
Per hop: TTL decrements by 1, L2 header is rewritten, IP header (src + dst) is unchanged.

4. Components Glossary one-line each

ComponentWhat it does
RIBRouting Information Base — the master routing table. Best route per prefix from all sources (connected, static, dynamic protocols).
FIBForwarding Information Base — pre-computed lookup table in TCAM. Built FROM RIB. The data plane uses this for line-rate per-packet lookup.
ARP cacheMaps next-hop IPs to MAC addresses. Required to build the new L2 header at each hop. Aging timer ~4h default.
Control planeCPU. Runs OSPF, BGP, EIGRP, ARP request/reply, ICMP, management. Builds the RIB. "Slow path."
Data planeASIC + TCAM. Per-packet lookup using FIB. Line rate. "Fast path." Doesn't decide — just executes.
Management planeSSH, SNMP, NETCONF, syslog, console. How humans + tools talk TO the router.
InterfacesPhysical (Gi/Te/Fa) or logical (loopback, tunnel, sub-interface). Each has its own IP + subnet + broadcast domain.
ACL enginePer-interface, per-direction packet filter. Standard (1-99) match src IP. Extended (100-199) match 5-tuple.
NAT engineInside-local ↔ inside-global translation. Static, dynamic, or PAT/overload. Edge routers always run NAT.

9. Edge vs Core vs Branch Roles when asked "which router"

Edge Router

  • Sits at internet boundary
  • Runs BGP with ISP
  • NAT private→public
  • ACL filters inbound
  • Often paired with NGFW
  • Cisco fit: ISR / Cat 8000

Core / Distribution

  • Inside the campus
  • High-speed L3 transit
  • Runs OSPF / EIGRP / IS-IS internally
  • NO NAT (private throughout)
  • Minimal ACL (trust inside)
  • Cisco fit: ASR / L3 switches (Cat 9300/9500)

Branch Router

  • Small office
  • WAN to HQ (DMVPN, MPLS, SD-WAN)
  • Often combined w/ Wi-Fi + voice
  • Static default route to HQ
  • Cisco fit: ISR 1000/4000

10. Cisco's Favorite Traps spot the wrong answer

Trap #1 — "Switch can route too"
If you see "L3 switch" in options, READ THE QUESTION. If it says "between IP networks at line rate in hardware" → L3 switch is often the BEST answer, not router. If it says "WAN edge with NAT + BGP" → router. Cisco distinguishes deliberately.
Trap #2 — "Hub vs switch vs router"
Hub = L1 (single collision domain, no learning). Switch = L2 (per-port collision, MAC table). Router = L3 (per-interface broadcast domain, IP routing). When question says "broadcast domains" → count router interfaces + switch VLANs.
Trap #3 — "MAC table on a router"
Routers DO NOT have a per-port MAC table. They have an ARP cache (next-hop IP → MAC). If an option says "the router builds a MAC address table per port" — wrong. That's a switch.
Trap #4 — "TTL doesn't change"
Wrong. TTL decrements by 1 at every router hop. Switches (L2) don't touch TTL. If TTL hits 0, packet is dropped + ICMP Time Exceeded sent to source.
Trap #5 — "IP source / dest changes per hop"
Wrong. L3 src/dst IP stays the same end-to-end. Only NAT (at edge boundary) rewrites. L2 src/dst MAC rewrites at every hop.
Trap #6 — "Static route AD is 0"
Wrong. Connected = 0. Static = 1. They're close but not equal. "Floating static" = static with manually-bumped AD (e.g., 5 or 200) so it only kicks in when primary fails.
Trap #7 — "Router needs to know every internet route"
Wrong. Branch/edge routers usually have 0.0.0.0/0 default route pointing at ISP. Only ISP-tier routers run full BGP table (~900K+ routes). CCNA scenarios use default routes for outbound.

11. Top 10 Things You MUST Know Cold if-only-this

Memorize until automatic — close exam answers come from these
  1. Router = L3 device, decision unit = destination IP, longest-prefix match.
  2. Each interface = own subnet + own broadcast domain. 4 interfaces in 4 subnets = 4 broadcast domains.
  3. RIB built by control plane from connected/static/dynamic. FIB pushed to TCAM for line-rate.
  4. Per hop: TTL−1, L2 header rewrites, IP header stays (except NAT).
  5. AD numbers: C=0, S=1, eBGP=20, EIGRP=90, OSPF=110, RIP=120, iBGP=200. Lower wins.
  6. Routing protocols are control plane. ASIC forwards using FIB. Don't confuse the two.
  7. ARP cache is per-interface IP→MAC, used to build new L2 header at each hop. Routers have ARP, NOT MAC tables.
  8. NAT lives at the edge router. ACL lives per-interface, per-direction.
  9. show ip route, show ip int brief, show ip arp, show version — read fluently.
  10. ISR = branch/SMB. ASR = enterprise core. Cat 8000 = modern SD-WAN. Pick by use case in exam scenarios.

12. Quick Self-Check no peeking

Read the question. Try to answer in your head. Click to reveal. If you got 5/5 cold, this section locked. If not, go back to the indicated section.

Q1. A router has 4 interfaces, each in a different IP subnet. How many broadcast domains?

Reveal answer

4. Each router interface = its own broadcast domain. Routers BREAK broadcast domains. Refer back: §1, §11 #2.

Q2. OSPF and EIGRP both teach R1 about 10.5.5.0/24. Which gets installed in RIB?

Reveal answer

EIGRP — AD 90 beats OSPF AD 110. Lower AD wins. Refer back: §6.

Q3. What is the AD of a static route?

Reveal answer

1. (Connected = 0, Static = 1, eBGP = 20.) Refer back: §6 — memorize the table.

Q4. A packet with TTL=1 arrives at R1. What happens?

Reveal answer

R1 decrements TTL to 0, drops the packet, and sends an ICMP Time Exceeded back to the source. This is how `traceroute` discovers routers — it sends packets with increasing TTLs (1, 2, 3...) and reads the ICMP Time Exceeded replies. Refer back: §3.

Q5. Which output identifies a router (vs L2 switch)?

Reveal answer

show ip route with routing-protocol entries (O/D/B/R/S codes) = routing table = router (or L3 switch). Pure L2 switches have no routing table — `show ip route` output is empty or doesn't exist. Also: per-interface IPs in show ip int brief = router. Refer back: §7.
