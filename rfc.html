RFCs You'll See on the CCNA

Every protocol on the exam was born as a Request For Comments — a numbered spec the IETF publishes. The number is just a sequential ID, but a few of them show up on the test as if they were vocabulary.

Why this matters for the exam

Cisco rarely asks you to recite an RFC number, but they routinely quote one in a question stem — "per RFC 1918, which of these is private?" If you don't recognize the number you waste seconds parsing it. Treat them like port numbers: pure recall, hacked into memory with mnemonics.

This page covers 30 RFCs grouped into 10 categories. The only one Cisco truly expects you to know cold is 1918. The rest you only need to recognize on sight.

IPv4 core Private / special DHCP NAT Routing FHRP IPv6 Network services Multicast QoS

The hack — why these numbers feel random

RFC numbers are sequential publication IDs, not topical. RFC 791 (IPv4) and RFC 793 (TCP) are close because they were published the same week in 1981. RFC 8200 (IPv6) is bigger because it was assigned 35 years later. There's no system. That's why mnemonics are the only way — you're memorizing arbitrary tokens.

Tip: When two RFCs are published together (791 IPv4 + 793 TCP, 4861 ND + 4862 SLAAC), remember them as a pair. Consecutive numbers = the WG shipped them on the same day.
