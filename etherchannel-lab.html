EtherChannel Lab — bundle 2-8 links into one logical pipe

Drag cables between switches, pick a mode per port, apply, and watch the bundle form (or fail). Then verify with the on-board IOS console — exactly the way you would in real life.

Protocols: LACP · PAgP · Static Bundle min: 2 · max: 8 Logical interface: Port-channel N
1 · The three protocols

LACP

Link Aggregation Control Protocol · IEEE 802.3ad · open standard
  • Negotiates the bundle by exchanging LACPDUs
  • Works on any vendor — Cisco, Arista, Juniper, NICs
  • Two modes: active (sends LACPDUs) and passive (only replies)
active passive

PAgP

Port Aggregation Protocol · Cisco proprietary
  • Negotiates the bundle by exchanging PAgP packets
  • Cisco-only (catalyst-on-catalyst)
  • Two modes: desirable (initiates) and auto (only replies)
desirable auto

Static

No negotiation · "on/on"
  • Forces the bundle without keepalives
  • Misconfig is dangerous — can create a Layer-2 loop
  • One mode: on · both ends must be set
on
Rule of thumb. Use LACP active/active when in doubt — fastest negotiation, vendor-neutral, default in 99% of modern designs. PAgP is legacy Cisco. Static (on) only when the other side cannot speak LACP/PAgP.

Tap any cell — green = bundle forms, white = bundle fails. The diagonal is most important: passive/passive and auto/auto never bundle (both sides waiting).

6 · Exam cheat-notes
Memory hooks. "Active wants to fight, passive waits to be hit." "Desirable desires, auto answers." Two passives or two autos → silence → no bundle.
Bundle requirements (must match on every member, both ends): speed, duplex, native VLAN, allowed VLAN list, trunking encapsulation, MTU, channel-group number, switchport mode (access vs trunk).
Common gotchas. ① Configuring channel-group before applying common settings can put the port in err-disabled. ② You cannot mix LACP and PAgP on the same channel-group. ③ Static (on) skips negotiation — both ends must be on, or you have a loop.
Load-balancing (informational). port-channel load-balance src-dst-ip hashes per flow — a single flow takes a single physical link. Adding more members never speeds up one flow; it gives parallel flows more capacity.
