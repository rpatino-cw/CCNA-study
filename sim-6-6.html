An LLM predicts the next token. The prompt shapes the prediction — vague prompt → vague output. Hallucination = confident-sounding fiction. RAG = bolt-on real docs so the model cites instead of guesses.

Hallucination detector

An LLM fills in plausible-sounding tokens when it doesn't know. Highlighted spans below are the kinds of factual claims most likely fabricated. Always verify before action.

"Cisco IOS 15.7(3)M4 introduced the new RFC 9847 OSPFv4 protocol, replacing OSPFv2. The default hello timer changed from 10 to 7 seconds."
Each highlighted claim is unverified — IOS 15.7(3)M4, RFC 9847, hello timer change. Real or fabricated? Without checking, you'd never know.

RAG — Retrieval-Augmented Generation

RAG = LLM + your docs. Before answering, the system retrieves the k most relevant chunks from a vector DB and injects them in the prompt. Output cites your docs instead of inventing.
