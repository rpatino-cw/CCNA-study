DR/BDR Election + LSA Flooding

Interactive: elect the DR/BDR on a shared segment, then watch LSAs flood through a multi-area topology.

Part A

DR/BDR Election: Multi-Access Segment

On a broadcast network (e.g. Ethernet), OSPF elects one Designated Router (DR) and one Backup DR (BDR). All others become DROTHER. Priority 0 permanently excludes a router from election. The election is non-preemptive: a higher-priority router arriving after the election does NOT take over.

Router Priority (0-255) Router-ID Role
Configure priorities and router-IDs, then click Run Election.
Non-preemptive: Router R5 (priority 255) joined the segment after the DR/BDR election already ran. It stays DROTHER. The current DR holds its role until it fails or OSPF resets. This is exam-critical.
Tie-break rule: Highest priority wins. If priorities are equal, the router with the highest Router-ID wins. A router with priority 0 can never become DR or BDR.
Part B

LSA Flooding + Area-Type Filtering

OSPF routers flood Link-State Advertisements (LSAs) within and between areas. Each LSA type has a specific originator and a specific scope. Area type determines which LSAs are permitted to enter an area. Select an area type, then trigger a link change to watch the flood.

LSA Originator Reference

TYPE 1: ROUTER LSA
Every Router
Describes links and costs. Flooded within the originating area only.
TYPE 2: NETWORK LSA
DR (on broadcast)
Lists routers on the multi-access segment. Area-local scope.
TYPE 3: SUMMARY LSA
ABR
Advertises intra-area prefixes into other areas. Blocked by totally stubby.
TYPE 4: ASBR SUMMARY
ABR
Tells routers how to reach the ASBR. Blocked in stub/totally-stubby/NSSA.
TYPE 5: EXTERNAL LSA
ASBR
Redistributed external routes. Flooded everywhere except stubs. AS-wide scope.
TYPE 7: NSSA EXTERNAL
ASBR in NSSA
Like T5 but stays within the NSSA. ABR converts T7 to T5 at the boundary.

Exam Walk-Away

  • DR = highest priority, tie-break = highest Router-ID. Priority 0 = forever ineligible.
  • Election is non-preemptive. A better router joining later stays DROTHER.
  • Every router originates T1. The DR originates T2. The ABR originates T3 and T4. The ASBR originates T5 (T7 in NSSA).
  • Area 0 is mandatory. All inter-area traffic must traverse the backbone (no area-to-area shortcuts).
  • Stub: blocks T4+T5, injects a default route. Totally Stubby: also blocks T3. NSSA: blocks T5 but allows T7 from a local ASBR, which the ABR converts to T5.
  • T7 never leaves the NSSA. The ABR translates it to T5 before flooding into area 0.
