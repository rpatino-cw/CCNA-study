WLAN Authentication Simdrill

PSK = one shared password (your home Wi-Fi). Dead simple, but everyone holds the same key — fire one employee, you rotate everyone's. 802.1X / EAP = per-user creds via RADIUS. Three actors: supplicant (you), authenticator (AP), authentication server (RADIUS).

WPA2-Personal vs WPA2-Enterprise

Personal (PSK)

  • One pre-shared key everyone uses
  • Setup: type SSID + password
  • Scale: home, small office (<25 ppl)
  • Risk: offline brute-force on captured handshake
  • Rotation: change password = reconfigure every device

Enterprise (802.1X / EAP)

  • Each user = unique credentials
  • Setup: AP + RADIUS + user store (AD/LDAP)
  • Scale: medium → enterprise
  • Risk: depends on EAP variant — EAP-TLS strongest
  • Rotation: revoke one user, others unaffected

802.1X actors

Three roles. The AP doesn't decide who passes — it just relays. The RADIUS server is the actual brain.

RADIUS (ISE/NPS)

4-way handshake (after auth, both PSK + Enterprise)

After the auth phase, both client and AP derive a PMK (Pairwise Master Key). The 4-way handshake derives session keys without ever sending the PMK over the air.

1
AP → Client: ANonce (random nonce)
2
Client → AP: SNonce + MIC (proves PMK knowledge)
3
AP → Client: GTK + install signal
4
Client → AP: ACK — keys installed, traffic encrypted
