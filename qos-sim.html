The one-line answer

QoS = give important traffic priority over the rest. Classify it (NBAR / ACL), mark it (DSCP / CoS), queue it (LLQ for voice, CBWFQ for video, FIFO for the rest), and either drop or shape what doesn't fit.

The four enemies: bandwidth, delay, jitter, loss. Voice hates delay+jitter+loss. Video hates jitter+loss. Bulk data only hates loss. QoS lets each get what it actually needs.

Exam tip — five queueing strategies FIFO first-in-first-out, no fairness · PQ high-priority always served first (starves low) · WFQ weighted fair share automatically · CBWFQ manual class-based bandwidth guarantees · LLQ = CBWFQ + a strict-priority queue (voice) that's also policed so it can't starve the rest. Exam tip — police vs shape Police = drop or remark excess. Inbound direction. No buffer, no delay, but loss spikes for TCP. Shape = buffer excess and release at CIR. Outbound direction. Adds delay but smooths the flow. Cisco mnemonic: "police inbound, shape outbound." Both use a token bucket; the difference is what happens when the bucket is empty.
Exam tip — congestion avoidance Tail drop = once the queue fills, drop everything. Triggers TCP global synchronization — every flow backs off together, link goes idle, all ramp up together, queue refills, repeat. WRED (Weighted Random Early Detection) drops random packets before the queue is full, scaled by drop probability per DSCP class. Different flows back off at different times → link stays utilized.

Queueing decision tree

Voice → LLQ (strict priority + policer). Video / mission-critical → CBWFQ with a bandwidth guarantee. Bulk / scavenger → default class with WRED. Never PQ alone in production — it starves everything below the top queue.

Marking — where

Switch (Layer 2) marks CoS in the 802.1Q tag (3 bits, 0-7). Router (Layer 3) marks DSCP in the IP ToS byte (6 bits, 0-63). End-to-end QoS = mark DSCP at the edge, every router/switch trusts it from there. AutoQoS or mls qos trust device cisco-phone creates the trust boundary.
