Routing & Forwarding · Layer 2 ↔ Layer 3

Network Layer vs. Data Link Layer

How a router uses Layer 3 to decide where a packet goes, and Layer 2 to deliver it across each individual link.

What is the core principle?

A router makes forwarding decisions based on Layer 3 IP addresses — the destination IP is end-to-end and does not change across hops (absent NAT). To physically move the frame from one interface to the next, the router constructs a brand-new Layer 2 header using MAC addresses, which are link-local and rebuilt at every single hop. Layer 3 determines the path; Layer 2 enables delivery across each link.

Overview

OSI layer responsibilities

The three layers relevant to routing and forwarding decisions.

Layer 3

Network Layer

Uses: IP Addressing (IPv4 / IPv6)
  • Path determination
  • Routing table lookup
  • Next-hop selection
Layer 2

Data Link Layer

Uses: MAC Addressing (Ethernet)
  • Frame encapsulation
  • Local link delivery
  • Address resolution (ARP / ND)
Layer 1

Physical Layer

Uses: Physical medium
  • Bit transmission over the medium
  • Copper, fiber, wireless
  • No addressing — only signals
Process

Packet handling — step by step

What happens inside a router from the moment a frame arrives to the moment it departs.

Layer 2 step Layer 3 step Transition
1
Frame received on interface L2 The router's NIC receives raw bits. Layers 1 and 2 hand the frame up the stack.
2
Destination MAC address verified L2 The frame is accepted only if its destination MAC matches the router's interface MAC, a broadcast address, or a relevant multicast address.
3
Layer 2 header removed Transition The Ethernet header and trailer are stripped. The bare IP packet is passed up to Layer 3 for processing.
4
IP header examined L3 The router reads the destination IP address. TTL is decremented by 1 and the IPv4 header checksum is recalculated.
5
Routing table lookup performed L3 The longest-prefix match algorithm selects the most specific matching route and its associated outgoing interface and next hop.
6
Next hop determined L3 The router identifies the IP address of the next device — either a neighboring router or the final destination host on a directly connected network.
7
Layer 2 address resolved Transition ARP (IPv4) or Neighbor Discovery (IPv6) maps the next-hop IP address to a MAC address. The result is cached in the ARP/neighbor table.
8
New Layer 2 header constructed L2 A fresh Ethernet header is built: the router's outgoing interface MAC becomes the source, and the resolved next-hop MAC becomes the destination.
9
Frame transmitted out interface L2 The re-encapsulated frame is passed to Layer 1 and sent toward the next hop.

Address Resolution

Resolving the next-hop MAC address

Before a router can construct the new L2 header, it must resolve the MAC address of the next hop. The mechanism depends on the IP version.

IPv4 — ARP

  • Protocol: Address Resolution Protocol
  • Mechanism: Broadcast request to the local subnet
  • Query: "Who has 10.0.0.1? Tell 10.0.0.10."
  • Target replies with its MAC address (unicast)
  • Result cached in the ARP table
  • EtherType: 0x0806 — not encapsulated in IP

IPv6 — Neighbor Discovery (NDP)

  • Protocol: Neighbor Discovery Protocol (ICMPv6)
  • Mechanism: Neighbor Solicitation to solicited-node multicast
  • No broadcasts — multicast only, far more efficient
  • Target replies with a Neighbor Advertisement
  • Result cached in the Neighbor (ND) table
  • Also handles router discovery and prefix delegation
Tables

Forwarding tables used by a router

A router consults multiple tables to make and execute its forwarding decision.

Routing Table

Determines the next hop

Contains network prefixes and their associated next-hop IPs or outgoing interfaces. Populated by static routes and routing protocols (OSPF, EIGRP, BGP).

Input
Destination IP address
Output
Next-hop IP + outgoing interface

ARP / Neighbor Table

Resolves Layer 2 address for the next hop

Maps known IP addresses to their MAC addresses. Dynamic entries expire after a timeout (Cisco IOS default: 4 hours). View with show ip arp.

Input
Next-hop IP address
Output
Next-hop MAC address

NAT Table

Tracks address translations (if NAT is active)

When NAT is configured, the router rewrites source or destination IP addresses and/or ports. Normal routers forward without modifying IP addresses.

Link Types

When MAC addressing applies

MAC addressing is a characteristic of Ethernet links — it is not universal to all WAN or tunnel link types.

Link TypeMAC Addressing?Notes
Ethernet LANYesStandard MAC addressing. ARP or ND resolves the next-hop MAC.
Ethernet WANYesModern WAN links increasingly use Ethernet handoffs. Same MAC rules apply.
PPP (Serial)NoPoint-to-point — only two endpoints. No need for MAC; ARP does not run.
HDLC (Serial)NoCisco default on serial interfaces. Point-to-point, no MAC addressing.
GRE / IP TunnelsNoTunnel endpoints are IP addresses. MAC is irrelevant to the tunnel itself.
Frame Relay (legacy)NoUses DLCIs, not MACs. Largely replaced by MPLS and Ethernet WAN.
Exam Focus

Key statements to know cold

These exact concepts appear on the CCNA 200-301 exam.

"Routing is performed using IP addresses. The destination IP address does not change across hops (absent NAT)."
"Forwarding across a link requires Layer 2 encapsulation. A new MAC header is constructed at every hop."
"MAC addresses are link-local. They are used for next-hop delivery only, never for routing decisions."
"IP addresses remain consistent across hops. MAC addresses do not."
"ARP maps Layer 3 IPv4 addresses to Layer 2 MAC addresses on Ethernet networks."
"IPv6 uses Neighbor Discovery Protocol (NDP), not ARP, to resolve MAC addresses."
"TTL is decremented by 1 at each router hop. When TTL reaches zero, the packet is dropped and an ICMP Time Exceeded message is sent to the source."
"Non-Ethernet links such as PPP and HDLC do not use MAC addressing. ARP does not run on these interfaces."
Watch Out

Common mistakes

✗"The destination IP changes as the packet is routed."

Reality: The destination IP remains constant end-to-end. Only the L2 MAC addresses are rebuilt at each hop. NAT is the deliberate exception where an IP address is rewritten.

✗"Routers use MAC addresses to make routing decisions."

Reality: Routing decisions are made exclusively at Layer 3 using IP addresses. MACs are stripped on ingress and rebuilt on egress — they play no role in which route is selected.

✗"ARP is required on every type of router link."

Reality: ARP runs only on broadcast-capable Ethernet links. Point-to-point links (PPP, HDLC) have no MAC addressing, so ARP is unnecessary and does not run on them.

✗"The source MAC identifies the original sender of the traffic."

Reality: The source MAC identifies only the device that transmitted the frame on that specific link segment. It changes at every hop. The source IP identifies the original sender end-to-end.

✗"If ARP fails, the packet is just delayed until it resolves."

Reality: If the next-hop MAC cannot be resolved, the packet is dropped. The router cannot construct the L2 header without a MAC address. TCP retransmission handles recovery at the transport layer.

✗"IPv6 uses ARP the same way as IPv4."

Reality: IPv6 replaced ARP with Neighbor Discovery Protocol (NDP). NDP uses ICMPv6 Neighbor Solicitation messages sent to a solicited-node multicast address — there is no broadcast in IPv6.
