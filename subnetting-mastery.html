Section 1

Binary Primer

Binary is base-2. Each bit is either 0 or 1. The rightmost bit = 20=1, the next = 21=2, continuing left, doubling each time. An octet (8 bits) can represent values from 0 (00000000) to 255 (11111111).

IP addresses are 32 bits, written as four octets separated by dots. Each octet is 8 bits, giving a range of 0–255 per octet. Click the cells below to toggle bits on/off and watch the decimal value change.

The Magic Number Shortcut

The fastest way to find subnet ranges without binary math:

Block Size Formula

Block size = 256 − (last interesting octet of mask)
Subnets start at multiples of the block size.

Subnetting Steps

Animated walkthrough. Problem: 172.16.0.0 /16 — need 6 subnets. Step through the process.

Step 0 of 5
1

How many bits to borrow?

We need 6 subnets. Find the smallest n where 2n ≥ 6.
21=2   22=4   23=8 ≥ 6 → borrow 3 bits

2

New prefix length

Original /16 + 3 borrowed bits = /19

3

Block size

/19 mask in the third octet = 224 (11100000).
Block size = 256 − 224 = 32 (increments in the third octet)

4

List subnet ranges

172.16.0.0/19 → .31.255
172.16.32.0/19 → .63.255
172.16.64.0/19 → .95.255
172.16.96.0/19 → .127.255
172.16.128.0/19 → .159.255
172.16.160.0/19 → .191.255

5

Hosts per subnet

Host bits remaining = 32 − 19 = 13 bits.
Usable hosts = 213 − 2 = 8192 − 2 = 8190 hosts per subnet

Route Summarization

Summarize multiple specific routes into one advertisement. Convert all networks to binary, find the longest common prefix across all of them — that length becomes the summary route's CIDR.

Example: summarize these 4 routes into one supernet:

Section 10

Exam Speed Tips

Eight techniques to solve subnetting questions in under 60 seconds on exam day.

1. Powers of 2

Memorize these before exam day. All subnetting math reduces to them.

2 4 8 16 32 64 128 256 512 1,024 2,048 4,096

2. Block Size Shortcut

Block = 256 − last interesting mask octet. Subnets start at multiples of the block size. No binary needed.
/28 → 256−240 = 16 → .0, .16, .32, .48…

3. Host Count Formula

Usable hosts = 2(host bits) − 2
Subtract 1 for network address, 1 for broadcast.
/27 → 32−27=5 host bits → 2⁵−2 = 30 usable

4. VLSM Order

Always allocate the largest subnet first. Work top-down from most hosts needed to fewest. Skipping order causes gaps and fragmentation.

5. /31 and /32

/31 = point-to-point link (RFC 3021). No broadcast concept — both addresses are usable. Used on router-to-router links.
/32 = single host. Used for loopbacks and null routes.

6. Wildcard Derivation

Wildcard = 255.255.255.255 − subnet mask (invert each octet).
255.255.255.224 → 0.0.0.31
Used in ACLs and OSPF network statements.

7. Same Subnet Check

Apply the mask to both IPs. If the network addresses match, they're in the same subnet.
10.0.0.5 AND 10.0.0.9 → /29?
Block=8: .0–.7 vs .8–.15 → different subnets

8. The Critical 4: /26–/29

These 4 prefixes are the most tested on the CCNA. Know them cold:

/26 = 64 /27 = 32 /28 = 16 /29 = 8
