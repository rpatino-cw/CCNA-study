IPv6 Subnetting Guide

The problem

Your ISP just delegated you this prefix. You need 4 VLANs for the house. Where do you start?

2001:db8:1234:ab00::/56
✗ Try 1: 2001:db8:1234:ab00::/57 — non-nibble step. Math works, hex stops aligning. Nobody does this.
✗ Try 2: 2001:db8:1234:ab00::/68 — past /64. Breaks SLAAC. Hosts can't autoconfigure.
✓ Try 3: 2001:db8:1234:ab00::/64, ab01::/64, ab02::/64, ab03::/64 — nibble-aligned, /64 LANs, subnet ID = the changing hex digits.

By the end of this page, the third answer should feel obvious — readable straight from a single table.

From a /48 — borrow nibbles, get children

Memorize one row. The pattern is "each nibble = ×16 children." Hot row = the most common deployment.

How to read it: Pick the prefix you have (left). Slide right one column = your subnet field size. Two columns over = how many of those subnets fit in a /48. The "Hex changes" column tells you which characters in hextet 4 are yours to set. Sanity check: Subnets-in-/48 × /64s-underneath = 65,536 every row. Universal formulas: # subnets = 2^(new − old). # /64s under any prefix /N = 2^(64 − N). Why /64? SLAAC + EUI-64 need exactly 64 host bits (RFC 4291). Going smaller breaks autoconfig. Why nibble? Each hex digit = 4 bits. /4 steps keep math readable in hex without dropping to binary.
Anchor · You already know how to do this

Same operation as IPv4 — bigger numbers, hex instead of decimal

If you can borrow bits in IPv4, you can borrow bits in IPv6. The only differences: alignment is 4 bits (not arbitrary), edge LAN is locked at /64, and you read hex instead of dotted decimal.

IPv4 — what you already do

Start192.168.1.0/24
Borrow2 bits
Result/26
Subnets2² = 4
Read offlast octet, multiples of 64
Children.0, .64, .128, .192
↔

IPv6 — same idea

Start2001:db8:1::/48
Borrow4 bits (one nibble)
Result/52
Subnets2⁴ = 16
Read offhextet 4, first hex digit
Children:0000::, :1000::, :2000::, … :f000::

The mental swap: IPv4 thinks in octets (8 bits, decimal 0–255). IPv6 thinks in nibbles (4 bits, hex 0–f). Smaller chunks, same arithmetic.

Walkthrough · The opening problem, solved

Solving the home /56 from the top of this page

ISP gave you 2001:db8:1234:ab00::/56. You need 4 VLANs. Use the master table.

  1. Find your row. You have /56. Master table row 3. Subnet field = 2 hex digits. /64s underneath = 256.
  2. Pick subnet IDs. Two hex digits give you 256 children (00–ff). Pick four: 00, 01, 02, 03 — sequential. Or memorable: 00, 10, 20, 30.
  3. Write the prefixes. Drop each subnet ID into the highlighted slot:
    • Trusted: 2001:db8:1234:ab00::/64
    • IoT: 2001:db8:1234:ab01::/64
    • Guest: 2001:db8:1234:ab02::/64
    • Lab: 2001:db8:1234:ab03::/64
  4. You're done. No binary conversion. No subnet mask math. Read straight from the table.

Four other ways to think about it

Method 2 · The honest one
Pure binary

Convert each hex digit to its 4-bit binary. Line up 128 bits. Mark the prefix boundary. Count host bits. Hosts = 2^(128 − prefix). Subnets = 2^borrowed.

Use when — proving the nibble trick is legit, or facing a non-aligned /50 textbook question.
Method 3 · The IPv4-brain bridge
Hex → decimal substitution

Convert the changing hex digit(s) to decimal. ab00 → 43776, ab01 → 43777. Subnets become regular incrementing numbers.

Use when — letters a-f still feel hostile. Drops away once hex stops looking foreign.
Method 4 · The pragmatist's shortcut
"Watch only hextet 4"

For 99% of /48-or-/56 → /64 designs, only the 4th hextet changes. Cover the first 3 hextets with your hand. Problem shrinks to 4 hex digits.

Use when — visual noise on exam day. Works for any /48 prefix delegation.
Method 5 · The verification habit
CLI confirm with sipcalc

sipcalc 2001:db8:abcd::/48 -S 4 splits into 16 /52s. Use to check hand math, never replace it. Install: brew install sipcalc.

Use when — building a real lab. Catches off-by-one nibble mistakes instantly.

Pitfalls — mistakes that cost you points

Subnetting on non-nibble boundaries

You can mathematically make a /50 or /53. Don't. Hex stops aligning, you lose the read-the-digit shortcut, and most calculators assume nibble boundaries.

Asking your ISP for "an IPv6 address"

If they give you a single /128, you can't subnet at all. Ask for prefix delegation — minimum /60, ideally /56 or /48. DHCPv6-PD handles it.

Counting "usable hosts" like IPv4

IPv6 has no broadcast. :: is reserved as subnet-router anycast (don't assign to a host), but every other address in a /64 is usable. No "minus 2" rule.

Going smaller than /64 on a LAN

Tempting if "you only need 4 hosts." Don't — SLAAC, ND-RA, mDNS, most v6 features quietly break. There's no shortage of /64s. Use one.

/127 used outside point-to-point

RFC 6164 permits /127 only on router-to-router links. On a user LAN, ND will misbehave. Don't.

What to actually remember

Edge LAN = /64. Subnet on /4 steps. Each nibble = ×16 children. The changing hex digit IS the subnet number. Master table covers the rest.
