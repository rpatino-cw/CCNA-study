Layer 1 — EMA proficiency

Each of the 97 objectives gets a single number 0.0–1.0. It's an exponential moving average over correctness. Recent answers count more than old ones.

// js/store.js (paraphrased) const EMA_ALPHA = 0.3; function updateProficiency(topicId, correct) { const old = getProficiency(topicId); // default 0 const next = (1 - 0.3) * old + 0.3 * (correct ? 1 : 0); setProficiency(topicId, round(next, 3)); }

One wrong answer drops you ~23 percentage points. That's intentional — fragile knowledge should not register as mastery.

Layer 2 — Mastery state machine

Proficiency alone isn't enough. A topic can be 0.85 from luck. The state machine adds quality gates: number of attempts, recent variance, optional lab completion.

stateDiagram-v2
  [*] --> NOT_STARTED
  NOT_STARTED --> PROBED: first diagnostic Q
  PROBED --> PRACTICING: first quiz attempt
  PRACTICING --> PROFICIENT: prof ≥ 0.70 over last 3+ quizzes
  PROFICIENT --> MASTERED: prof ≥ 0.85 + lab passed + FSRS gap ≥ 7d
  PROFICIENT --> PRACTICING: prof drops < 0.70
  MASTERED --> PROFICIENT: missed FSRS review

Layer 3 — FSRS spaced repetition

From js/fsrs.js. Implements the Free Spaced Repetition Scheduler — same algorithm Anki uses by default.

// after each review: stability = f(prev_stability, rating, retrievability) difficulty = f(prev_difficulty, rating) next_due_date = today + stability days // rating values: green → 1.0 (easy, double interval) yellow → 0.5 (medium, slight extension) red → 0.2 (hard, reset interval) none → 0 (lapse, full reset)

Cards you nail keep getting longer intervals (1d → 3d → 7d → 21d → 60d → …). Cards you miss reset to tomorrow. The scheduler tries to surface each card right when you'd otherwise forget it (~85% retention target).

Why this design wins

Naive: random questions

  • Bores you on topics already mastered
  • Doesn't compound on weak spots
  • No memory decay handling
  • Diagnostic = noise

This app's stack

  • EMA reflects recent performance, forgives one bad day but punishes streaks of wrong
  • State machine requires multiple confirmations before "mastery" unlocks
  • FSRS handles forgetting curves automatically
  • Picker drains weak spots first while keeping strong ones spaced
All three layers are vanilla JS, ~700 LOC combined. No ML, no server, no dependencies. State lives entirely in localStorage under ccna_* keys.
