Script load pattern

Mandatory order on every page (from project CLAUDE.md):

sequenceDiagram
  participant H as HTML <head>
  participant I as Inline config
  participant D as Deferred scripts
  participant E as DOMContentLoaded
  H->>I: parse window.masteryConfig = {...}
  H->>D: queue store.js, nav.js, ...
  Note over D: Browser fetches in parallel,
executes in source order
  D->>D: store.js runs first
  D->>D: mastery-tracker.js (uses store)
  D->>D: nav.js (uses store)
  D->>E: DOMContentLoaded fires
  E->>E: page-specific handlers run
Why no bundler? Every JS module is an IIFE that attaches to window (e.g. window.store). This makes the site work from file:// with zero build step. Trade-off: no tree-shaking, no TypeScript, manual dep order.
