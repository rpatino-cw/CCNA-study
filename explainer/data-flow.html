Proficiency math (EMA)

Proficiency is an exponential moving average over correctness. Recent answers weigh more than old ones. From js/store.js:

EMA_ALPHA = 0.3 new_prof = round( (1 - 0.3) × old_prof + 0.3 × (correct ? 1 : 0), 3 )

So 1 correct answer at proficiency 0 takes you to 0.3. Two more correct → 0.51, then 0.66. It takes ~5 correct answers to cross the 0.7 "weak threshold."

FSRS (spaced repetition)

fsrs.js implements the Free Spaced Repetition Scheduler. After every flashcard review, the card gets:

stability ← f(prev_stability, rating, retrievability) difficulty ← f(prev_difficulty, rating) next_due ← today + stability days

Ratings: green (1.0), yellow (0.5), red (0.2), none (0). Result: cards you nail re-appear in 7+ days; cards you miss re-appear tomorrow.

  T1->>T1: store.updateProficiency()

  BC->>T2: onmessage event
  T2->>T2: re-render proficiency bar
  Note over T2: no reload needed

js/sync.js opens a BroadcastChannel('ccna-sync'). Any store mutation publishes; every other tab listens and updates UI live.

Auto-backup

js/auto-backup.js snapshots all ccna_* keys every 5 minutes into ccna_backup_history (last 10 kept). Settings page has Restore dropdown to roll back if state corrupts.
