L2 Security Simdrill

L2 attacks ride below routing — they trick the switch, not the router. DHCP snooping kills rogue DHCP servers. DAI (Dynamic ARP Inspection) builds on snooping to kill ARP-poisoning. Port-security caps MAC addresses per port.

Pick an attack scenario

Defense: OFF

How they layer

DHCP snooping logs every legit IP→MAC binding. DAI uses that table to verify ARPs are honest. Port-security caps MAC count to stop CAM-table flood. They compose.

DHCP Snooping

Trust uplink port (where real DHCP server is). All other ports = untrusted. Block DHCP server replies (OFFER/ACK) from untrusted ports. Build IP→MAC binding table.

DAI

Inspects ARP packets. Cross-checks against DHCP snooping binding table. Drops ARPs that lie about IP→MAC.

Port-Security

Caps max MACs per port (e.g. 1-3). Sticky learn = remember & burn into config. Violation modes: protect (silent drop), restrict (drop + log), shutdown (err-disable port).
