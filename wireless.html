The 4-Part Split

PART 1 · RF & CELLS
  • Three bands: 2.4 GHz (long range, few channels, crowded), 5 GHz (more channels, faster, shorter range), 6 GHz (Wi-Fi 6E only, widest, least interference, shortest range).
  • 2.4 GHz non-overlapping channels: 1, 6, 11. Each Wi-Fi channel is 20 MHz wide but spaced so only 1/6/11 do not overlap.
  • Channel width: 20 MHz base. Bonding gives 40 / 80 / 160 MHz. Wider = faster but fewer non-overlapping channels and more interference. Stay 20 MHz on 2.4 GHz.
  • Naming chain: SSID = the network name (text). BSSID = the AP radio MAC (the actual identifier). BSS = one AP cell. ESS = multiple APs, same SSID, joined by a switch for roaming.
🧠 Bands climb, range drops: 2.4 reaches far but slow, 6 is fast but short. Channels: One, Six, Eleven stay clean.
⚠️ Trap: SSID vs BSSID: SSID is the typed name (can repeat across many APs); BSSID is the AP radio MAC and is unique per cell. Also: 2.4 GHz has more total channels (1 to 11/14) but only 1/6/11 are non-overlapping. Do not say "3 channels", say "3 non-overlapping". And ac is 5 GHz only (no 2.4); ax/Wi-Fi 6 covers both.
PART 2 · AP → WLC (CAPWAP)
  • Autonomous AP = standalone, full config on the AP itself (CLI/GUI per AP). Lightweight AP (LAP) = zero local config, controlled by a WLC. Cisco term for the pair: split-MAC.
  • CAPWAP (Control And Provisioning of Wireless Access Points) builds two tunnels from LAP to WLC: control on UDP 5246, data on UDP 5247. CAPWAP runs over IP and is encrypted (DTLS) on the control tunnel by default.
  • Split-MAC = real-time MAC stays on the AP (beacons, probe responses, ACKs, encryption, buffering); everything else moves to the WLC (association, authentication/802.1X, key mgmt, roaming, RF/RRM, QoS, VLAN-to-WLAN).
  • AP join order (memorize): 1) AP gets IP via DHCP, 2) discovers WLCs (CAPWAP Discovery Request/Response), 3) selects a WLC and sends Join Request, 4) WLC may push a new image (AP downloads, reboots), 5) WLC pushes config, AP enters RUN state.
🧠 "Control 6, data 7" (5246/5247). For switchports: "AP Access, WLC Trunk."
⚠️ Trap: Reversing the ports (5246 = control, 5247 = data) and reversing the switchports. The AP port is ACCESS (not trunk) because CAPWAP tunnels all client VLANs over IP, so the AP only needs one VLAN; the WLC uplink is the TRUNK. Also: control tunnel is DTLS-encrypted by default, data tunnel is not.
PART 3 · CLIENT AUTH
  • Three auth methods: Open (no key, anyone joins), PSK (one shared password for all), 802.1X/EAP (each user own login, checked by RADIUS).
  • Crypto ladder: WEP (broken, RC4) then WPA (TKIP, patch) then WPA2 (AES/CCMP, the standard) then WPA3 (SAE, newest).
  • WPA2 = AES/CCMP. WPA3 = AES/GCMP with SAE (Simultaneous Authentication of Equals, replaces the PSK 4-way handshake).
  • Personal = PSK (home, one password). Enterprise = 802.1X/EAP + RADIUS (work, per-user login). This Personal/Enterprise split applies to WPA, WPA2, and WPA3.
🧠 Crypto ladder: WEP WPA WPA2 WPA3 = 'We Were Pretty Awful, Stronger Always Eventually' (WEP-RC4, WPA-TKIP, WPA2-AES, WPA3-SAE). RADIUS ports: '18-twelve you log IN, 18-thirteen you write it DOWN' (1812 authn, 1813 accounting). Join steps: 'Scan, Auth, Associate, Key' = SAAK.
⚠️ Trap: Classic traps: (1) WPA2 uses AES/CCMP, WPA3 uses SAE. Do not say WPA3=AES as the headline answer, SAE is the distinguishing feature. (2) RADIUS is UDP 1812/1813, not TCP and not 1645/1646 (those are the old legacy ports the exam may dangle as a wrong choice). (3) Personal=PSK, Enterprise=802.1X+RADIUS. If the stem says 'per-user credentials' or 'authentication server', the answer is Enterprise/802.1X, not PSK. (4) The 802.11 'authentication' step (step 2) is NOT the real security, it is usually Open System and just legacy ceremony, the real auth happens in the key exchange/EAP step.
PART 4 · DATA PATH
  • Client associates to an SSID. The SSID is the over-the-air name; the WLAN profile maps that SSID to a WLC dynamic interface.
  • The dynamic interface ties the WLAN to a VLAN ID and an IP subnet. SSID maps to WLAN maps to dynamic interface maps to VLAN.
  • Client DHCP request goes over the air, into the CAPWAP data tunnel to the WLC, then out the WLC interface into that VLAN. DHCP scope lives in the wired VLAN, not the AP.
  • Data flow: client to AP (RF) to CAPWAP data tunnel (UDP 5246 control, UDP 5247 data) to WLC to wired VLAN to default gateway.
🧠 SSID to WLAN to Dynamic Interface to VLAN. AP tunnels, WLC trunks.
⚠️ Trap: Thinking the client VLAN must be on the AP switchport. It is NOT. The AP port is access in the AP-mgmt VLAN; client traffic is tunneled inside CAPWAP to the WLC, so the trunk carrying client VLANs is the WLC uplink. Also do not swap the ports: 5246 = control, 5247 = data.

Trace One Client: power-on → ping

Narrate this and the blur is gone. 📍 = where each part matters.

1. Scan / discover

Client listens for beacons or sends probes to find SSIDs.

📍 On 2.4 GHz it only hears APs on ch 1/6/11.

2. 802.11 Open auth

Legacy open-system auth. Not the real security check.

📍 Real auth happens in step 4.

3. Association

Client joins the AP/BSS, gets an Association ID.

📍 Lightweight AP relays to the WLC for the decision.

4. Real authentication

WPA2 4-way handshake (AES-CCMP), or 802.1X/EAP + RADIUS for Enterprise.

📍 WLC holds the keys and policy.

5. WLAN to VLAN

WLAN maps to a dynamic interface = one VLAN / subnet.

📍 Client now inherits that VLAN.

6. DHCP (DORA)

DISCOVER, OFFER, REQUEST, ACK.

📍 WLC relays over the data tunnel; DHCP server lives on the VLAN.

7. ARP + ping

Client ARPs for its gateway, sends the ping.

📍 CAPWAP data (UDP 5247) tunnels the frame to the WLC, which forwards on the VLAN trunk.

Tricky Q (top misses)

A laptop shows two APs broadcasting SSID CORP. What uniquely identifies each AP radio?
→ The BSSID (each AP radio's MAC address).
What switchport mode should connect a WLC to the distribution switch?
→ Trunk (802.1Q).
WPA2-Personal uses which authentication method?
→ Pre-shared key (PSK) with a 4-way handshake; encryption is AES-CCMP.
Which AP mode provides full-time dedicated RF scanning and does not serve clients?
→ Monitor mode.
On 2.4 GHz, which channel set avoids overlap?
→ 1, 6, and 11.
What does the lightweight AP do with client data frames?
→ Encapsulates them in a CAPWAP data tunnel (UDP 5247) to the WLC.
