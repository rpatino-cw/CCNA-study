IPsec VPN Simdrill

IPsec is two phases. Phase 1 (IKE) = build a secure tunnel just to negotiate. Phase 2 (IPsec SA) = build the actual data tunnel. Different keys, different jobs. Click each phase to see what's happening on the wire.

IKE negotiation in motion

The Two Phases

Phase 1 — IKE / ISAKMP

  • → Authenticate peers (PSK or certs)
  • → Diffie-Hellman key exchange
  • → Negotiate hash + cipher
  • → Build ISAKMP SA (mgmt tunnel)

Phase 2 — IPsec SA

  • → Inside the Phase 1 tunnel
  • → Negotiate ESP or AH
  • → Pick transport vs tunnel mode
  • → Build IPsec SA (data tunnel)
Phase 1 builds the management channel. Phase 2 uses it to set up the actual encrypted data path. Without Phase 1, peers can't trust each other to negotiate Phase 2.

Transport vs Tunnel mode

Transport = encrypts payload only, original IP header kept. Used host-to-host. Tunnel = encrypts entire original packet, wraps in new IP header. Used site-to-site (the original IPs are private — must hide them).

Transport mode

IP hdrESP hdrTCP+data (encrypted)ESP trailer
Original IPs visible. End-to-end between hosts.

Tunnel mode

NEW IP hdrESP hdrorig IP+TCP+data (encrypted)ESP trailer
Original IPs hidden inside. Site-to-site default.

Site-to-site vs Remote-access

Site-to-site = router-to-router, always-on, transparent to users (HQ ↔ Branch). Remote-access = client (laptop) to concentrator, on-demand, user-initiated (AnyConnect, OpenVPN). Both can use IPsec; remote-access often uses SSL/TLS instead.
