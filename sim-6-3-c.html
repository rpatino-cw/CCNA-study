The Controller Sandwichdrill

A controller sits in the middle. Apps + humans sit above and call northbound APIs (REST, JSON — high-level: "give me all sites"). Devices sit below and the controller talks to them via southbound APIs (NETCONF, RESTCONF, OpenFlow, gNMI — low-level: "edit this YANG node"). Up = abstract / business. Down = concrete / device.

The sandwich

Compass test: if you're going up (abstract, intent, business) you're hitting a northbound API. If you're going down (concrete, device-specific config) you're hitting a southbound API.

NB vs SB at a glance

Northbound

Faces apps + humans. Coarse-grained. Speaks REST + JSON. Authentication usually OAuth / token. Examples: GET /dna/intent/api/v1/network-device, POST /onboarding/pnp-device, the Meraki Dashboard API, controller webhooks.

Southbound

Faces devices. Fine-grained, often model-driven. NETCONF over SSH (with YANG models), RESTCONF over HTTPS, gNMI over gRPC, OpenFlow, and yes — even old-school CLI/SNMP/TFTP count. Telemetry usually lives here too.

Worked example — Python script using a northbound API

# 1) Operator runs Python on laptop import requests url = "https://dnac.example.com/dna/intent/api/v1/network-device" hdr = {"X-Auth-Token": token} r = requests.get(url, headers=hdr) # NORTHBOUND call devices = r.json()["response"] # 2) DNA Center — having received the NB call — fans out SOUTHBOUND # to each device via NETCONF/RESTCONF, gathering data per device: # <rpc><get-config><source><running/></source></get-config></rpc> # Returns YANG-modeled data. Controller assembles + responds to the laptop.

Two API calls happened — one northbound (Python → controller, REST/JSON) and many southbound (controller → switches, NETCONF/YANG). The Python script never touched a switch directly. That's the whole point.

Faded prompt — your turn

Question: a Python script uses ncclient to send edit-config directly to a switch. Which side is that?

Answer: southbound. The script is doing what a controller would normally do — talking NETCONF to a device. There's no controller in this picture; the script became the "controller" for one device. NB vs SB is about direction relative to the controller layer, not about who's typing the command.
