Top-down evaluation walkthrough
How ACL evaluation works (recipe)
  1. Top-down, first match wins. Walk the ACL line-by-line. The first line whose pattern fits the packet decides — permit or deny — and evaluation STOPS.
  2. Implicit deny at the end. If no line matches, the packet is dropped. Always.
  3. Wildcard mask, not subnet mask. Inverted bits — 0 = must match, 1 = don't care. 0.0.0.255 = match the first 3 octets exactly, last octet anything.
  4. Standard ACL (1-99): matches source IP only. Apply close to destination.
  5. Extended ACL (100-199): matches protocol + src + dst + ports. Apply close to source.
  6. Keywords: host 1.2.3.4 = 1.2.3.4 0.0.0.0; any = 0.0.0.0 255.255.255.255.
Trap example. What does Cisco love? Putting a broad permit BEFORE a specific deny. The broad permit fires first, the deny never gets reached. The drill catches this.
