Build a WLAN, the WLC Waydrill

A Catalyst 9800 WLC GUI is mostly four tabs. General (name + WLAN ID + status). Security (Layer 2 = WPA2/PSK + AES). Advanced/QoS (priority profile, client limits). Interfaces (which dynamic interface = which VLAN, controls DHCP + default gateway for clients). Walk the wizard below — typical mistakes flagged inline.

What each tab actually controls

1. General

The label, the SSID, the WLAN ID, the on/off switch. Status = Enabled is the easy miss — admin builds the WLAN, then forgets to flip it on.

2. Security

WPA2 + AES + PSK is the canonical "WPA2 Personal" combo. AKM = PSK (vs 802.1X). PSK length: 8–63 ASCII.

3. Advanced / QoS

QoS Platinum/Gold/Silver/Bronze marks DSCP. Client caps + idle timeout + broadcast on/off live here.

4. Interface

Which dynamic interface (VLAN) carries the clients off the WLC. Wrong VLAN = clients associate but get no DHCP.

Worked example — equivalent CLI (Cat 9800 IOS-XE)

! WLAN profile WLC(config)# wlan Corp-WiFi-Profile 1 Corp-WiFi WLC(config-wlan)# no shutdown ! Security: WPA2 + AES + PSK WLC(config-wlan)# security wpa wpa2 ciphers aes WLC(config-wlan)# security wpa akm psk set-key ascii 0 C0rp-Wifi-S3cret-2026! WLC(config-wlan)# security wpa psk set-key ascii 0 C0rp-Wifi-S3cret-2026! ! Map WLAN to a policy profile (interface = VLAN) WLC(config)# wireless profile policy default-policy-profile WLC(config-wireless-policy)# vlan 10 WLC(config-wireless-policy)# no shutdown ! Bind WLAN to a policy tag attached to AP WLC(config)# wireless tag policy default-policy-tag WLC(config-policy-tag)# wlan Corp-WiFi-Profile policy default-policy-profile

Faded prompt — your turn

"Clients associate to Corp-WiFi but never get an IP and the page never loads." Walk the four tabs and identify the most likely-broken setting.

Answer: Tab 4 — Interface mapping. The WLAN is bound to a dynamic interface that doesn't reach the DHCP scope. Either the dynamic interface is on a VLAN with no DHCP relay or the wrong VLAN entirely. Confirm with show wireless client summary + DHCP server logs.

Other tab-by-tab failure modes:
Tab 1: Status = Disabled → clients see nothing.
Tab 2: WPA3-only when phones still on WPA2 → fail to associate.
Tab 3: Max clients = 1 → second user kicked off.
Tab 4: Wrong VLAN → no DHCP.
