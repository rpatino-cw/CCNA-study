3.1

Routing table components

The routing table is the router's contract with itself: every destination network it has heard of, where to forward toward it, and how trustworthy that information is. Read it left to right.

Exam trapOne destination can have multiple candidate routes; only one ends up in this table. Learn to read why.

Anatomy of a single line

Click any colored token to learn what it means.

O    10.1.2.0/24 [110/65] via 10.0.0.2, 00:12:34, GigabitEthernet0/1
C    10.0.0.0/30 is directly connected, GigabitEthernet0/1
L    10.0.0.1/32 is directly connected, GigabitEthernet0/1
S    10.99.0.0/16 [1/0] via 192.0.2.5
S*   0.0.0.0/0 [1/0] via 203.0.113.1

B    192.168.50.0/24 [20/0] via 198.51.100.1, 03:55:01

What the brackets mean: [AD/metric]

First number = administrative distance (how trusted the source is). Second = metric (how good this path is, source-specific units).

3.2

How a router picks a path

Three filters in strict order: longest prefix match → administrative distance → metric. The first filter to find a single winner stops the contest.

Step 1 · Longest prefix match (most specific wins)

Type any destination and watch the table filter. Longest prefix that still matches takes the packet — it doesn't matter which protocol learned it at this stage.

Step 2 · Administrative distance ladder

When two protocols hand the router the same prefix length, the lower AD wins. Memorize this ladder cold — exam expects it.

Mnemonic: CSE-EI-OISR-EE-iU — Connected, Static, EIGRP-summ, eBGP, EIGRP, IGRP, OSPF, IS-IS, RIP, EIGRP-ext, iBGP, Unknown.

Step 3 · Metric (only inside the same protocol)

If both routes were learned by the same protocol with the same prefix, the lower metric installs. Equal metric → load-balance (ECMP).

3.3

Static routing — IPv4 & IPv6

Hand-written routes. Predictable, low-overhead, brittle. Four flavors map to the entire CCNA blueprint.

The four shapes — click each tile

Default route (gateway of last resort)

Catches everything not in the table. Push branch traffic to the ISP with one line.

ip route 0.0.0.0 0.0.0.0 203.0.113.1
Network route

Specific destination subnet. Most common static.

ip route 10.99.0.0 255.255.0.0 192.0.2.5
Host route (/32 or /128)

Pinpoint one address. Useful for management hosts, SLA probes, blackholes.

ip route 192.168.50.10 255.255.255.255 10.0.0.2
Floating static (backup)

AD bumped above the dynamic protocol so it only installs when the primary route disappears.

ip route 10.99.0.0 255.255.0.0 198.51.100.1 130

Recursive vs directly-attached vs fully-specified

FormSyntaxWhat the router doesWhen to use
Recursiveip route 10.0.0.0 255.0.0.0 192.0.2.5Looks up 192.0.2.5 in its own table to find the egress interface — extra lookup per packet without CEF.Default — works on any media.
Directly attachedip route 10.0.0.0 255.0.0.0 Serial0/0Treats the destination as on-link. ARPs every dest IP — only safe on point-to-point links.P2P serial / GRE only.
Fully specifiedip route 10.0.0.0 255.0.0.0 GigabitEthernet0/1 192.0.2.5Both the egress interface and the next-hop are pinned. No recursion, no ARPing arbitrary hosts.Multi-access (Ethernet) when you want to be explicit.

IPv6 mirrors

Same shapes, different keyword. Always enable v6 routing first.

# enable v6 forwarding
ipv6 unicast-routing

# default route
ipv6 route ::/0 2001:db8:abcd::1

# network route
ipv6 route 2001:db8:99::/48 2001:db8:0:1::2

# host route (/128)
ipv6 route 2001:db8:50::10/128 2001:db8:0:1::2

# floating static (default AD=1, raise above dynamic)
ipv6 route 2001:db8:99::/48 2001:db8:0:2::2 130

# link-local next-hop MUST include exit interface
ipv6 route 2001:db8:99::/48 GigabitEthernet0/1 fe80::2
TrapLink-local next-hops are not routable globally. Every router has its own fe80::/10 — you must specify the egress interface.
3.4

OSPFv2 — single area

Open Shortest Path First. Link-state, AD 110, hello/dead 10/40 on broadcast and P2P. Builds an identical LSDB on every router in the area, then runs Dijkstra.

Single area = area 0CCNA scope is a single area, conventionally the backbone (area 0). Multi-area is CCNP territory but the exam still expects you to know what an ABR / LSDB is.

Adjacency requirements (must all match)

FieldWhy it must match
Area IDRouters refuse to flood LSAs across mismatched areas.
Hello & Dead intervalsListener and talker have to agree on the heartbeat clock.
Subnet (same prefix on the link)OSPFv2 requires the two interfaces to share an IP subnet.
MTUIf different, stuck in ExStart/Exchange — DBD packets get dropped.
Authentication (if enabled)Same type (none / clear / md5) and same key on both sides.
Network type compatibilityBroadcast ↔ broadcast, P2P ↔ P2P. Mismatches break DR election.
Stub flagBoth sides agree the area is stub or it's not.

Network types — point-to-point vs broadcast (DR/BDR)

Broadcast segments elect a Designated Router so n routers exchange LSAs as 2(n-1) flows instead of n(n-1)/2. Click to highlight.

Router-ID selection (in this exact order)

  1. router-id A.B.C.D command (manual). Wins everything.
  2. Highest up loopback IPv4 address.
  3. Highest up physical interface IPv4 address.
Stale RID gotchaThe router caches its RID. After changing IPs or adding a loopback, run clear ip ospf process for the new RID to take effect.

Minimum config that brings up an adjacency

! On both routers:
router ospf 1
 router-id 1.1.1.1
 network 10.0.0.0 0.0.0.255 area 0
 passive-interface default
 no passive-interface GigabitEthernet0/1

! Or interface-mode (modern, preferred):
interface GigabitEthernet0/1
 ip ospf 1 area 0
 ip ospf network point-to-point   ! skip DR election on a /30
 ip ospf hello-interval 10
 ip ospf dead-interval 40

! Verify:
show ip ospf neighbor          ! state should be FULL or 2-WAY (DROTHER↔DROTHER)
show ip ospf interface brief
show ip route ospf

Wildcard math — common confusion

OSPF uses an inverted mask. Flip every bit of the subnet mask.

Subnet maskWildcardCatches
255.255.255.00.0.0.255One /24
255.255.255.2520.0.0.3One /30
255.255.0.00.0.255.255One /16 (256× /24s)
255.255.255.2550.0.0.0Exactly one IP — useful for network 1.1.1.1 0.0.0.0 area 0 on a loopback.
3.5

First-Hop Redundancy Protocols

Hosts only know one default gateway. FHRPs put a virtual IP/MAC behind two or more routers so that gateway never disappears.

The three you must know

ProtocolOwnerVirtual MACHello / HoldActive selectionLoad share?
HSRP Cisco Cisco proprietary 0000.0c07.acXX (group ID) 3 s / 10 s (default) Highest priority (default 100), then highest IP. Preempt off by default. Per-subnet only (different groups on different VLANs)
VRRP RFC 5798 IETF open standard 0000.5e00.01XX 1 s / 3 s (default) Highest priority. Preempt on by default. The router owning the VIP is master automatically (priority 255). Per-subnet only
GLBP Cisco Cisco proprietary 0007.b400.XXYY 3 s / 10 s One Active Virtual Gateway (AVG) elects up to 4 Active Virtual Forwarders (AVFs). Yes — true per-host load share. AVG hands out different vMACs.

HSRP failover — animated

PC has gateway 10.0.0.254 (the VIP). Press fail to drop the active router and watch standby take over.

Object tracking (interface tracking)

A common HSRP/VRRP gotcha: the WAN goes down but the LAN side stays up, so the active router happily forwards traffic into a black hole. Tracking solves this.

! decrement priority by 20 if WAN interface drops
track 1 interface GigabitEthernet0/2 line-protocol
interface GigabitEthernet0/1
 standby 10 ip 10.0.0.254
 standby 10 priority 110
 standby 10 preempt
 standby 10 track 1 decrement 20
