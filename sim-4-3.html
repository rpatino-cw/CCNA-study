DHCP + DNS in Motion animated

Boot a fresh client → it has nothing → DORA gets it an IP. Then it asks for `www.example.com` → DNS recursion walks the tree. Click play, watch each message fly.

DHCP — DORA Handshake (UDP 67/68)

1. Discover Client broadcasts to 255.255.255.255 src=0.0.0.0 — "any DHCP server out there?"
2. Offer Server proposes IP, mask, gateway, DNS, lease (still broadcast — client has no IP yet)
3. Request Client broadcasts: "I want THIS offer from server X" — others know to drop their offers
4. Acknowledge Server confirms. Client commits IP 192.168.1.50/24, GW .1, DNS 8.8.8.8, lease 24h.

DNS Resolution (UDP/TCP 53)

1. Stub query PC asks 8.8.8.8: "A record for www.example.com?"
2. → Root Resolver hits a root: "who handles .com?"
3. → TLD Root replies: "ask the .com TLD nameservers"
4. → Authoritative .com replies: "ns1.example.com is authoritative"
5. Answer ns1.example.com replies: A = 93.184.216.34
6. Cached return Resolver caches per TTL → returns to PC. Browser opens TCP/443.

DNS Record Types

A hostname → IPv4. www → 93.184.216.34
AAAA hostname → IPv6. www → 2606:2800::
CNAME alias → another name. blog → www.example.com
MX mail server. example.com → mail pri 10
NS nameserver authority for the zone
PTR reverse: IP → hostname (in-addr.arpa)
TXT arbitrary text — SPF, DKIM, domain verification
SOA zone metadata — primary NS, serial, TTLs
