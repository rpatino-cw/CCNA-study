01

The idea, in one breath

A subnet mask only ever has one octet doing real work. Find that octet, subtract it from 256, and you've got a number that tells you exactly how big each subnet is and where every boundary sits.

Why it works

A mask octet is either 255 (all network bits, frozen), 0 (all host bits, free), or something in between — that "in between" octet is where the network is sliced.

256 − that octet = the magic number. It's the size of one subnet block, measured in the same octet. Walk up the column in jumps of that size and you hit every subnet ID in the network.

04

Messer's worked example

Given 192.168.1.0/24, design a subnet scheme supporting at least 40 devices per subnet. Step through the method as Messer teaches it.

  1. Count the host bits you need

    40 usable devices means we need at least 40 + 2 (network + broadcast) = 42 addresses.

    25 = 32 (too small)  ·  26 = 64 ✓

    Six host bits gives 64 addresses, 62 usable. Plenty.

  2. Borrow the rest for the network

    32 total bits − 6 host bits = 26 network bits. New mask: /26.

    /26 → 255.255.255.192

  3. Identify the interesting octet

    The fourth octet is neither 0 nor 255. 192 is the interesting octet.

  4. Run the magic number

    256 − 192 = 64

    Each subnet is 64 addresses wide in the fourth octet.

  5. Walk the subnet IDs

    Start at 0, jump by the magic number until you hit 256.

    192.168.1.0 · 64 · 128 · 192

    Four subnets, each holding 62 usable hosts. Done — no binary required.

  6. Read the boundaries off the column

    Subnet ID192.168.1.64 First host.65 Last host.126 Broadcast.127

    Broadcast = next subnet ID − 1. First/last host = subnet ID + 1 / broadcast − 1.
