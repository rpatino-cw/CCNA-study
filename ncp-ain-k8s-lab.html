Kubernetes RDMA Lab

You have kubectl access to a bare-metal GPU cluster. The NVIDIA Network Operator manages networking components via the NicClusterPolicy CRD. Pods expose RDMA and GPUDirect RDMA through secondary interfaces attached by Multus. Type help for the full command list. Switch to Troubleshooting Scenarios to diagnose real K8s RDMA faults.

Try these

  • kubectl get nicclusterpolicy Show Network Operator policy and its status
  • kubectl -n nvidia-network-operator get pods All Network Operator component pods and Ready state
  • kubectl get nad -A NetworkAttachmentDefinitions across all namespaces
  • kubectl describe pod rdma-workload-0 Pod spec, events, and resource allocation
  • kubectl get node gpu-node-01 -o yaml Node labels and allocatable RDMA resources
  • kubectl logs rdma-workload-0 Pod stdout: RDMA init messages and errors
  • kubectl exec rdma-workload-0 -- lsmod Kernel modules loaded in pod (peermem, ib_uverbs)
  • kubectl exec rdma-workload-0 -- ip link Network interfaces visible inside the pod netns
