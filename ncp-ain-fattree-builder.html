Why This Matters (n1, n15, n29)

[know]AllReduce collectives are barrier-synchronous: every rank waits on the slowest. Any oversubscription on the compute path directly caps bus bandwidth -- the entire step is gated on the slowest link.
[know]1:1 nonblocking: uplink count per leaf equals downlink count. SU example = 8 leaf x 6 spine on NDR400. Blocking begins the moment uplinks are fewer than downlinks.
[know]Rail-optimized wiring: NIC-0 of EVERY node goes to the same leaf (rail 0), NIC-1 to the same leaf (rail 1). Same-rail GPU-to-GPU traffic is one hop, staying on the leaf. Cross-rail traffic climbs to spine only when crossing rails.
[know]DGX H100: 8 compute ConnectX-7 NICs, 8 rails, 8 leaf switches per SU. Rail count equals GPU count per server. A non-rail-optimized design adds spine hops for traffic that should be local.
[know]Storage fabric is intentionally near 4:3 oversubscribed because bulk I/O tolerates blocking that collectives cannot. Compute fabric is always 1:1.
[inferred]Bisection bandwidth = half the total bandwidth crossing the imaginary vertical cut through the spine layer. For a 1:1 fat-tree, bisection BW = (spine switches x ports-per-spine x link-speed) / 2.

Formulas Used

Downlinks per leaf = switch radix minus uplinks per leaf. The remaining ports face the compute nodes (servers/GPUs).
Oversubscription ratio = downlinks per leaf : uplinks per leaf. 1:1 is nonblocking. 2:1 means each host gets half bandwidth on average at the leaf.[assumption: uniform traffic, all ports active]
Total GPU count = leaf switches x GPUs-per-leaf. Each leaf hosts a set of servers; each server NIC maps to one GPU rail.
Uplink cable count = leaf switches x uplinks-per-leaf. These are the leaf-to-spine links.
Downlink cable count = leaf switches x downlinks-per-leaf. These are the leaf-to-server/NIC links.[assumption: all downlink ports cabled]
Total cable count = uplink cables + downlink cables.
Bisection bandwidth = (spine switches x uplinks-per-leaf x link-speed) / 2. This is the bandwidth across the imaginary bisecting cut through the spine layer.[assumption: each spine port connected to one leaf uplink, all active]
Nonblocking verdict = uplinks per leaf >= downlinks per leaf. True when every downlink host can simultaneously talk at full speed through the fabric.
