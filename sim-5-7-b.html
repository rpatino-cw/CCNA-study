Three Letters That Aren't One Jobdrill

AAA is three things in a trench coat. Authentication: who are you? Authorization: what can you do? Accounting: what did you do? Two protocols carry it: TACACS+ (Cisco-flavored, TCP/49, encrypts whole packet, separates the three As — perfect for device admin login), and RADIUS (open, UDP/1812+1813, encrypts only the password, combines AuthN+AuthZ — perfect for 802.1X / VPN / Wi-Fi).

The 3 As — what each one decides

Authentication

WHO are you? Username + password? Cert? OTP? RSA token? The user proves identity to the AAA server. Output: yes/no + (often) a session ticket.

Authorization

WHAT can you do? After auth succeeds, the server returns a policy: which commands you can run (TACACS+ command sets), which VLAN you go on (RADIUS dynamic VLAN), what session timeout, etc.

Accounting

WHAT did you do? Per-command logs, session start/stop, bytes in/out. For audit + billing + forensics. Sent to the AAA server (and usually then to a SIEM).

Pick the right protocol per scenario

Worked example — AAA method-list config (TACACS+ for admin login)

SW1(config)# aaa new-model SW1(config)# tacacs server ISE-1 SW1(config-server-tacacs)# address ipv4 10.99.1.50 SW1(config-server-tacacs)# key Cisco123! ! method lists — TACACS+ first, fall back to local user if AAA server is unreachable SW1(config)# aaa authentication login default group tacacs+ local SW1(config)# aaa authorization exec default group tacacs+ local SW1(config)# aaa authorization commands 15 default group tacacs+ local SW1(config)# aaa accounting commands 15 default start-stop group tacacs+ ! apply to vty SW1(config)# line vty 0 15 SW1(config-line)# login authentication default SW1(config-line)# authorization commands 15 default SW1(config-line)# accounting commands 15 default

Key habit: always include local as a fallback in method lists. If the AAA server is unreachable, you can still log in with a local account and fix the network. Forgetting this is how engineers lock themselves out of production gear.

Faded prompt — your turn

Helpdesk manager wants "audit who ran which command on every switch — including show commands." Which protocol + which AAA module + which command level?

Answer: TACACS+ (RADIUS doesn't do per-command), accounting module (recording, not allowing), command level 1 + 15 (1 = user-mode `show`, 15 = privileged-exec).

SW1(config)# aaa accounting commands 1 default start-stop group tacacs+ SW1(config)# aaa accounting commands 15 default start-stop group tacacs+
