Topology
Three-tier campus + dual-ISP edge

Two pods (A and B). Each pod has a distribution pair (DSW) and three access switches (ASW). Both core switches (CSW1/CSW2) connect to all four DSWs. R1 sits on top with two routed links into the cores and two NAT'd links out to ISP-A and ISP-B. The routed core (R1 + CSWs + DSWs) runs OSPF on /30 transit links plus /32 loopbacks.

HSRP convention used in this lab

For every user VLAN the DSW pair shares a virtual IP that ends in .1 (the gateway). DSW-x1 gets .2, DSW-x2 gets .3. The grader expects sequential standby group numbers and priority 105 on the active, a config that "works" with different group numbers or priority 110 still loses points.

The 8 parts, key commands + traps
Save or score zero

The .pka grader checks each device's startup-config. Run do write (or copy running-config startup-config) on every device or you get no points even when the running-config is perfect. Per-device full configs live in the GitHub repos linked above; the hints below are the must-know commands and exam traps for each part.

01 Initial setup 03:27

Hostname each device by its layer role (edge/core/distribution/access). Set the enable secret, a local username for SSH later, and secure the console line.

Show hint, base device hardening
Any device
Switch(config)# hostname DSW-A1
DSW-A1(config)# enable secret YOURSECRET
DSW-A1(config)# username admin secret YOURSECRET
DSW-A1(config)# line console 0
DSW-A1(config-line)# password YOURSECRET
DSW-A1(config-line)# login
02 VLANs & Layer-2 EtherChannel 17:59

Create VLANs 10/20/30/40/99 everywhere. Bundle the redundant inter-switch links into Layer-2 EtherChannels (PAgP), set them as trunks with native VLAN 99, configure access ports for hosts, and shut every unused interface.

EtherChannel mode trap (high exam frequency)

PAgP desirable actively asks; auto only answers. auto + auto never forms a channel (both wait). LACP equivalent: active asks, passive answers, passive+passive also fails. Both ends must also match speed, duplex, allowed VLANs, and trunk mode or the bundle stays suspended.

Show hint, L2 PAgP bundle + trunk
Switch (both ends)
SW(config)# interface range gig 1/0/4 - 5
SW(config-if-range)# channel-group 1 mode desirable
SW(config-if-range)# exit
SW(config)# interface port-channel 1
SW(config-if)# switchport trunk encapsulation dot1q
SW(config-if)# switchport mode trunk
SW(config-if)# switchport trunk native vlan 99
03 IP addressing, L3 EtherChannel & HSRP 42:39

Address the routed /30 links and loopbacks. Make the CSW1↔CSW2 bundle a Layer-3 EtherChannel (no switchport on the port-channel). Configure HSRP on every DSW SVI so each VLAN has a redundant virtual gateway.

L2 vs L3 EtherChannel

L2 channel = trunk between switches. L3 channel = routed link: put no switchport on the port-channel interface, then an IP. Add members with channel-group N mode active. Verify with show etherchannel summary, flag SU = Layer-2 up, RU = Layer-3 up, P = port bundled.

Show hint, HSRP virtual gateway
DSW-A1 (active for VLAN 10)
DSW-A1(config)# interface vlan 10
DSW-A1(config-if)# ip address 10.1.0.2 255.255.255.0
DSW-A1(config-if)# standby 10 ip 10.1.0.1
DSW-A1(config-if)# standby 10 priority 105
DSW-A1(config-if)# standby 10 preempt
04 Rapid Spanning Tree (RSTP) 01:05:56

Run spanning-tree mode rapid-pvst. Make the distribution/core switches the root for their VLANs by lowering priority. The trunk from ASW-A1 to the WLC needs spanning-tree portfast trunk so the AP/WLC link comes up instantly.

Root bridge = lowest priority, then lowest MAC

Set the root with spanning-tree vlan 10 root primary (sets priority 24576) or hard-code priority 0. Align the STP root with the HSRP active switch so traffic does not hairpin. portfast trunk only belongs on trunks to a single host-like device (a WLC), never switch-to-switch.

Show hint, RSTP root + portfast trunk
DSW-A1 / ASW-A1
DSW-A1(config)# spanning-tree mode rapid-pvst
DSW-A1(config)# spanning-tree vlan 10,20,40 root primary
! WLC uplink on the access switch
ASW-A1(config)# interface fa0/2
ASW-A1(config-if)# spanning-tree portfast trunk
05 Static & dynamic routing (OSPF) 01:14:41

Enable OSPF on R1, both cores, and all four distribution switches across the /30 transit links and loopbacks. Put a default route on R1 toward each ISP, then make R1 inject that default into OSPF as an ASBR.

Default-route injection, default-information originate

R1 has two static defaults (one per ISP). default-information originate under router ospf advertises the default into the area so every internal device learns the way out. Use ip ospf network point-to-point on the routed links to skip DR/BDR and converge fast.

Show hint, OSPF + default origination
R1
R1(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1
R1(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.5
R1(config)# router ospf 1
R1(config-router)# router-id 10.0.0.76
R1(config-router)# network 10.0.0.32 0.0.0.3 area 0
R1(config-router)# default-information originate
06 Network services: DHCP, DNS, NTP, SNMP, Syslog, FTP, SSH, NAT 01:29:02

R1 hosts DHCP pools for the user VLANs (exclude the first ten usable addresses of each). Add DNS, NTP, SNMP, Syslog, FTP, and SSH. Finish with NAT/PAT so the inside networks reach the ISPs.

DHCP + NAT traps

Exclude addresses before defining the pool. Across HSRP, the DHCP default-router must be the VIP (.1), not a physical SVI. For NAT: mark inside/outside interfaces, write an ACL of inside source networks, then ip nat inside source list ACL interface G0/0/0 overload (PAT). SSH needs a hostname, domain-name, crypto key generate rsa, and transport input ssh on the vty lines.

Show hint, DHCP pool + PAT
R1
R1(config)# ip dhcp excluded-address 10.1.0.1 10.1.0.10
R1(config)# ip dhcp pool PCs_A
R1(dhcp-config)# network 10.1.0.0 255.255.255.0
R1(dhcp-config)# default-router 10.1.0.1
! PAT to ISP-A
R1(config)# ip nat inside source list 1 interface g0/0/0 overload
07 Security: ACLs & Layer-2 protection 02:08:23

Write an extended ACL controlling traffic between Office A and Office B. On access switches add port security, DHCP snooping, and Dynamic ARP Inspection (DAI).

Snooping + DAI trust direction

DHCP snooping: trust the uplink toward the DHCP server, leave host ports untrusted. DAI rides on the snooping binding table, so enable snooping first or valid hosts get blackholed. Port security: switchport port-security + mac-address sticky, default violation shutdown. Extended ACLs go close to the source; standard ACLs close to the destination.

Show hint, port security + snooping
ASW (access + uplink)
ASW(config)# ip dhcp snooping
ASW(config)# ip dhcp snooping vlan 10,20,30,40
ASW(config)# interface gig 0/1
ASW(config-if)# ip dhcp snooping trust
ASW(config)# interface fa0/1
ASW(config-if)# switchport port-security mac-address sticky
08 IPv6 + Wireless (WLC) 02:21:56

Add IPv6 addressing where called for, then build the wireless: the WLC management interface in VLAN 99, a dynamic interface for the Wi-Fi VLAN 40, and a WLAN/SSID mapped to it. The two lightweight APs register to the WLC (no FlexConnect).

WLC essentials for the CCNA

LWAPs are zero-touch: they pull an address via DHCP, find the WLC, and download config, no CLI on the AP. On the WLC GUI you create the dynamic interface (VLAN 40 + IP), then a WLAN bound to that interface with WPA2 security. The AP-to-switch link is a trunk with PortFast (Part 4).
