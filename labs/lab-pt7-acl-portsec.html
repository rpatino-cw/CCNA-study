Week 7

ACLs & Port Security

~50 min Topics 5.6 · 5.7 Packet Tracer
Objectives
  • 01Create a named extended ACL "BLOCK-ENGINEERING" that denies VLAN 20 traffic to the server on port 80/443 and permits everything else
  • 02Apply the ACL inbound on R1's Fa0/0.20 subinterface
  • 03Configure sticky port security on SW1 Fa0/1 (PC1's port): max 1 MAC, violation shutdown
  • 04Verify ACL behavior: PC1 (Sales) can ping server, PC2 (Engineering) cannot reach server port 80
  • 05Test port security violation: connect a different device to Fa0/1 and confirm err-disabled

Step-by-Step Build
01 Build the topology in Packet Tracer

Build on top of the Lab PT-3 topology, or rebuild from scratch: R1 (Cisco 2901) with RoaS subinterfaces Fa0/0.10, .20, .30, SW1 (Cisco 2960) with VLANs 10/20/30 and trunk on Fa0/24. Add the Server to SW1 Fa0/4 in VLAN 30. Assign the server IP 192.168.30.100/24 with gateway 192.168.30.254.

02 Verify inter-VLAN routing is working

Before adding the ACL, confirm that all PCs can reach the server. PC2 should be able to ping 192.168.30.100 at this stage. Once you apply the ACL in Step 4, that will change.

PC2 — Desktop → Command Prompt
PC2> ping 192.168.30.100    (should succeed before ACL is applied)
03 Create named extended ACL on R1

Named extended ACLs use ip access-list extended <name>. Each line is an ACE (Access Control Entry). Order matters — ACEs are evaluated top-down. Deny the Engineering subnet from reaching the server on TCP 80 and 443, then permit everything else. The implicit deny at the end of every ACL would block all traffic if you forget the permit ip any any.

R1
R1(config)# ip access-list extended BLOCK-ENGINEERING
R1(config-ext-nacl)# deny tcp 192.168.20.0 0.0.0.255 host 192.168.30.100 eq 80
R1(config-ext-nacl)# deny tcp 192.168.20.0 0.0.0.255 host 192.168.30.100 eq 443
R1(config-ext-nacl)# permit ip any any
R1(config-ext-nacl)# exit

Wildcard mask note: 0.0.0.255 matches any host in 192.168.20.0/24. The inverse of subnet mask 255.255.255.0. host 192.168.30.100 is shorthand for 192.168.30.100 0.0.0.0.

04 Apply ACL inbound on VLAN 20 subinterface

Apply the ACL inbound on Fa0/0.20 — traffic from the Engineering VLAN arrives here, so this is where to intercept it. Applying inbound means the ACL is checked before routing, which is the most efficient placement for extended ACLs (stops unwanted traffic close to the source).

R1
R1(config)# interface fastethernet 0/0.20
R1(config-subif)# ip access-group BLOCK-ENGINEERING in
R1(config-subif)# exit
05 Verify ACL behavior

PC2 (Engineering) should now fail to reach the server on port 80. In Packet Tracer, use the Web Browser application on PC2 (Desktop tab) and try to open http://192.168.30.100 — it should time out. PC1 (Sales) should succeed. The ACL hit counters in show ip access-lists confirm which rules are matching.

R1
R1# show ip access-lists BLOCK-ENGINEERING
! Look for non-zero match counts on deny lines after PC2 attempts
PC2 — Desktop → Web Browser
URL: http://192.168.30.100    (should time out — denied by ACL)
URL: http://192.168.10.1      (can still reach other hosts — permit any any)
06 Configure sticky port security on SW1 Fa0/1

Port security restricts which MAC addresses can use a port. Sticky mode dynamically learns the first MAC that connects and saves it to the running-config as a secure MAC. violation shutdown puts the port in err-disabled state when a violation occurs — requiring manual recovery.

SW1
SW1(config)# interface fastethernet 0/1
SW1(config-if)# switchport mode access
SW1(config-if)# switchport access vlan 10
SW1(config-if)# switchport port-security
SW1(config-if)# switchport port-security maximum 1
SW1(config-if)# switchport port-security mac-address sticky
SW1(config-if)# switchport port-security violation shutdown
SW1(config-if)# exit
07 Verify port security and trigger a violation

Confirm port security is active on Fa0/1 after PC1 has sent traffic (the sticky MAC should be learned). To trigger a violation: in Packet Tracer, change PC1's MAC address (Desktop → IP Configuration → FastEthernet → change MAC) or swap PC1's cable to a different PC. The port should immediately go err-disabled. Recover with shutdown then no shutdown.

SW1
SW1# show port-security interface fastethernet 0/1
! Look for: Security Enabled: Enabled, Sticky MAC, Violation Mode: Shutdown

SW1# show port-security
! After violation: shows MaxSecAddr=1, CurrentAddr=1, SecurityViolation count

SW1# show interfaces fastethernet 0/1 status
! After violation: Status column shows "err-disabled"

! --- Manual recovery after err-disabled ---
SW1(config)# interface fastethernet 0/1
SW1(config-if)# shutdown
SW1(config-if)# no shutdown
