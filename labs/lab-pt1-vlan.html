Step-by-Step Build
01 Build the topology in Packet Tracer

Add two Cisco 2960 switches and six PCs. Connect each PC to its switch port as shown in the topology diagram. Connect SW1 Fa0/24 to SW2 Fa0/24 for the trunk. Use Copper Straight-Through cables for PC-to-switch, Copper Cross-Over for switch-to-switch (PT may auto-detect).

02 Configure VLANs on SW1

Enter global config, create VLANs 10, 20, 30, and 99. Best practice: also create the native VLAN (99) explicitly so it exists in the database.

SW1
Switch> enable
Switch# configure terminal
SW1(config)# hostname SW1
SW1(config)# vlan 10
SW1(config-vlan)# name SALES
SW1(config-vlan)# vlan 20
SW1(config-vlan)# name ENGINEERING
SW1(config-vlan)# vlan 30
SW1(config-vlan)# name MANAGEMENT
SW1(config-vlan)# vlan 99
SW1(config-vlan)# name NATIVE
SW1(config-vlan)# exit
03 Assign access ports on SW1

Set each PC-facing port to mode access and assign it to the correct VLAN. Explicitly setting mode access prevents the port from becoming a trunk via DTP negotiation.

SW1
SW1(config)# interface fastethernet 0/1
SW1(config-if)# switchport mode access
SW1(config-if)# switchport access vlan 10
SW1(config-if)# exit
SW1(config)# interface fastethernet 0/2
SW1(config-if)# switchport mode access
SW1(config-if)# switchport access vlan 20
SW1(config-if)# exit
SW1(config)# interface fastethernet 0/3
SW1(config-if)# switchport mode access
SW1(config-if)# switchport access vlan 30
SW1(config-if)# exit
04 Configure the trunk port on SW1

Set Fa0/24 to trunk mode, specify dot1Q encapsulation, set native VLAN 99, and disable DTP with nonegotiate. Limiting allowed VLANs to only what you need is a security best practice.

SW1
SW1(config)# interface fastethernet 0/24
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport trunk encapsulation dot1q
SW1(config-if)# switchport trunk native vlan 99
SW1(config-if)# switchport trunk allowed vlan 10,20,30,99
SW1(config-if)# switchport nonegotiate
SW1(config-if)# exit

Note: Packet Tracer 2960 switches may not require trunk encapsulation dot1q (it's the only option). If the command errors, skip it — the switch will use dot1Q automatically.

05 Shutdown unused ports on SW1

Disable ports Fa0/4 through Fa0/23 to reduce attack surface. The interface range command handles this efficiently.

SW1
SW1(config)# interface range fastethernet 0/4 - 23
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# switchport access vlan 99
SW1(config-if-range)# shutdown
SW1(config-if-range)# exit
06 Mirror all configuration on SW2

Repeat Steps 2–5 on SW2 with identical VLAN database, access port assignments for PC4/PC5/PC6, and the same trunk configuration on Fa0/24. The VLANs must match on both sides of the trunk or traffic will be dropped.

SW2
Switch> enable
Switch# configure terminal
SW2(config)# hostname SW2
SW2(config)# vlan 10
SW2(config-vlan)# name SALES
SW2(config-vlan)# vlan 20
SW2(config-vlan)# name ENGINEERING
SW2(config-vlan)# vlan 30
SW2(config-vlan)# name MANAGEMENT
SW2(config-vlan)# vlan 99
SW2(config-vlan)# name NATIVE
SW2(config-vlan)# exit
! --- Access ports ---
SW2(config)# interface fastethernet 0/1
SW2(config-if)# switchport mode access
SW2(config-if)# switchport access vlan 10
SW2(config-if)# exit
SW2(config)# interface fastethernet 0/2
SW2(config-if)# switchport mode access
SW2(config-if)# switchport access vlan 20
SW2(config-if)# exit
SW2(config)# interface fastethernet 0/3
SW2(config-if)# switchport mode access
SW2(config-if)# switchport access vlan 30
SW2(config-if)# exit
! --- Trunk ---
SW2(config)# interface fastethernet 0/24
SW2(config-if)# switchport mode trunk
SW2(config-if)# switchport trunk native vlan 99
SW2(config-if)# switchport trunk allowed vlan 10,20,30,99
SW2(config-if)# switchport nonegotiate
SW2(config-if)# exit
! --- Unused ports ---
SW2(config)# interface range fastethernet 0/4 - 23
SW2(config-if-range)# switchport mode access
SW2(config-if-range)# switchport access vlan 99
SW2(config-if-range)# shutdown
SW2(config-if-range)# end
07 Configure PC IP addresses

In Packet Tracer, click each PC → Desktop tab → IP Configuration. Enter the IP and subnet mask from the address table. No default gateway needed for this lab (no inter-VLAN routing yet — that's Lab PT-3).
