Week 2

STP Root Election + EtherChannel

~45 min Topics 2.4 · 2.5 Packet Tracer
Objectives
  • 01Set SW1 as root bridge for all VLANs (priority 4096) and SW2 as secondary root (priority 8192)
  • 02Build a 2-port LACP EtherChannel (Po1) between SW1 and SW2 using Fa0/23 and Fa0/24
  • 03Configure Po1 as an 802.1Q trunk with VLANs 10 and 20 allowed
  • 04Observe SW3's blocking port using show spanning-tree and understand why STP chose that port
  • 05Verify EtherChannel is active with show etherchannel summary — Po1 should show as "SU"
Topology
         SW1 (Root — priority 4096)
        /    \
  Fa0/23      Fa0/1
  Fa0/24      |
  (LACP       |
  EtherCh.)   |
        \    /
         SW2 ──── Fa0/1 ──── SW3 (priority 32768)
    (Secondary — priority 8192)

  SW1–SW2: Fa0/23 + Fa0/24 → Port-channel 1 (LACP, trunk, VLANs 10,20)
  SW1–SW3: Fa0/1  (trunk)
  SW2–SW3: Fa0/1  (trunk) ← SW3's port toward SW2 will be BLOCKING

Step-by-Step Build
01 Build the topology in Packet Tracer

Add three Cisco 2960 switches (SW1, SW2, SW3). Cable them in a triangle: connect SW1 Fa0/23 and Fa0/24 both to SW2 Fa0/23 and Fa0/24 (two cables for the EtherChannel). Connect SW1 Fa0/1 to SW3 Fa0/1, and SW2 Fa0/1 to SW3 Fa0/2. Use Copper Cross-Over for switch-to-switch (PT may auto-detect with Auto MDI-X).

Note: Before configuring EtherChannel, Packet Tracer may show the redundant SW1–SW2 links as amber (STP blocking). That is expected — STP will recalculate once EtherChannel is up.

02 Create VLANs 10 and 20 on all three switches

All three switches need VLANs 10 and 20 in their VLAN database before trunk links will carry them. Run the same commands on SW1, SW2, and SW3.

SW1 (repeat on SW2 and SW3)
Switch> enable
Switch# configure terminal
SW1(config)# hostname SW1
SW1(config)# vlan 10
SW1(config-vlan)# name SALES
SW1(config-vlan)# vlan 20
SW1(config-vlan)# name ENGINEERING
SW1(config-vlan)# exit
03 Configure STP priorities — SW1 root, SW2 secondary

STP bridge priority must be a multiple of 4096 (0–61440). Lower priority wins the root election. Setting SW1 to 4096 guarantees it wins. SW2 at 8192 is the backup — if SW1 fails, SW2 becomes root. SW3 keeps the default (32768) and stays non-root.

SW1 — Root Bridge
SW1(config)# spanning-tree vlan 1,10,20 priority 4096
SW2 — Secondary Root
SW2(config)# spanning-tree vlan 1,10,20 priority 8192

Tip: An alternative is spanning-tree vlan 1,10,20 root primary (sets priority to 24576 or lower) and root secondary (sets 28672). Using explicit values as above gives you full control and is better for exam scenarios.

04 Configure LACP EtherChannel on SW1

Bundle Fa0/23 and Fa0/24 into channel-group 1 using LACP mode active. Then configure the resulting Port-channel 1 as a trunk allowing VLANs 10 and 20. The physical ports inherit their config from the port-channel — set trunk mode on Po1, not on the physical interfaces.

SW1
! --- Bundle physical ports into EtherChannel ---
SW1(config)# interface range fastethernet 0/23 - 24
SW1(config-if-range)# channel-group 1 mode active
SW1(config-if-range)# exit
! --- Configure the logical port-channel as trunk ---
SW1(config)# interface port-channel 1
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport trunk allowed vlan 10,20
SW1(config-if)# exit
05 Configure LACP EtherChannel on SW2

Mirror the exact same EtherChannel config on SW2. Both sides must use mode active (or one active + one passive — but never passive + passive, which will not form a channel). Config must be identical: same VLANs, same trunk mode.

SW2
! --- Bundle physical ports into EtherChannel ---
SW2(config)# interface range fastethernet 0/23 - 24
SW2(config-if-range)# channel-group 1 mode active
SW2(config-if-range)# exit
! --- Configure the logical port-channel as trunk ---
SW2(config)# interface port-channel 1
SW2(config-if)# switchport mode trunk
SW2(config-if)# switchport trunk allowed vlan 10,20
SW2(config-if)# exit
06 Configure trunk links to SW3

SW3 connects to both SW1 (via Fa0/1) and SW2 (via Fa0/2 on SW3 side, Fa0/1 on SW2 side). Configure both uplinks as trunks. STP will keep one in forwarding and put the other into blocking once it converges.

SW1 — uplink to SW3
SW1(config)# interface fastethernet 0/1
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport trunk allowed vlan 10,20
SW1(config-if)# exit
SW2 — uplink to SW3
SW2(config)# interface fastethernet 0/1
SW2(config-if)# switchport mode trunk
SW2(config-if)# switchport trunk allowed vlan 10,20
SW2(config-if)# exit
SW3 — both uplinks
SW3(config)# interface fastethernet 0/1
SW3(config-if)# switchport mode trunk
SW3(config-if)# switchport trunk allowed vlan 10,20
SW3(config-if)# exit
SW3(config)# interface fastethernet 0/2
SW3(config-if)# switchport mode trunk
SW3(config-if)# switchport trunk allowed vlan 10,20
SW3(config-if)# exit
07 Verify STP topology and identify the blocking port on SW3

After STP converges (allow ~30 seconds in PT), check spanning-tree on each switch. SW1 should show "This bridge is the root." On SW3, one port will be in BLK (blocking) state — the port toward SW2 (Fa0/2), because the path SW3→SW1 is cheaper than SW3→SW2→SW1.

Why SW3 Fa0/2 blocks: SW3 must pick its root port — the port with the lowest cost path to SW1. Fa0/1 reaches SW1 directly (cost 19 for 100 Mbps). Fa0/2 reaches SW1 via SW2 (cost 19 + 19 = 38). Lower cost wins, so Fa0/1 becomes the root port and Fa0/2 is blocked.
