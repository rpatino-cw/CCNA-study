Week 6

NAT/PAT & DHCP

~45 min Topics 4.1 · 4.3 Packet Tracer
Objectives
  • 01Configure a DHCP pool on R1 to assign addresses in the 10.0.1.0/24 range to inside PCs
  • 02Exclude 10.0.1.1–10.0.1.10 from DHCP (reserved for router and static devices)
  • 03Configure PAT (overload) to translate all inside 10.0.1.x addresses to R1's outside IP 203.0.113.1
  • 04Verify DHCP bindings with show ip dhcp binding
  • 05Verify NAT translations with show ip nat translations after a ping

Step-by-Step Build
01 Build the topology in Packet Tracer

Add one Cisco 2901 router (R1), one Cisco 2960 switch (SW1), four PCs, and one Server. Connect PC1–PC4 to SW1 with copper straight-through cables. Connect SW1 to R1 Fa0/0. Connect R1 Fa0/1 directly to the Web Server with a copper cross-over cable (or use auto-detect).

Set the Web Server IP to 203.0.113.100 / 255.255.255.0 manually via Desktop → IP Configuration. Leave PCs set to DHCP for now — they will get addresses after Step 3.

02 Configure R1 interfaces

Assign IP addresses to both router interfaces and mark each with its NAT role. The ip nat inside and ip nat outside commands are required — NAT only processes traffic crossing that boundary.

R1
Router> enable
Router# configure terminal
R1(config)# hostname R1
! --- Inside interface (LAN) ---
R1(config)# interface fastethernet 0/0
R1(config-if)# ip address 10.0.1.1 255.255.255.0
R1(config-if)# ip nat inside
R1(config-if)# no shutdown
R1(config-if)# exit
! --- Outside interface (ISP link) ---
R1(config)# interface fastethernet 0/1
R1(config-if)# ip address 203.0.113.1 255.255.255.0
R1(config-if)# ip nat outside
R1(config-if)# no shutdown
R1(config-if)# exit
03 Configure DHCP pool on R1

Exclude the first 10 addresses (reserved for the router and any static devices), then create the DHCP pool. The default-router command sets the gateway PCs will receive — they need this to reach the outside network.

R1
R1(config)# ip dhcp excluded-address 10.0.1.1 10.0.1.10
R1(config)# ip dhcp pool LAN
R1(dhcp-config)# network 10.0.1.0 255.255.255.0
R1(dhcp-config)# default-router 10.0.1.1
R1(dhcp-config)# dns-server 8.8.8.8
R1(dhcp-config)# lease 7
R1(dhcp-config)# exit

Now click each PC → Desktop → IP Configuration → select DHCP. PCs should receive addresses starting at 10.0.1.11.

04 Create an ACL to match inside traffic

NAT overload requires an access list that identifies which source addresses should be translated. Standard ACL 1 matches any source in the 10.0.1.0/24 range. The wildcard mask 0.0.0.255 is the inverse of the subnet mask.

R1
R1(config)# access-list 1 permit 10.0.1.0 0.0.0.255
05 Configure PAT overload

Bind ACL 1 to the outside interface with the overload keyword. This enables PAT — traffic matching the ACL will be translated to R1's Fa0/1 IP, differentiated by unique source port numbers. No separate NAT pool is needed.

R1
R1(config)# ip nat inside source list 1 interface fastethernet 0/1 overload
06 Add a default route toward the outside

Translated packets need a path to the outside. In this simulated topology, point the default route at the Web Server's IP as the next-hop. In a real network this would be the ISP gateway.

R1
R1(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.100
R1(config)# end
R1# write memory

Note: In a production environment the next-hop would be the ISP router's IP, not the destination server. Packet Tracer's simplified routing model accepts the server IP here.
