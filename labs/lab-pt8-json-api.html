Analysis Exercises
01 HTTP Verb Mapping to IOS Equivalents

RESTCONF maps standard HTTP methods to network configuration operations. Study this table — the CCNA exam frequently asks which HTTP method corresponds to a given action.

HTTP Verb IOS Equivalent Behavior
GET show commands Read-only retrieval. Does not modify any configuration.
PUT Replace a config section Replaces the entire resource. Like overwriting a config block — existing values not in the payload are removed.
POST Create new resource Creates a new entry (e.g., add a new interface or route). Fails if the resource already exists.
PATCH Modify existing resource Merges changes into the existing config. Like adding lines to an existing config section without removing others.
DELETE Remove config Removes the specified resource. Equivalent to using no on a config command.

Key insight: GET is the safe, read-only verb. PUT is destructive — it replaces, not merges. Use PATCH when you want to add to existing configuration without erasing it.

02 JSON Payload Analysis — Interface Configuration

Below is a realistic RESTCONF JSON response for a router interface. Study it and answer the mapping questions that follow.

RESTCONF GET response — ietf-interfaces
{
  "ietf-interfaces:interface": {
    "name": "GigabitEthernet1",
    "description": "WAN Link",
    "type": "iana-if-type:ethernetCsmacd",
    "enabled": true,
    "ietf-ip:ipv4": {
      "address": [
        {
          "ip": "10.0.0.1",
          "prefix-length": 24
        }
      ]
    }
  }
}

Mapping questions — think through each before reading the answer:

What IOS command does "enabled": true correspond to? Answer: no shutdown — the interface is administratively up. If enabled were false, it maps to shutdown.

What does "prefix-length": 24 map to? Answer: subnet mask 255.255.255.0 — RESTCONF uses CIDR prefix length instead of dotted-decimal subnet masks.

What IOS command does the entire payload represent? Answer: interface GigabitEthernet1 / description WAN Link / ip address 10.0.0.1 255.255.255.0 / no shutdown

03 RESTCONF URL Structure

RESTCONF URLs follow a predictable pattern. Break down this example URL and understand each segment — exam questions often ask what a specific part of the URL represents.

RESTCONF URL — retrieve GigabitEthernet1 config
https://192.168.1.1/restconf/data/ietf-interfaces:interfaces/interface=GigabitEthernet1
URL Segment Meaning
/restconf/data/ Datastore path — this is where running/candidate configuration lives. All config reads/writes go through /data/.
ietf-interfaces YANG module name (RFC 7223). The part before the colon is always the module that defines this data model.
:interfaces Top-level container within the YANG module. The colon separates module from data node.
/interface=GigabitEthernet1 List key — identifies a specific instance within the interface list. The value after = is the key value (the interface name).

Pattern to memorize: /restconf/data/[module]:[container]/[list]=[key]

04 YANG Model Navigation — Paths to IOS Equivalents

YANG (Yet Another Next Generation) is a data modeling language (RFC 6020/7950) that defines the structure, types, and relationships of network configuration data. It is not a protocol — NETCONF and RESTCONF are the protocols that use YANG models.

YANG Path IOS Equivalent
ietf-interfaces:interfaces/interface/name Interface name (e.g., GigabitEthernet1)
ietf-interfaces:interfaces/interface/enabled Admin state — no shutdown (true) / shutdown (false)
ietf-ip:ipv4/address/ip IPv4 address in the ip address command
ietf-ip:ipv4/address/prefix-length Subnet mask length (CIDR notation)
ietf-routing:routing/control-plane-protocols Routing protocol configuration block

NETCONF vs. RESTCONF comparison:

Feature NETCONF RESTCONF
Transport SSH (port 830) HTTPS (port 443)
Data format XML only JSON or XML
Operations get, get-config, edit-config, lock, unlock, commit HTTP verbs (GET, PUT, POST, PATCH, DELETE)
Transactions Full transactional support (lock/commit) No native transaction support
Best for Complex, transactional bulk changes Simple reads/writes, REST tooling (curl, Postman)

Both use the same YANG data models — the model describes what can be configured, not how the protocol transports it.
