Week 4

Multi-Router Single-Area OSPF

~60 min Topic 3.4 Packet Tracer
Objectives
  • 01Cable three routers in a triangle (R1–R2–R3) with serial or FastEthernet point-to-point links
  • 02Configure loopback interfaces on each router to use as stable OSPF Router IDs
  • 03Enable OSPF process 1 on each router and advertise all connected networks into Area 0
  • 04Verify all three adjacencies reach Full state using show ip ospf neighbor
  • 05Confirm OSPF-learned (O) routes appear in each router's routing table
  • 06Test end-to-end reachability by pinging loopback addresses across all three routers
Topology
Why a triangle?

Three routers in a full mesh means each router has two OSPF neighbors. You'll see 6 total adjacencies (2 per router × 3 routers). This tests that OSPF converges across multiple paths and that routing tables are fully populated — you can't fake it with a single link.

Step-by-Step Build
01 Build the topology in Packet Tracer

Add three Cisco 2901 routers. Connect R1 Fa0/0 → R2 Fa0/0, R1 Fa0/1 → R3 Fa0/0, and R2 Fa0/1 → R3 Fa0/1 using Copper Cross-Over cables (or Auto for PT). Do not connect any PCs — this lab is router-only.

02 Configure R1 — interfaces + loopback

Set hostnames, configure loopback0, and bring up both FastEthernet interfaces with IP addresses. A loopback never goes down, so OSPF will always use it as the Router ID.

R1
Router> enable
Router# configure terminal
Router(config)# hostname R1
! Loopback for stable Router ID
R1(config)# interface loopback 0
R1(config-if)# ip address 1.1.1.1 255.255.255.255
R1(config-if)# exit
! Link to R2
R1(config)# interface fastethernet 0/0
R1(config-if)# ip address 10.12.0.1 255.255.255.252
R1(config-if)# no shutdown
R1(config-if)# exit
! Link to R3
R1(config)# interface fastethernet 0/1
R1(config-if)# ip address 10.13.0.1 255.255.255.252
R1(config-if)# no shutdown
R1(config-if)# exit
03 Configure R2 and R3 interfaces

Repeat the same pattern on R2 and R3. Each gets a loopback + two point-to-point links.

R2
Router(config)# hostname R2
R2(config)# interface loopback 0
R2(config-if)# ip address 2.2.2.2 255.255.255.255
R2(config-if)# exit
R2(config)# interface fastethernet 0/0
R2(config-if)# ip address 10.12.0.2 255.255.255.252
R2(config-if)# no shutdown
R2(config-if)# exit
R2(config)# interface fastethernet 0/1
R2(config-if)# ip address 10.23.0.1 255.255.255.252
R2(config-if)# no shutdown
R2(config-if)# exit
R3
Router(config)# hostname R3
R3(config)# interface loopback 0
R3(config-if)# ip address 3.3.3.3 255.255.255.255
R3(config-if)# exit
R3(config)# interface fastethernet 0/0
R3(config-if)# ip address 10.13.0.2 255.255.255.252
R3(config-if)# no shutdown
R3(config-if)# exit
R3(config)# interface fastethernet 0/1
R3(config-if)# ip address 10.23.0.2 255.255.255.252
R3(config-if)# no shutdown
R3(config-if)# exit
04 Enable OSPF on R1 — Area 0
OSPF network command — wildcard mask

The wildcard mask is the inverse of the subnet mask. /30 subnet (255.255.255.252) → wildcard 0.0.0.3. /32 loopback (255.255.255.255) → wildcard 0.0.0.0. Use network [ip] [wildcard] area [id]. The more specific the wildcard, the less chance of accidentally advertising the wrong interface.

R1
R1(config)# router ospf 1
R1(config-router)# router-id 1.1.1.1
! Advertise loopback
R1(config-router)# network 1.1.1.1 0.0.0.0 area 0
! Advertise R1-R2 link
R1(config-router)# network 10.12.0.0 0.0.0.3 area 0
! Advertise R1-R3 link
R1(config-router)# network 10.13.0.0 0.0.0.3 area 0
R1(config-router)# exit
05 Enable OSPF on R2 and R3

Each router advertises its loopback and its two links. After this step you should start seeing OSPF adjacency messages in the PT console.

R2
R2(config)# router ospf 1
R2(config-router)# router-id 2.2.2.2
R2(config-router)# network 2.2.2.2 0.0.0.0 area 0
R2(config-router)# network 10.12.0.0 0.0.0.3 area 0
R2(config-router)# network 10.23.0.0 0.0.0.3 area 0
R2(config-router)# exit
R3
R3(config)# router ospf 1
R3(config-router)# router-id 3.3.3.3
R3(config-router)# network 3.3.3.3 0.0.0.0 area 0
R3(config-router)# network 10.13.0.0 0.0.0.3 area 0
R3(config-router)# network 10.23.0.0 0.0.0.3 area 0
R3(config-router)# exit
06 Set passive interface on loopbacks

Loopbacks should never send OSPF Hellos — there's nothing on the other end to form adjacency with. Use passive-interface to suppress Hello packets while still advertising the network in OSPF LSAs.

R1 (repeat on R2 and R3)
R1(config)# router ospf 1
R1(config-router)# passive-interface loopback 0
R1(config-router)# exit
