Packet Tracer Setup

Enable one-click topology building — click a button in the lab, and the network builds itself inside Packet Tracer. Takes about 5 minutes to set up, works for every lab after that.

You need two things: a small Python script running in your terminal (the bridge), and an extension loaded in Packet Tracer (Builder-MCP.pts). Both are free, open source, and run entirely on your machine.
Step 1 — Install Packet Tracer
Already have Packet Tracer? Skip to Step 2.
1

Create a free Cisco NetAcad account

Go to netacad.com and sign up. You do not need to enroll in a course — the account is free and gives you access to the download.

2

Download and install

After logging in: Resources → Download Packet Tracer. Pick your OS (Windows, macOS, Linux). Install with default settings. On macOS you may need to approve it in System Settings → Privacy & Security.

Step 2 — Load the Builder extension

Builder-MCP.pts is a free extension that lets Packet Tracer receive commands from your browser. It runs inside PT and does nothing until the bridge server sends it a topology.

1

Download Builder-MCP.pts

Download Builder-MCP.pts

Save it somewhere you won't accidentally delete it (e.g., your Documents folder).

2

Load it in Packet Tracer

Open Packet Tracer, then:

Extensions → Scripting → Configure PT Script Modules

Click Add, select the Builder-MCP.pts file you downloaded, and make sure it is set to On Startup.

3

Restart Packet Tracer

Close and reopen Packet Tracer. The extension loads automatically on startup from now on. You never need to do this step again.

One-time setup. Once the extension is added, it loads automatically every time you open Packet Tracer. You won't need to touch this menu again.
Step 3 — Start the bridge server

The bridge is a tiny Python script that relays commands from your browser to Packet Tracer. No install needed — just run one command.

Open a terminal and paste:

$ python3 -c "$(curl -fsSL https://raw.githubusercontent.com/rpatino-cw/CCNA-study/main/pt-bridge.py)" copy

Works on macOS, Linux, and WSL. Requires Python 3 and curl (both pre-installed on most systems).

Windows (PowerShell) alternative
> irm https://raw.githubusercontent.com/rpatino-cw/CCNA-study/main/pt-bridge.py | python copy

Or download pt-bridge.py and run python pt-bridge.py

Keep the terminal open. The bridge runs until you close it (Ctrl+C). You need it running every time you want to use the "Build in Packet Tracer" buttons. It does not install anything on your system.
Test your setup

Make sure both Packet Tracer and the bridge server are running, then click the button below.

Troubleshooting

"Packet Tracer not detected" after clicking Build

The bridge server isn't running. Open a terminal and run the command from Step 3. Keep it open while you use the labs.

Bridge says "Waiting for Packet Tracer to connect"

Packet Tracer is open but Builder-MCP.pts isn't loaded. Go to Extensions → Scripting → Configure PT Script Modules and make sure Builder-MCP.pts is listed and set to On Startup. Restart PT after adding it.

"python3: command not found"

Python 3 isn't installed or isn't in your PATH. On macOS, run xcode-select --install to get it. On Windows, install from python.org.

Do I need to do this every time?

The extension (Step 2) is one-time — it auto-loads on startup. The bridge server (Step 3) needs to be running each session. Just keep that terminal open while you study.

Can I use the labs without this?

Yes. Every lab works without the bridge — you just build the topology manually in Packet Tracer by following the instructions. The bridge just automates that first step.

Is this safe?

The bridge runs entirely on your machine (127.0.0.1 — localhost only). Nothing is sent to the internet. The Python script has zero dependencies and you can read the source: pt-bridge.py
