STP / RSTP (802.1D, 802.1w, PortFast, BPDU Guard)

STP prevents Layer 2 loops by electing one root bridge and blocking redundant paths, leaving one loop-free active topology.

▶ Start
Force this switch to be root (safe macro, sets low priority)spanning-tree vlan 1 root primary
Backup rootspanning-tree vlan 1 root secondary
Set priority manually (must be multiple of 4096)spanning-tree vlan 1 priority 4096
Use Rapid-PVST (RSTP per VLAN)spanning-tree mode rapid-pvst
PortFast on one access port (skip listen/learn)interface gi0/1 ; spanning-tree portfast
PortFast on all access ports globallyspanning-tree portfast default
BPDU Guard on the port (err-disable if BPDU seen)interface gi0/1 ; spanning-tree bpduguard enable
BPDU Guard globally (on all portfast ports)spanning-tree portfast bpduguard default
🔍 Verify
show spanning-tree→ Per VLAN: Root ID + Bridge ID. Look at port Role (Root/Desg/Altn/Back) and Sts (FWD/BLK/LRN).
show spanning-tree vlan 1→ 'This bridge is the root' line if local is root; else Root ID points to the elected root MAC/priority.
show spanning-tree root→ Root ID, root cost, and the Root Port (which local port leads to root). Cost 0 = this is root.
show spanning-tree summary→ Mode (rstp / ieee), and PortFast/BPDU Guard default enabled/disabled.
show spanning-tree interface gi0/1 detail→ Role, state, port cost, and BPDU counts. Confirms PortFast/edge status.
show interfaces status err-disabled→ err-disabled = BPDU Guard tripped (a switch/BPDU plugged into a PortFast port).
❓ Likely Q+A
How is the root bridge elected, and the tiebreak?
Lowest Bridge ID wins. BID = Priority (default 32768) + sys-ext (VLAN id) + MAC. Compare priority first; tie -> lowest MAC address wins.
What are the 802.1D port roles and states?
Roles: Root Port (best path to root), Designated Port (forwards on a segment), Non-designated/Blocking. States: Blocking, Listening, Learning, Forwarding, Disabled.
How does a switch pick its Root Port?
1) Lowest root path cost, 2) lowest sender (neighbor) BID, 3) lowest sender port ID. Root bridge itself has no root port.
What are the default STP path costs?
10Mbps=100, 100Mbps(Fa)=19, 1Gbps(Gi)=4, 10Gbps=2. Cost accumulates toward root (added on receiving switch).
Which port blocks, and what are RSTP states/port types?
The non-designated port on the higher-cost/higher-BID side blocks. RSTP states: Discarding, Learning, Forwarding. RSTP roots: Root, Designated, Alternate (backup to root), Backup.
What do PortFast and BPDU Guard do?
PortFast: access/edge port skips listening+learning, goes straight to Forwarding (no 30s delay). BPDU Guard: if a BPDU arrives on a PortFast port, port goes err-disabled (stops a switch loop).
✅ Working when
  • Exactly one root bridge per VLAN: one switch shows 'This bridge is the root' (root cost 0).
  • Every non-root switch has exactly one Root Port in FWD.
  • Each segment/link has exactly one Designated Port forwarding; redundant ports show BLK (802.1D) or Altn/Discarding (RSTP).
  • No port flapping and no err-disabled ports from BPDU Guard (show interfaces status).
  • Mode matches intent: show spanning-tree summary reads rstp for Rapid-PVST.
🧠 Root picks LOWEST BID (priority then MAC). Port picks LOWEST: Cost, BID, Port-ID. States: Blocking Listens, Learns, Forwards (B-L-L-F).
⚠️ Trap: Priority must be a multiple of 4096, and the VLAN id is added via sys-ext (so real BID is 32768+VLAN). RSTP collapses 5 states into 3 (Discarding/Learning/Forwarding). BPDU Guard err-disables on BPDU; do NOT confuse with BPDU Filter (which drops BPDUs silently).

ACLs (standard + extended, numbered + named, interface + VTY)

Ordered permit/deny list, top-down first-match-wins with a hidden deny at the end; standard filters source IP only (place near destination), extended filters the 5-tuple (place near source).

▶ Start
Standard numbered (deny one host, permit rest)access-list 10 deny host 192.168.1.5 access-list 10 permit any
Standard named (same, named mode)ip access-list standard BLOCK-PC deny host 192.168.1.5 permit any
Wildcard mask = inverse of subnet mask0.0.0.255 = /24 0.0.0.0 = one host 255.255.255.255 = any
ORDER MATTERS: first match wins, then stopput specific deny BEFORE permit any (permit any first makes deny dead)
Extended numbered (block HTTPS, permit rest)access-list 100 deny tcp 10.0.0.0 0.0.0.255 host 2.2.2.2 eq 443 access-list 100 permit ip any any
Extended named (insertable entries)ip access-list extended WEB deny tcp 10.0.0.0 0.0.0.255 host 2.2.2.2 eq 443 permit ip any any
Apply to interface (standard near dest, extended near src)interface gi0/1 ip access-group 10 out (or: ip access-group 100 in)
Apply to VTY (access-class, NOT access-group)line vty 0 4 access-class 10 in
🔍 Verify
show access-lists→ every ACE with (N matches) hit counters; implicit deny is NOT shown but is always there
show ip access-lists 100→ one named/numbered list with sequence numbers 10/20/30 and per-line match counts
show ip interface gi0/1→ line 'Outgoing access list is 10' / 'Inbound access list is 100' confirms direction applied
show running-config | section access-list→ exact ACEs and order; confirm specific entries sit above permit any
clear access-list counters→ zeros all hit counts so you can re-test cleanly and watch counters move
❓ Likely Q+A
Where do you place a standard ACL vs an extended ACL?
Standard near the DESTINATION (only knows source IP, so placing near source would over-block). Extended near the SOURCE (drops unwanted traffic early).
What is at the end of every ACL and is it visible?
An implicit deny any (extended: deny ip any any). Always there, NOT shown in show access-lists. No match = drop.
Numbered ranges: standard vs extended?
Standard = 1-99 (and 1300-1999). Extended = 100-199 (and 2000-2699).
Wildcard mask for a /26 (mask 255.255.255.192)?
0.0.0.63. Trick: subnet mask + wildcard = 255.255.255.255, so 255 minus each octet. /24 = 0.0.0.255.
How does the router process an ACL?
Top-down, first match wins, then stops. So a permit any placed before a deny X makes that deny unreachable. Order matters.
How do you restrict SSH/Telnet to the router by source IP?
Apply a standard ACL to the VTY lines with access-class N in (NOT ip access-group, which is interface-only).
✅ Working when
  • Run clear access-list counters, then show access-lists shows all ACEs at 0 matches
  • Send traffic that should be PERMITTED: it passes AND the permit line's (N matches) counter climbs
  • Send traffic that should be DENIED: it drops/times out AND the deny line's counter climbs
  • show ip interface confirms the list is bound to the right interface AND direction (in/out)
  • Traffic with no explicit match is dropped, proving the implicit deny is active even though it is not printed
🧠 Standard = Source only, near Destination. Extended = Everything (5-tuple), near source. SD / ES.
⚠️ Trap: Wildcard mask is the INVERSE of a subnet mask (0=match, 1=ignore): /24 is 0.0.0.255, never 255.255.255.0. Also: VTY uses access-class (not access-group); extended ACLs reject a bare IP (use host X or X 0.0.0.0); and standard goes near destination while extended goes near source (do not swap them).

OSPFv2 (CCNA 200-301)

OSPFv2 is a link-state IGP that floods LSAs, builds an identical area database, runs SPF (Dijkstra) per router, and picks lowest-cost paths (cost = 10^8 / bandwidth).

▶ Start
Start process (pid is local-only, 1-65535)router ospf 1
Force a stable router-id (do this first)router-id 1.1.1.1
Advertise nets: address, WILDCARD mask, areanetwork 10.0.12.0 0.0.0.255 area 0
Match a single /30 link exactlynetwork 10.0.12.0 0.0.0.3 area 0
Per-interface alternative to network stmtinterface g0/0 ; ip ospf 1 area 0
Stop hellos out a LAN/loopback (still advertised)passive-interface g0/1
Set cost directly on an interfaceinterface g0/0 ; ip ospf cost 10
Set reference bw so >1G links differ (do on all RTRs)router ospf 1 ; auto-cost reference-bandwidth 10000
🔍 Verify
show ip ospf neighbor→ State must read FULL (or FULL/DR, FULL/BDR, FULL/DROTHER). 2WAY between two DROTHERs on a LAN is normal.
show ip ospf interface brief→ Each OSPF interface listed with correct Area, Cost, State, Nbrs count.
show ip route ospf→ O = intra-area, O IA = inter-area routes. AD 110. Confirm expected prefixes present.
show ip protocols→ Process id, router-id, networks advertised, passive interfaces, neighbors.
show ip ospf neighbor (check timers if stuck)→ Stuck in INIT = one-way hellos (ACL/passive). Stuck in EXSTART/EXCHANGE = MTU mismatch.
❓ Likely Q+A
Neighbor state order to reach adjacency?
Down, Init, 2Way, Exstart, Exchange, Loading, Full. (2Way = bidirectional seen; Full = databases synced.)
DR/BDR election: who wins and tiebreak?
Highest OSPF interface priority wins (default 1). Tie broken by highest router-id. Priority 0 = never DR/BDR. Election is non-preemptive (no takeover once elected).
Router-ID selection order?
1) manual router-id, 2) highest IP on an up loopback, 3) highest IP on any up physical interface. Set at process start; needs clear ip ospf process to change live.
What must match to form an adjacency?
Same area, same hello AND dead timers, same subnet/mask on the link, matching MTU, same auth, same area type (stub flags), same network type. (Router-id and process id do NOT need to match; RIDs must differ.)
Default hello and dead timers?
Broadcast/point-to-point: hello 10s, dead 40s (4x hello). Mismatched timers block adjacency.
Cost formula and area-0 rule?
Cost = reference-bw / interface-bw = 10^8 / bps (default ref 100 Mbps). 1G and 10G both round to cost 1 unless ref-bw raised. Area 0 is the backbone; all other areas must connect to it.
✅ Working when
  • show ip ospf neighbor: every expected neighbor present and State = FULL (or 2WAY only between two DROTHERs).
  • show ip route ospf: expected remote prefixes show as O / O IA with AD 110.
  • Routing table has no missing or duplicate next-hops; best path matches lowest cumulative cost.
  • End-to-end ping across the OSPF domain succeeds (loopback to loopback, sourced if needed).
  • show ip ospf interface brief: correct interfaces in area 0, no link stuck/missing, neighbor counts as expected.
🧠 Neighbor states: Down Init 2way Exstart Exchange Loading Full = "Don't Inhale 2 Extra Exhausting Long Fumes." Election: Priority then RID, 0 never plays.
⚠️ Trap: network statements use a WILDCARD mask, not a subnet mask (0.0.0.255 not 255.255.255.0). And cost uses 10^8/bw with DEFAULT ref-bw 100M, so 1G and 10G links both = cost 1 until you raise auto-cost reference-bandwidth on EVERY router. Stuck EXSTART/EXCHANGE = MTU mismatch; stuck INIT = one-way hello.
