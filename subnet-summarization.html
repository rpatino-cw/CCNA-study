Subnet Summarization

Given a list of contiguous subnets, find the single summary route (supernet) that covers them all. Critical for keeping routing tables small. The exam tests both directions: derive the summary, or verify which networks fall inside one.

Binary alignment + walkthrough
How summarization works (recipe)
  1. Write each network in binary. 32 bits per address.
  2. Mark the leading bits that match in ALL addresses. Walk left-to-right; the first column where any two networks differ ends the match.
  3. Common-bit count = summary prefix length.
  4. Summary network = take any of the input networks and zero out everything past the common bits.
  5. Sanity check: the summary must cover ALL inputs and not extend further than needed.
Example. Summarize 192.168.0.0/24, 192.168.1.0/24, 192.168.2.0/24, 192.168.3.0/24.
192.168.0.0  =  11000000.10101000.00000000.00000000
192.168.1.0  =  11000000.10101000.00000001.00000000
192.168.2.0  =  11000000.10101000.00000010.00000000
192.168.3.0  =  11000000.10101000.00000011.00000000
                                       ^^^^^^^^ first 22 bits match
Summary:        11000000.10101000.00000000.00000000  →  192.168.0.0/22
Trap: if the bottom 2 bits ranged 1, 2, 3, 4 instead of 0–3, no clean /22 summary exists. Networks must be aligned to the summary boundary.
