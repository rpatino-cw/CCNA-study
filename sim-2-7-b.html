Wires Behind the Wi-Fidrill

Wireless looks magic but it sits on Ethernet. Lightweight APs tunnel everything in CAPWAP back to the WLC — so the AP switchport is access in the AP-management VLAN. Autonomous APs bridge SSIDs to VLANs locally — so their switchport is a trunk. WLC uplinks = trunk. WLC LAG = static EtherChannel (mode on) — most WLCs don't speak LACP/PAgP for LAG. Why each role fits

Lightweight AP → Access SW

Access port in the AP-management VLAN. CAPWAP encapsulates ALL client traffic in IP/UDP back to the WLC. The switch only delivers one VLAN's unicast — no client-VLAN trunking required.

Autonomous AP → Switch

Trunk port. The AP itself bridges SSID-to-VLAN. Multiple SSIDs → multiple VLANs → 802.1Q-tagged uplink so the switch knows which VLAN each client frame belongs to.

Access SW → Dist SW

Trunk. Carries every client VLAN, AP-mgmt VLAN, voice, and so on between layers.

WLC ⇆ Dist (LAG)

Static EtherChannel — mode on. Cisco WLC LAG historically uses no negotiation — both sides must be set to on. The bundle is a trunk carrying client + management VLANs.

Worked example — switch config for an LWAP and a WLC LAG

! Access switch port to a lightweight AP SW1(config)# interface FastEthernet0/12 SW1(config-if)# switchport mode access SW1(config-if)# switchport access vlan 30 ! AP-mgmt VLAN SW1(config-if)# spanning-tree portfast SW1(config-if)# spanning-tree bpduguard enable ! Distribution switch — LAG to WLC (mode on, both sides) DistSW(config)# interface range Te1/0/47-48 DistSW(config-if-range)# channel-group 10 mode on DistSW(config)# interface Port-channel10 DistSW(config-if)# switchport mode trunk DistSW(config-if)# switchport trunk allowed vlan 10,20,30,99 ! WLC side (Catalyst 9800) WLC(config)# port-channel load-balance src-dst-ip

Faded prompt — your turn

Migration: site replaces autonomous APs with a Catalyst 9800 WLC + lightweight APs. Existing AP switchports are configured as trunks. After the swap, what should those ports become — and why?

Answer: change to access ports in the AP-mgmt VLAN. Lightweight APs don't bridge SSIDs locally; CAPWAP tunnels every client frame back to the WLC. The switch only needs to carry one VLAN to the AP. Leaving them trunks works but is sloppy — wider broadcast domain, more attack surface, more confusing for the next operator.
