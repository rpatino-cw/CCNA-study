Domain 1 — Network Fundamentals

20%

Roles, topologies, cabling, IP addressing (v4 + v6), wireless principles, virtualization, switching basics. Foundation domain — most other domains assume this.

1.1Network device roles
  • Router — L3, routes between networks, IP forwarding, NAT, ACLs.
  • L2 Switch — MAC table, forwards frames within VLAN.
  • L3 Switch — switching + routing, SVIs for inter-VLAN.
  • Next-Gen FW — stateful + app awareness + IPS + URL filter.
  • IPS inline (blocks) · IDS passive (alerts).
  • AP — wireless bridge to wired LAN. Autonomous = standalone. Lightweight = CAPWAP-tunneled to WLC.
  • WLC — central control of lightweight APs (config, policy, RF).
  • DNA Center — campus SDN controller (intent-based, assurance, telemetry).
  • Endpoints — PCs, phones, printers, IoT (consumers, not infrastructure).
  • Servers — provide services (DNS, DHCP, file, web).
  • PoE — power over Ethernet for IP phones, APs, cameras.
PoE standards — power budget per port
Checkpoint
An IPS detects an attack signature. What does it do?
B. IPS is inline and blocks. IDS is passive and alerts only. Easy exam trap: read carefully — "prevention" = inline = IPS.

1.2Topology architectures
  • 2-tier (collapsed core) — access + (distribution+core) merged. Small/medium campus.
  • 3-tier — access → distribution → core. Large campus, scalable.
  • Spine-leaf — every leaf to every spine, equal-cost paths, predictable latency. Modern DC.
  • WAN — connects sites across geography (MPLS, internet, leased line, SD-WAN).
  • SOHO — single multi-function device (router + switch + AP + firewall).
  • On-prem = your hardware. Cloud = pay-as-you-go (IaaS/PaaS/SaaS).
Spine-leaf — every leaf reaches every spine in 1 hop

1.8 / 1.9IPv6 — addressing & types
  • 128 bits, 8 hex groups of 4. :: collapses one run of zeros (only once per address).
  • Global Unicast 2000::/3 — public.
  • Unique Local FC00::/7 — private (FD00::/8 used in practice).
  • Link-Local FE80::/10 — every IPv6 iface has one, never routed.
  • Multicast FF00::/8 — replaces broadcast. FF02::1 all nodes, FF02::2 all routers.
  • Anycast — multiple hosts share an address; closest one wins.
  • Loopback ::1/128 · Unspecified ::/128.
  • SLAAC — host generates IID, listens for RA, builds address. DHCPv6 stateful or stateless.
Modified EUI-64 — generate IID from MAC
Checkpoint
Which IPv6 prefix is link-local?
C. FE80::/10 = link-local. FC00::/7 = unique local (private). FF00::/8 = multicast. 2000::/3 = global unicast (2001:: is a common subset).

  • VM — full guest OS on a hypervisor. Type 1 = bare-metal (ESXi, Hyper-V, KVM). Type 2 = hosted (VirtualBox, VMware Workstation).

1.13Switching concepts
  • Switch learns source MAC → port mapping into MAC table (CAM).
  • Unknown unicast = flood to all ports in same VLAN except inbound.
  • Broadcast (FF:FF:FF:FF:FF:FF) = flood within VLAN.
  • Known unicast = forward to mapped port only.
  • Cut-through = forwards after reading dest MAC (low latency, can pass errors). Store-and-forward = full frame + CRC check (default).
  • ARP — IP→MAC lookup. ARP request = broadcast. ARP reply = unicast.
Checkpoint
A switch receives a frame with destination MAC not in its table. What does it do?
A. Unknown unicast = flood within VLAN. The reply will populate the MAC table; future frames go directly.

Domain 2 — Network Access

20%

VLANs, trunks, STP, EtherChannel, wireless infrastructure (WLC, AP modes, WLAN config). Switching mastery domain.

2.1VLANs (access vs trunk)
  • VLAN range 1–4094. VLAN 1 = default. 1002–1005 reserved (legacy Token Ring/FDDI).
  • Access port = single VLAN, no tag.
  • Trunk port = carries multiple VLANs, 802.1Q tag (4 bytes inserted).
  • Native VLAN = untagged on trunk. Default 1. Mismatch = security + CDP error.
  • Voice VLAN = tagged voice, untagged data on access port (special config).
Switch CLIaccess port + trunk
Outcome
  • VLAN 10 created and named.
  • gi0/1 = access in VLAN 10 (single-VLAN endpoint).
  • gi0/24 = trunk carrying 10/20/30 with native 99 (mismatch-proof if both ends agree).
  • Verify: show vlan brief, show interfaces trunk.
Sim2.1 VLANs
2.2802.1Q trunks
  • 802.1Q tag = 4 bytes inserted after src MAC: TPID 0x8100 + PCP/DEI + VLAN ID.
  • Native VLAN traffic is not tagged.
  • DTP = Dynamic Trunking Protocol (Cisco). Modes: dynamic auto, dynamic desirable, trunk, access, nonegotiate.
  • Best practice: hard-set switchport mode trunk + switchport nonegotiate on infra links.
Sim2.2 802.1Q Tag
2.3CDP / LLDP + EtherChannel
  • CDP — Cisco-proprietary. show cdp neighbors. Multicast 01:00:0C:CC:CC:CC.
  • LLDP — IEEE 802.1AB, vendor-neutral. show lldp neighbors.
  • Both: hello every 60s default, hold time 180s. Disable on edge ports for security.

2.4Spanning Tree (STP)
  • Prevents L2 loops. BPDU = Bridge Protocol Data Unit (every 2 sec).
  • Root election: lowest BID wins. BID = priority (default 32768) + MAC. Lower MAC tiebreaks if priorities tie.
  • Port roles: Root (toward root), Designated (away from root), Blocking/Alternate (loop-prevention).
  • 802.1D states: Blocking → Listening → Learning → Forwarding (~50s).
  • RSTP (802.1w): Discarding → Learning → Forwarding (~6s).
  • PortFast — edge ports skip listening/learning. BPDU Guard — err-disable port if a BPDU is received on a PortFast port.
  • PVST+ = per-VLAN STP (Cisco default). Rapid-PVST+ = RSTP per VLAN.
Root election decision tree
Checkpoint
Three switches all have priority 32768. Which becomes root?
B. Tiebreaker = lowest MAC address. To deterministically pick a root, set spanning-tree vlan 1 root primary or lower priority manually (4096 increments).

3.1Routing table components
  • Codes: C connected · L local /32 · S static · D EIGRP · O OSPF · B BGP · * default.
  • Longest prefix match wins — most specific route picked, regardless of AD/metric.
  • Administrative Distance tie-breaks routes from different sources.

Checkpoint
Router has both O 10.1.0.0/16 (AD 110) and S 10.1.1.0/24 (AD 1). Packet for 10.1.1.5 takes which route?
C. Longest prefix match always wins. /24 > /16. AD only matters when two sources have the same prefix length. Classic exam trap.

3.4OSPFv2 single-area
  • Link-state, runs Dijkstra SPF, AD 110.
  • Cost = 10⁸ / bandwidth (default reference). Override with auto-cost reference-bandwidth or per-interface ip ospf cost.
  • Router ID selection: highest configured loopback IP → highest physical IP → manual router-id. Stable RIDs prevent reconvergence.
  • Hello 10s / Dead 40s on broadcast. Both must match for adjacency.
  • Network types: broadcast (DR/BDR elected), point-to-point (no DR), non-broadcast.
  • DR election: highest priority, then highest RID. Priority 0 = never DR.
  • Adjacency states: Down → Init → 2-Way → ExStart → Exchange → Loading → Full.
  • Neighbor must match: area ID, hello/dead timers, subnet, auth, MTU, area type, stub flag.
OSPF neighbor states (in order)
R1 CLIOSPF area 0
Outcome
  • Process 1 with stable RID 1.1.1.1.
  • Network statement enables OSPF on matching interfaces (wildcard 0.0.0.255 = /24).
  • Passive interface = no Hellos sent (LAN with hosts only).
  • Per-interface enable + p2p type skips DR/BDR election.
  • Verify: show ip ospf neighbor · show ip ospf interface brief.
Checkpoint
Two OSPF routers stuck in 2-Way state on a broadcast segment. What's true?
B. 2-Way is normal between non-DR/BDR routers. Only DR↔all and BDR↔all go Full. DROther↔DROther stays at 2-Way by design.

5.4IPsec VPN
  • Site-to-site — gateway-to-gateway (always on).
  • Remote-access — client-to-gateway (SSL/TLS or IPsec).
  • IPsec phases: IKE Phase 1 (ISAKMP SA, peer auth) → Phase 2 (IPsec SA, data encryption).
  • Protocols: AH (auth, no encryption) · ESP (auth + encryption — almost always used).
  • Modes: Transport (host-to-host, original IP header preserved) · Tunnel (gateway-to-gateway, new IP header).
  • DMVPN, GRE (no encryption alone — wrap with IPsec), GETVPN.

5.5Access Control Lists (ACLs)
  • Standard 1–99, 1300–1999 — match source IP only. Place near destination.
  • Extended 100–199, 2000–2699 — match src/dst/proto/port. Place near source.
  • Named ACLs preferred today. Top-down evaluation. Implicit deny all at end.
  • Wildcard mask = inverse of subnet mask. /24 → 0.0.0.255. host x.x.x.x = 0.0.0.0 wildcard.

6.1Traditional vs controller-based
  • Traditional — distributed control plane, per-device CLI config. Each device runs its own routing/STP/etc.
  • Controller-based (SDN) — control plane centralized. Controller pushes intent → southbound APIs (NETCONF, OpenFlow) → devices.
  • Northbound = controller ↔ apps/admin (REST). Southbound = controller ↔ devices.
  • Cisco solutions: DNA Center (campus), ACI (data center, APIC), vManage (SD-WAN).

6.2Network management approaches
  • Imperative (CLI: "do X, then Y") vs declarative (intent: "the state should be X").
  • Per-device manual vs controller-driven vs API-driven.
  • Telemetry: SNMP polling (legacy) vs streaming telemetry (modern, push-based, gNMI).

Which is valid JSON?
A. JSON requires double-quoted keys + values, square brackets for arrays. B is JS object literal (single quotes invalid). C is YAML. D is XML.

Test-Taking Tactics

        edge
      

      
    
        
  • Flag and move on. If a question takes >90 sec, flag it. Come back.
    • 
        
  • Sims: if a syntax escapes you, type ? at any prompt — Cisco IOS context help works in the sim. Always run show run after configuring to verify.
    • 
        
  • Drag-and-drop: lock the obvious matches first, then narrow remaining options.
    • 
        
  • Read every word. "Which two" vs "which" · "best" vs "valid" · "MOST likely" — these flip answers.
    • 
        
  • Don't second-guess if first instinct matches a known fact. Override only with new evidence.
    • 
        
  • Eliminate. Remove 2 wrong answers, then guess between 2 if forced — 50% > 25%.
    • 
        
  • Time check at Q40 and Q80. Should have ~75 min left at Q40, ~30 min at Q80.
    • 
        
  • Sims first or last? Some prefer last (max time), some first (mind fresh). Pick what worked in your Boson runs.
    • 
        
  • Subnetting: always sketch on whiteboard. Network → broadcast → range → answer. Don't math in head.
    • 
        
  • Caffeine, water, snack — exam is 2 hours; don't crash at minute 90.
    • 
      

      
Mindset: the test rewards careful readers. Most wrong answers come from misreading, not lack of knowledge. Slow down on the first read.
