One Port, Two VLANsdrill

An IP phone is a tiny 3-port switch: uplink, PC pass-through, internal phone. Plug a laptop into the phone, plug the phone into the wall — both share one switchport. The switch tags voice traffic on the voice VLAN and leaves data untagged on the access VLAN. The phone learns its tag from CDP / LLDP-MED. Build the config below — break it on purpose to see what fails.

The daisy-chain

A voice-VLAN port is not a trunk. It looks like an access port that quietly passes one extra tagged VLAN — the voice VLAN — and the switch only accepts that one tag. Everything else from the wall stays untagged on the access VLAN.

What the wire actually carries

Why one port and not a trunk

Voice-VLAN access port

Switch behavior: untagged data → access VLAN, tagged voice → voice VLAN. Phone learns the voice tag via CDP/LLDP-MED. show interface trunk does NOT list it as a trunk. Mac-learning on this port is per-VLAN, both sides clean.

Trunk

Carries every allowed VLAN as 802.1Q-tagged. Native VLAN traps. PC behind a phone would have to participate in 802.1Q (most NICs don't). Mac-learning gets noisy. Wrong tool for end-user ports.

Worked example — verifying the port

! voice-VLAN access port — note "trunk" is NOT in the output SW1# show interfaces FastEthernet0/1 switchport Name: Fa0/1 Switchport: Enabled Administrative Mode: static access Operational Mode: static access Access Mode VLAN: 10 (DATA) Trunking Native Mode VLAN: 1 (default) Voice VLAN: 110 (VOICE) Trunking VLANs Enabled: ALL ... SW1# show cdp neighbor detail | include Phone|Port|Voice Device ID: SEP00112233AABB (Cisco IP Phone 7945) Interface: FastEthernet0/1, Port ID (outgoing port): SW PORT Voice VLAN: 110

Faded prompt — your turn

Phone won't register. PC behind it has DHCP and reaches the gateway. Sniff shows no 802.1Q tag from the phone. Which one line is missing?

  • Answer: switchport voice vlan 110 on Fa0/1.
  • Without it, CDP/LLDP-MED never advertises a voice VLAN to the phone, so the phone tags nothing and falls back to the access VLAN, where the call manager isn't reachable.
