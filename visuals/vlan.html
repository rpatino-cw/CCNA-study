802.1Q Tag Insertion Animation

Watch how an Ethernet frame gets a 4-byte 802.1Q tag inserted as it enters a trunk port, and stripped on the receiving end.

Click "Play Tag Insertion" to watch the 802.1Q tag get inserted into an Ethernet frame as it enters the trunk link, then stripped on the other end.

802.1Q Tag Structure (4 bytes)

TPID
0x8100
16 bits PCP
Priority
3 bits DEI
Drop
1 bit VID
VLAN ID
12 bits

TPID = Tag Protocol Identifier (always 0x8100 for 802.1Q).
PCP = Priority Code Point (QoS, 0-7).
DEI = Drop Eligible Indicator.
VID = VLAN Identifier (0-4095, usable 1-4094).

Key Points

  • The tag is inserted BETWEEN Src MAC and EtherType -- the frame gets 4 bytes larger.
  • FCS is recalculated after tag insertion (and after removal).
  • Only trunk ports add/remove 802.1Q tags.
  • Access ports deliver frames with NO tag to end devices.
  • Maximum frame size increases from 1518 to 1522 bytes.

Native VLAN Comparison: Tagged vs Untagged on Trunk

See the difference between a tagged frame (VLAN 10) and an untagged frame (native VLAN) crossing the same trunk link simultaneously.

Click "Play Comparison" to see both a tagged and an untagged (native VLAN) frame cross the trunk simultaneously.
Why native VLAN mismatch is dangerous: If SW1 has native VLAN 1 and SW2 has native VLAN 99, an untagged frame from VLAN 1 on SW1 will be placed into VLAN 99 on SW2. This causes traffic to leak between VLANs -- a classic VLAN hopping attack vector. Best practice: Always match native VLANs on both ends, use an unused VLAN (e.g., 999), and consider vlan dot1q tag native to tag even native VLAN traffic.

CDP/LLDP Discovery
Click "Play Voice VLAN Flow" to see how the switch uses CDP/LLDP to tell the IP phone which VLAN to tag voice traffic on, while data from the PC passes through untagged.
Voice VLAN key concept: The switch port is configured as an access port with an additional voice VLAN. Data from the PC arrives untagged (VLAN 10). Voice from the IP phone arrives 802.1Q-tagged with VLAN 150. The switch uses CDP or LLDP to inform the phone which VLAN to use for voice. This gives voice traffic QoS priority (CoS 5 by default) without requiring the PC to understand VLANs.

Router-on-a-Stick Topology

Click "Animate" to see how a frame from VLAN 10 reaches VLAN 20 through the router. The frame travels up the trunk to the router, gets routed between subinterfaces, then returns down the same trunk to the destination VLAN.
Router Configuration
! Router subinterfaces
interface Gi0/0.10
  encapsulation dot1Q 10
  ip address 192.168.10.1 255.255.255.0
interface Gi0/0.20
  encapsulation dot1Q 20
  ip address 192.168.20.1 255.255.255.0
Switch Configuration
! Trunk to router
interface Gi0/1
  switchport trunk encapsulation dot1q
  switchport mode trunk
  switchport trunk allowed vlan 10,20

! Access ports
interface Gi0/2
  switchport mode access
  switchport access vlan 10
interface Gi0/3
  switchport mode access
  switchport access vlan 20

Layer 3 Switch with SVIs

Click "Animate" to see how inter-VLAN routing works internally on a Layer 3 switch. The frame enters the switch, is routed by the ASIC between SVIs, and exits on the destination VLAN -- all within the same switch.
Layer 3 Switch Configuration
! Enable routing
ip routing

! SVI for VLAN 10
interface vlan 10
  ip address 192.168.10.1 255.255.255.0
  no shutdown

! SVI for VLAN 20
interface vlan 20
  ip address 192.168.20.1 255.255.255.0
  no shutdown

! Access ports
interface Gi0/1
  switchport mode access
  switchport access vlan 10
interface Gi0/2
  switchport mode access
  switchport access vlan 20
Comparison
! Router-on-a-Stick
  - Requires external router
  - Single trunk link (bottleneck)
  - "Hairpin" traffic pattern
  - Cheaper equipment

! Layer 3 Switch (SVI)
  - All-in-one device
  - Wire-speed routing via ASIC
  - No trunk bottleneck
  - More expensive but scalable
  - Preferred for enterprise
Key advantage: With an L3 switch, inter-VLAN traffic is routed internally by the switch ASIC at wire speed. There is no need for traffic to exit the switch and hairpin back through a router, eliminating the trunk link bottleneck inherent in Router-on-a-Stick designs.
Creating VLANs 
! Create a VLAN and name it
Switch(config)# vlan 10
Switch(config-vlan)# name Sales

Switch(config)# vlan 20
Switch(config-vlan)# name Engineering
! Verify VLANs
Switch# show vlan brief
Access Ports 
! Assign port to VLAN as access
Switch(config)# interface Gi0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10
! Verify
Switch# show interfaces Gi0/1 switchport
Trunk Ports 
! Configure trunk port
Switch(config)# interface Gi0/24
Switch(config-if)# switchport trunk encapsulation dot1q
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk native vlan 99
Switch(config-if)# switchport trunk allowed vlan 10,20,30
! Verify trunk
Switch# show interfaces trunk

DTP (Dynamic Trunking Protocol) Negotiation Matrix
Dynamic Auto Dynamic Desirable Trunk Access
Dynamic Auto Access Trunk Trunk Access
Dynamic Desirable Trunk Trunk Trunk Access
Trunk Trunk Trunk Trunk Limited
Access Access Access Limited Access
Best Practice: Always explicitly set port mode and use switchport nonegotiate to disable DTP. This prevents unintended trunk formation and improves security.
