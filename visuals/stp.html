StateReplaces
DiscardingDisabled + Blocking + Listening
LearningSame as STP Learning
ForwardingSame as STP Forwarding
Convergence: ~1-2 seconds
Port Roles: Root, Designated, Alternate, Backup
Key Advantages:
  • Proposal/Agreement mechanism for rapid transitions
  • Alternate port = instant failover (pre-computed backup)
  • Edge ports (PortFast equivalent) built into protocol
  • No need to wait for timer expiry
  • All bridges generate BPDUs (not just root)

STP Election Rules

  1. Lowest Bridge ID wins Root Bridge -- Bridge ID = Priority (16 bits) + MAC Address (48 bits). Priority compared first; if tied, lowest MAC wins.
  2. Root Ports -- On each non-root switch, the port with the best (lowest cost) path to root.
  3. Designated Ports -- On each network segment, the port that provides the best path toward the root bridge.
  4. Blocked / Non-Designated Ports -- All other ports are blocked to prevent loops. In RSTP these become Alternate or Backup ports.

STP Security Features

✓

PortFast

Recommended

What it does: Skips the Listening and Learning states, transitioning directly from Blocking to Forwarding. Eliminates the 30-second delay for end devices.

When to use: On access ports connected to end devices (PCs, printers, servers). Never use on ports connected to other switches -- it can cause temporary loops.

Configuration
! Per-interface interface GigabitEthernet0/1 spanning-tree portfast ! Global (all access ports) spanning-tree portfast default
✓

BPDU Guard

Recommended

What it does: If a BPDU is received on a PortFast-enabled port, the port is immediately placed into err-disabled state. Prevents rogue switches from disrupting the STP topology.

When to use: Always pair with PortFast on access ports. Essential for protecting against accidental or malicious switch connections.

Configuration
! Per-interface interface GigabitEthernet0/1 spanning-tree bpduguard enable ! Global (all PortFast ports) spanning-tree portfast bpduguard default ! Auto-recovery errdisable recovery cause bpduguard errdisable recovery interval 300
!

BPDU Filter

Use with Caution

What it does: Stops sending and receiving BPDUs on a port. Effectively disables STP on that port. Per-interface mode is different from global mode.

When to use: Rare scenarios where you need to completely disable STP on a port. Warning: Can cause loops if misconfigured since the port will have no STP protection.

Configuration
! Per-interface (dangerous -- completely disables STP) interface GigabitEthernet0/1 spanning-tree bpdufilter enable ! Global (only on PortFast ports, reverts if BPDU received) spanning-tree portfast bpdufilter default
✓

Root Guard

Recommended

What it does: Prevents a port from becoming a Root Port. If a superior BPDU is received, the port is placed into root-inconsistent (blocking) state. Protects the root bridge placement.

When to use: On designated ports facing toward the network edge, where no switch should be able to become root bridge. Protects your intended STP topology.

Configuration
interface GigabitEthernet0/1 spanning-tree guard root
!

Loop Guard

Use with Caution

What it does: Prevents alternate/backup ports from transitioning to designated/forwarding if BPDUs stop being received (e.g., due to a unidirectional link failure). The port enters loop-inconsistent state instead.

When to use: On non-designated (blocking) ports, particularly on redundant links. Complements UDLD (Unidirectional Link Detection). Cannot be used on the same port as Root Guard.

Configuration
! Per-interface interface GigabitEthernet0/1 spanning-tree guard loop ! Global spanning-tree loopguard default
