3.4 Single-Area OSPFv2 — Read This Before Anything Else

Cisco 200-301 §3.4.a-d. OSPF = link-state, AD 110, cost 10⁸/BW. Single area focus = area 0 backbone. Memorize neighbor states + DR/BDR election + router-ID rules cold. ~10% of exam = highest single-topic ROI.

1. The 30-Second Answer read this twice

OSPF = link-state IGP. Routers flood LSAs to all neighbors → each builds an identical Link-State Database (LSDB) → SPF (Dijkstra) computes the shortest path tree → results go in routing table. AD 110. Cost = 10⁸/bandwidth. Hello/dead = 10s/40s on broadcast + P2P. Single area = everything in area 0.
  • Neighbor adjacencies (3.4.a) — formed via Hello packets (multicast 224.0.0.5). State machine: Down → Init → 2-Way → ExStart → Exchange → Loading → Full.
  • Point-to-point (3.4.b) — exactly 2 neighbors per link (e.g. serial WAN). NO DR/BDR election.
  • Broadcast (3.4.c) — Ethernet, 2+ neighbors. DR + BDR elected. Other routers (DROther) form full adjacency only with DR/BDR (224.0.0.6); DR floods to all (224.0.0.5).
  • Router ID (3.4.d) — unique 32-bit ID per router. Selection priority: 1) router-id X.X.X.X manual, 2) highest loopback IP, 3) highest active interface IP. Set at OSPF process startup; doesn't auto-update if interface changes.

Memorize: AD 110 · cost 10⁸/BW · hello 10s · dead 40s · multicast 224.0.0.5 (all OSPF) + 224.0.0.6 (DR/BDR)

2. OSPF Anatomy at a Glance visual

OSPF is a 4-layer process: Hello finds neighbors → LSAs flood to build identical LSDBs → SPF computes shortest paths → routes installed in RIB. DR/BDR optimization only on multi-access network types.

3. Neighbor Adjacencies — 3.4.a state machine

OSPF speaks Hello on every OSPF-enabled interface. Two routers form a neighbor relationship when their Hellos match on key parameters.

Hello must agree on:

  • Area ID
  • Subnet (must be on same subnet/mask)
  • Hello + dead intervals (10s/40s broadcast; 30s/120s NBMA — but in CCNA scope mostly 10/40)
  • Authentication (if configured — keys must match)
  • Stub area flag
  • MTU (must match for full adjacency to form past ExStart)

Neighbor state machine:

StateMeaning
DownNo Hellos heard from neighbor
AttemptNBMA only — actively trying to contact neighbor
InitHello received, but our router-ID NOT yet listed in neighbor's Hello (one-way)
2-WayBoth routers see each other in each other's Hellos. DR/BDR election happens here on broadcast networks. DROther stays at 2-Way with other DROthers.
ExStartMaster/slave election (higher router-ID = master). Begin DBD exchange.
ExchangeDBD packets exchanged describing each side's LSDB.
LoadingLSRs sent for missing LSAs; LSU replies fill the LSDB.
FullLSDBs synchronized. Adjacency complete. Stable.
Mnemonic: "Down · Init · 2-Way · ExStart · Exchange · Loading · Full" — DI-2-EX-EX-LF. Memorize. Most exam Qs hit on which state happens when.

Trap to recognize
CCNA usually focuses on broadcast (Ethernet) and point-to-point. NBMA exists in stems mostly to confuse. If the link is Ethernet → broadcast → DR/BDR elected. If it\'s serial PPP → P2P → no DR/BDR.

5. DR / BDR Election broadcast networks only

On broadcast (Ethernet) and NBMA networks, OSPF elects a Designated Router (DR) and Backup DR (BDR) per segment. Reduces flooding overhead — instead of N×N adjacencies, all routers form full adjacency only with DR + BDR.

Election rules (in order):

  1. Highest OSPF priority wins (range 0-255, default 1). Priority 0 = NEVER becomes DR/BDR (opt-out).
  2. Highest router-ID wins as tiebreaker.
  3. Election is non-preemptive — once a DR is elected, it stays even if a higher-priority router joins later. Reload required to re-elect (or clear ip ospf process).

Roles:

  • DR — receives LSAs from all routers via 224.0.0.6 (AllDRouters); floods to all via 224.0.0.5 (AllSPFRouters).
  • BDR — listens silently; takes over if DR dies.
  • DROther — every non-DR/non-BDR router. Forms FULL adjacency with DR + BDR only. Other DROthers stay at 2-Way state with each other (intentional efficiency, NOT a problem).

Set priority: ip ospf priority 100 on the interface. Set 0 to opt out: ip ospf priority 0.

Mnemonic: "DROthers stay 2-Way" — this is normal, not a bug. Only DR + BDR reach Full with everyone.

6. Router ID — 3.4.d unique 32-bit ID

The Router ID (RID) is a unique 32-bit number that identifies the router in OSPF. Looks like an IP address but doesn\'t have to be one. Required.

RID selection priority (router picks the FIRST that exists, at OSPF process startup):

  1. Manually configured: router-id X.X.X.X under router ospf N. Always wins. Recommended.
  2. Highest loopback IP (any active loopback, even if not OSPF-enabled). Loopbacks always up = stable RID.
  3. Highest active interface IP at process startup time.

Important: RID is set ONCE at OSPF process startup and DOES NOT auto-change if you add a higher loopback later. To force re-evaluation: clear ip ospf process or reload.

R1(config)# router ospf 1 R1(config-router)# router-id 1.1.1.1

Best practice: always manually configure the RID. Avoids surprises when interfaces flap or new ones get added.

Trap to recognize
If you change the RID after OSPF is running, it does NOT take effect until you clear ip ospf process. Common gotcha — config looks right but neighbors haven\'t reconverged.

7. Configuration memorize cold

Two ways to enable OSPF on an interface:

(a) Network statement under OSPF process (legacy / classic):

R1(config)# router ospf 1 R1(config-router)# router-id 1.1.1.1 R1(config-router)# network 10.10.10.0 0.0.0.255 area 0 R1(config-router)# network 192.168.1.0 0.0.0.255 area 0
  • Process ID (1) — locally significant. Doesn\'t need to match neighbors.
  • WILDCARD mask (0.0.0.255 = /24). NOT subnet mask. Bit-inverse (0 = match, 1 = ignore).
  • area 0 — single area = everything in backbone area.

(b) Interface mode (modern, recommended):

R1(config)# router ospf 1 R1(config-router)# router-id 1.1.1.1 R1(config-router)# exit R1(config)# interface GigabitEthernet0/0 R1(config-if)# ip ospf 1 area 0
  • Cleaner: enable per-interface explicitly, no wildcard math.
  • Same effect as network statement.

Suppress Hellos on host-facing interfaces: passive-interface GigabitEthernet0/2 — OSPF route still advertised but no Hellos sent (saves CPU + prevents accidental adjacency with hosts).

8. Cost + Metric 10⁸ / bandwidth

OSPF cost is computed per-link as: cost = reference-bandwidth (10⁸ default) ÷ interface bandwidth in bps. Path cost = sum of all egress link costs.

Interface speedDefault cost
10 Gbps1 (rounded up from 0.01)
1 Gbps1
100 Mbps (Fast Ethernet)1
10 Mbps (Ethernet)10
1.544 Mbps (T1)64
56 Kbps1785

Problem: default reference bandwidth = 100 Mbps. Modern 1G/10G/100G links all show cost 1 — can\'t differentiate. Fix: raise reference: auto-cost reference-bandwidth 100000 (= 100 Gbps reference). Must do on EVERY router for consistency.

Manual cost override on an interface: ip ospf cost 50. Overrides calculated cost.

OSPF metric in routing table = total path cost. O 10.10.0.0/24 [110/65] → AD 110, total cost 65.

11. Cisco's Favorite Traps spot the wrong answer

Trap #1 — "DROthers in 2-WAY state means broken"
Wrong. 2-WAY between two DROthers is NORMAL. Only DR + BDR form FULL adjacency with every router. DROthers form FULL only with DR/BDR; with other DROthers they stay at 2-Way intentionally.
Trap #2 — "OSPF process IDs must match"
Wrong. Process ID is locally significant. R1 can run "ospf 1" and R2 can run "ospf 99" — they\'ll still form adjacency if hello/area/auth match. Process ID is just for the local box.
Trap #3 — "Network statement uses subnet mask"
Wrong. Uses WILDCARD mask (0.0.0.255 for /24). Bit-inverse of subnet mask. Mix them up = OSPF doesn\'t enable on the right interfaces.
Trap #4 — "Highest interface IP wins for router ID"
Half-wrong. Manual router-id wins first. Then highest LOOPBACK. Then highest interface IP. Loopback beats physical interface even if physical IP is higher.
Trap #5 — "Router ID auto-updates when interfaces change"
Wrong. RID is set at OSPF process startup and stays. Add a higher loopback later → RID does NOT change until you clear ip ospf process or reload.
Trap #6 — "Priority 0 means router cannot become BDR but can become DR"
Wrong. Priority 0 = opt-out of BOTH DR and BDR election entirely. Stays as DROther forever on that segment.
Trap #7 — "DR election is preemptive — higher-priority router takes over"
Wrong. OSPF DR election is NON-PREEMPTIVE. Once elected, DR stays. Even if a router joins with higher priority later. To re-elect: clear ospf process or reload.

12. Top 10 Must-Know if-only-this

Memorize until automatic
  1. OSPF magic: AD 110 · Cost = 10⁸/BW · Hello 10s · Dead 40s · Multicast 224.0.0.5 (all) and 224.0.0.6 (DR/BDR).
  2. Neighbor states: Down → Init → 2-Way → ExStart → Exchange → Loading → Full. Memorize order cold.
  3. DR/BDR election: highest priority (default 1, 0=opt-out) → highest router-ID. NON-PREEMPTIVE.
  4. DROthers stay at 2-Way with each other — normal, not broken.
  5. Router-ID priority: manual → highest loopback → highest active interface. Set ONCE at process startup; doesn\'t auto-update.
  6. Network types: broadcast (Ethernet) = DR/BDR. Point-to-point = no DR/BDR. Loopback advertised as /32.
  7. Network statement uses WILDCARD MASK: network 10.10.10.0 0.0.0.255 area 0.
  8. Process ID is LOCALLY SIGNIFICANT — doesn\'t need to match between routers.
  9. Hello must agree on: area, subnet, hello/dead intervals, auth, MTU, stub flag. Mismatch = no adjacency.
  10. Verify: show ip ospf neighbor, show ip ospf interface brief, show ip protocols, show ip route ospf.

13. Self-Check no peeking

Q1. List the 7 OSPF neighbor states in order.

Reveal

Down · Init · 2-Way · ExStart · Exchange · Loading · Full. 2-Way is where DR/BDR election happens on broadcast networks. Refer back: §3.

Q2. R1 and R2 are both on a Gigabit Ethernet segment. R1 priority 100, R2 priority 1. R3 (priority 200) joins LATER. Who is DR?

Reveal

R1. DR election is non-preemptive. R1 was elected when only R1+R2 existed. R3\'s higher priority is ignored — R3 becomes DROther until next election (reload or clear ip ospf process). Refer back: §5, §11 trap #7.

Q3. Two OSPF DROthers on a multi-access segment — what state should they be in with each other?

Reveal

2-WAY/DROTHER. Normal. Only DR + BDR reach FULL with every router. DROthers stay at 2-Way with each other — intentional efficiency. Refer back: §11 trap #1.

Q4. Router has manual router-id 1.1.1.1, loopback 9.9.9.9, and physical interface 8.8.8.8. What\'s the OSPF RID?

Reveal

1.1.1.1 (manual wins). Manual config beats loopback beats physical. Refer back: §6.

Q5. Configure OSPF area 0 on interface Gi0/0 using interface-mode (modern method). Process ID 1.

Reveal

router ospf 1 (defines process), then on the interface: interface Gi0/0 + ip ospf 1 area 0. Refer back: §7.
