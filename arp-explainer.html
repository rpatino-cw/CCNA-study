ARP Table Explainer

Address Resolution Protocol maps an IP to a MAC on the local subnet. Watch the broadcast-request / unicast-reply exchange, then see spoofing and proxy ARP.

Normal Resolution

Host A needs Host C's MAC.

Cache Hit

Entry already in table, no broadcast.

ARP Spoof

Attacker poisons the cache.

Proxy ARP

Router answers on behalf of another subnet.

Six-Step Resolution

1
Check local cache. If mapping exists and hasn't expired, skip to step 6.
2
Build ARP request. "Who has 192.168.1.30? Tell 192.168.1.10."
3
Broadcast. Destination MAC = FF:FF:FF:FF:FF:FF. Every NIC on the subnet reads the frame.
4
Target responds. Only the host with that IP answers — a unicast reply with its MAC.
5
Cache it. Add to ARP table as Dynamic. Age starts counting; default 20 min on Cisco, 60 s idle on Windows.
6
Send the original frame. Now Layer 2 knows where to go.
Security: ARP has no authentication. An attacker can send unsolicited replies to poison your cache and sit in the middle (arpspoof, Ettercap, Cain). Mitigations: Dynamic ARP Inspection (DAI) with DHCP snooping on Cisco switches, port security, static ARP for critical hosts, arpwatch for monitoring.
