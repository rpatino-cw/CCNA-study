How the math works (universal recipe)
For any "given parent CIDR, need N subnets and H hosts" question:
  1. Bits to borrow for subnets (s): smallest s where 2^s ≥ N.
  2. Bits to keep for hosts (h): smallest h where 2^h - 2 ≥ H.
  3. Budget check: parent prefix + s + h ≤ 32. If over, problem doesn't fit.
  4. New prefix: parent prefix + s.
  5. Mask: convert /N to dotted-decimal (8 ones per full octet, partial via 256 - block size).
Example. 172.16.0.0/16, 40 subnets, 830 hosts.
s = ceil(log2 40) = 6 (because 2^6 = 64 ≥ 40).
h: 2^10 - 2 = 1022 ≥ 830, so h = 10.
16 + 6 + 10 = 32 ✓
New prefix = /16 + 6 = /22 → mask 255.255.252.0.
Result: 64 subnets, 1022 hosts each.

For host counts subtract 2 (network + broadcast). For subnet counts no subtraction (modern routers; CCNA assumes ip subnet-zero).
