IPv6, Made Visual

128 bits, eight hex groups, no broadcast, and a name for every kind of address. Type one in and watch it decompose.

Why IPv6?

IPv4 ran out of public addresses around 2011. IPv6 gives every device on Earth roughly 10²⁸ addresses each, kills broadcast, builds in autoconfiguration (SLAAC), and bakes link-local communication into every interface. The format looks scarier than IPv4 but follows three rules: 128 bits, eight 4-hex groups, and one collapse of consecutive zeros with ::.

Six address types

IPv6 replaces the IPv4 broadcast/private/public split with one prefix-driven map. Each prefix tells you exactly what the address is for.

TypePrefixScopeNotes
Global Unicast2000::/3Internet-routableThe "public" IPv6. First three bits are 001, so all start with 2 or 3 in the first hextet (e.g. 2001:db8::).
Unique Local (ULA)FC00::/7Site-local, not routable on the internetThe replacement for RFC 1918 (10.x, 192.168.x). In practice you'll see FD00::/8 with a random 40-bit Global ID.
Link-LocalFE80::/10One link only — never crosses a routerAuto-configured on every IPv6 interface. Used for neighbor discovery, OSPFv3, EIGRP for IPv6, and DHCPv6.
MulticastFF00::/8One-to-manyReplaces broadcast. Critical: FF02::1 all-nodes, FF02::2 all-routers, FF02::5/6 OSPFv3, FF02::A EIGRP for IPv6.
Anycast(no separate prefix)One-to-nearestSame address assigned to multiple devices — routing delivers to the closest. Looks like a unicast; you just configure the same address on multiple routers.

No broadcast. IPv6 deletes broadcast entirely — the FF00::/8 multicast block replaces every broadcast use case. Memorize FF02::1 (all hosts on the link) and FF02::2 (all routers on the link). They are the closest thing IPv6 has to "255.255.255.255".
Interactive · Address generation

EUI-64 builder

Modified EUI-64 turns a 48-bit MAC address into a 64-bit interface ID. Three steps: split the MAC, jam FFFE in the middle, then flip the universal/local bit. Click through to watch each step.

1
Take the 48-bit MAC
00:1B:44:11:3A:B7
2
Split into two halves (OUI · NIC)
001B44 · 113AB7
3
Insert FFFE between the halves
001B44 FFFE 113AB7
4
Flip the 7th bit (Universal/Local)
00 → 02 · 02 1B 44 FF FE 11 3A B7
5
Format as the 64-bit interface ID
021B:44FF:FE11:3AB7
6
Prepend a /64 prefix → full address
FE80::21B:44FF:FE11:3AB7 (link-local · auto)

Why flip the 7th bit? In a MAC, that bit is 0 = globally unique, 1 = locally administered. EUI-64 inverts the meaning so a 1 now indicates "burned-in." This means a normal MAC like 00:... becomes 02:... in the interface ID — most exam questions test exactly this flip. Modern OSes prefer RFC 4941 privacy addresses (random IID) by default, but EUI-64 is still a CCNA exam staple and shows up on Cisco router interfaces with ipv6 address ... eui-64.

How a host gets an IPv6 address

IPv4 has DHCP and that's it. IPv6 has three options — and routers tell hosts which one to use through flags in the Router Advertisement. Click Run to watch a host go from "no IP" to "fully configured."

1 · Host generates a link-local. Auto-creates FE80::/64 using EUI-64 or RFC 4941 randomness. Runs DAD (NS to itself) to make sure no neighbor already owns it.
2 · Host sends Router Solicitation. Multicast to FF02::2 (all-routers). "Anyone here? What prefix should I use?"
3 · Router replies with RA. Multicast to FF02::1 (all-nodes) — carries the /64 prefix, default gateway, and three flags: M (managed/DHCPv6), O (other config), A (autonomous/SLAAC).
4 · Host builds its GUA. Concatenates the announced prefix with its interface ID (EUI-64 or random). Runs DAD again on the full address.
5 · Optional: DHCPv6. If RA flags M=1, the host also runs DHCPv6 (UDP 547→546) for full address assignment. If only O=1, it asks DHCPv6 for DNS/NTP only — address came from SLAAC.

SLAAC

A=1, M=0, O=0
  • Host builds its own address from RA prefix + IID
  • No DHCP server needed
  • No record of who has what — challenge for audit
  • DNS via RDNSS option in RA (modern OSes)

Stateless DHCPv6

A=1, M=0, O=1
  • Address from SLAAC, options from DHCPv6
  • DHCPv6 server hands out DNS, NTP, domain name
  • No address tracking on the DHCP server
  • Common on enterprise networks

Stateful DHCPv6

A=0, M=1
  • DHCPv6 assigns the full address (like IPv4 DHCP)
  • Server keeps a lease database — auditable
  • Host still gets a link-local from FE80::/10
  • No router? Use a DHCPv6 relay agent

Watch out

Common mistakes

✗"You can use :: twice in one address."

Reality: Only once. Two :: would make the length ambiguous — there'd be no way to know how many zero hextets each represents.

✗"IPv6 has broadcast like IPv4."

Reality: No broadcast. Multicast (FF00::/8) replaces it. FF02::1 is the closest equivalent for "everyone on this link."

✗"Link-local is optional."

Reality: Every IPv6-enabled interface must have a link-local. Routing protocols (OSPFv3, EIGRP) and ND use it as the next-hop, not the GUA.

✗"EUI-64 leaves the MAC unchanged."

Reality: The 7th bit of the first byte is flipped. 00:1B:44… in a MAC becomes 02:1B:44… in the interface ID. Forgetting this flip is the most-tested EUI-64 trap.

✗"SLAAC needs a DHCPv6 server."

Reality: SLAAC is router-only. The host generates the IID itself and learns the prefix from the RA. DHCPv6 is a separate (and optional) service.

✗"Cisco enables IPv6 by default."

Reality: No. You need ipv6 unicast-routing globally, plus ipv6 address … or ipv6 enable on each interface. Without the global command the router won't forward IPv6 or send RAs.
