The exam shortcut

Collisions = normal on half-duplex, zero on full-duplex. Late collisions (collisions past 64 bytes / 512 bit-times) = duplex mismatch. That's the answer 95% of the time on the exam.

CRC / input errors = bad cable, EMI, bad NIC, dirty fiber. Input drops = ingress queue full (host can't keep up). Output drops = egress queue full (link too slow / QoS dropping).

What each counter actually means

Collisions half-duplex normal

Two stations transmit at the same time on a shared medium. Detected within the first 64 bytes (512 bit-times). Both back off, retry.

  • Normal on a hub or any half-duplex link.
  • Zero on full-duplex — no shared medium, no contention.
  • If you see collisions on a switch port, something is half-duplex on at least one side.
  • CSMA/CD: CS listen, MA anyone goes, CD detect, JAM 32 bits, exponential backoff.

Late collisions EXAM KEY

Collision detected past byte 64 (after the slot time window). Means the sender thought the medium was free but it wasn't.

  • #1 cause: duplex mismatch. One side full-duplex (no carrier sense), one side half-duplex.
  • #2 cause: cable too long, exceeds Ethernet timing budget.
  • Fix: hard-code both sides to the same speed/duplex, or set both to auto.
  • Late collisions on a full-duplex link → check the other side; auto-negotiation likely failed.

CRC, frame, input errors L1

Receiver re-computes the FCS (Frame Check Sequence). If it doesn't match the trailer, frame is dropped and CRC counter increments.

  • CRC bumps without frame bumps → bit-level corruption (EMI, marginal cable, faulty NIC).
  • Frame bumps too → bits AND framing wrong (loose connector, dying transceiver).
  • Runts = under 64 bytes. Giants = over 1518 (or 9216 for jumbo).
  • Fix: replace cable/SFP, move away from EMI source (motor, ballast, fluorescent), check ground.

Input vs output drops buffer / QoS

Drops = packet was received correctly but the device couldn't keep it. Different from errors (which are corruption).

  • Input drops = ingress queue full. CPU/ASIC can't process fast enough, or buffer too small for burst.
  • Output drops = egress queue full. Link too slow for offered load, or QoS policy is dropping the class.
  • QoS doesn't make a slow link fast — it picks what to drop when the link is full.
  • Fix: bigger buffers (limited), faster link, EtherChannel, or a QoS policy that protects priority traffic.
