Exam Objective 3.1

Connected vs Local Routes

When you configure an IP address on an interface, the router creates TWO routing table entries — a connected route (/24 subnet) and a local route (/32 host for the interface IP itself).

R1Gi0/0: .1
10.1.1.0/24
R2Gi0/0: .2
10.1.2.0/24
R3Gi0/0: .3
CodeRouteMeaningUse
C10.1.1.0/24 via Gi0/0Connected — the subnet R1's Gi0/0 lives onTells the router "I can reach any .0-.255 address on this link directly"
L10.1.1.1/32 via Gi0/0Local — R1's own interface IP, host routeTells the router "10.1.1.1 is ME, don't forward it — process locally"
C10.1.2.0/24 via Gi0/1Connected — the subnet R1's Gi0/1 lives onAnother directly attached network
L10.1.2.1/32 via Gi0/1Local — R1's own IP on the second linkHost route for R1's own address
Why does this matter? Connected routes have AD 0 — nothing beats them. Local /32 routes are how the router knows to process traffic destined to its own interface. If someone pings 10.1.1.1, the L 10.1.1.1/32 local route matches (longest prefix) and the router responds.
