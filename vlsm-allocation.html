Step-by-step solver
How VLSM allocation works (recipe)
  1. Sort biggest-first. Allocate the largest host requirement first. Always.
  2. For each department:
    • Find smallest h where 2^h - 2 ≥ hosts. That's your host bits.
    • Prefix = 32 - h. Block size = 2^h.
  3. Round the pointer up to the nearest multiple of block size (subnets must align to their own block boundary).
  4. Assign: network = pointer, mask = prefix, broadcast = network + block − 1.
  5. Advance pointer = network + block. Move to next department.
  6. Stop and fail if pointer ≥ parent broadcast.
Example. 192.168.1.0/24 — HR 50, Eng 25, Sales 12, Mgmt 2.
HR    50 → 6 host bits → /26, block 64 → 192.168.1.0/26    bcast .63
Eng   25 → 5 host bits → /27, block 32 → 192.168.1.64/27   bcast .95
Sales 12 → 4 host bits → /28, block 16 → 192.168.1.96/28   bcast .111
Mgmt   2 → 2 host bits → /30, block  4 → 192.168.1.112/30  bcast .115
