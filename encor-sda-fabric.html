SD-Access Fabric: Three Planes

Cisco SD-Access splits the fabric into three distinct planes. The exam expects you to name the protocol for each. Toggle a plane to highlight it, then animate a packet to see all three in action.

Fabric Topology
LISP
Control Plane
The LISP map-server/map-resolver maps endpoint identity (EID) to fabric location (RLOC). When an edge node sees an unknown destination, it sends a LISP Map-Request. The control-plane node replies with the RLOC of the egress edge node.
VXLAN (with SGT)
Data Plane
VXLAN encapsulates the original Ethernet frame across the IP underlay. The outer header carries the egress RLOC as destination. The VXLAN header carries a VNI (Virtual Network Identifier) for macro-segmentation and embeds the SGT for policy.
Cisco TrustSec / CTS
Policy Plane
At egress, the edge node reads the SGT from the VXLAN header and enforces the SGACL. SGTs enable micro-segmentation: policy follows identity, not topology. VNs (Virtual Networks) provide macro-segmentation (separate VRFs).
Orchestrator
Catalyst Center (DNA Center)
Automates fabric provisioning. Assigns fabric roles (edge, border, control-plane, intermediate), configures LISP, VXLAN, and TrustSec policy from a single pane.
Packet walk — A to B
  • 1. Edge1 does not know Host B's location. It sends a LISP Map-Request to the control-plane node (map-server).
  • 2. Map-server looks up Host B's EID and replies with Edge2's RLOC. Control plane resolved.
  • 3. Edge1 VXLAN-encapsulates the frame: outer IP dst = Edge2 RLOC, VNI = 5001, SGT = 10. Sends across the underlay.
  • 4. Underlay routes the VXLAN packet to Edge2's RLOC. Data plane delivery complete.
  • 5. Edge2 decapsulates VXLAN, reads SGT 10. Looks up SGACL: SGT 10 to Host B's group.
  • 6. SGACL ALLOW: packet delivered to Host B. (Toggle SGACL to DENY to see it dropped.)
