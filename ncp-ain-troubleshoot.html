How to Troubleshoot an AI Fabric

Troubleshooting a GPU cluster fabric is not guessing. It is a layered, counter-driven loop: pin the symptom to a layer, read the fabric with the right tool, form one hypothesis, prove or kill it with a counter delta, fix at the right layer, then verify end-to-end. This page teaches the method, gives a symptom-to-tool reflex rail, a counter-to-fault decision table, and 15 guided walkthrough labs (8 InfiniBand + 7 Spectrum-X) so you learn WHY each command, in WHAT order, reading WHICH counter.

The loop

Seven steps, every time

Run them in order. Most failed fabric troubleshooting skips step 1 (define the scope) or step 7 (verify with a full re-run) and chases the wrong fault for hours.

1

Define

Which job, which rank, which GPU, which node and port, since when? One link or the whole pod? One collective or all traffic?

2

Gather counters

Read the fabric; do not guess. ibdiagnet, perfquery, nv show qos roce counters. Counters over assumptions.

3

List causes

What layers could produce this symptom? Rank by layer and by how fast a counter can confirm. Work the list, not your gut.

4

Hypothesis

One cause, one layer. Change one thing at a time. If you change two, you cannot know which fixed it.

5

Test

Prove or kill the hypothesis with a counter delta or a collective re-run. If wrong, back out the change, return to step 4.

6

Fix

Smallest change at the correct layer: cable swap, SM priority, PFC priority, ECN threshold, or routing re-convergence.

7

Verify

Re-run ibdiagnet end to end; run nccl-tests; confirm counters stop climbing. Document the root cause.

Where to start

The five-layer fabric stack (L1 to L5)

Jump into the stack at the layer matching the symptom class. "Many ranks dark, hardware suspect" starts at L1. "Collective slow but all links show up" starts at L4 or L5. Climbing the full ladder from L1 every time costs 20 minutes you do not have.

Reflexes

Symptom to tool

Before you theorize, map the symptom phrase to the one tool that confirms it and the layer it lives in. These are the phrases the exam and the console lab both use.

Decision table

Counter climbing, suspect this

If a counter is non-zero and rising, this table names the fault class. Confirm with the tool, then fix at the listed layer.
