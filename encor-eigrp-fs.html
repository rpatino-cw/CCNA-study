EIGRP DUAL: Successor and Feasible Successor

Interactive explorer: the RD < FD feasibility condition, instant FS failover, and DUAL going Active when no FS exists.

Key Concepts

FD, RD, and the Feasibility Condition

EIGRP uses DUAL to maintain a loop-free routing table. The metric to reach a destination is called a composite metric (bandwidth + delay by default, K1=1, K3=1). Understanding three terms unlocks the whole algorithm.

FD: Feasible Distance
Your best metric
The lowest total EIGRP metric R1 has ever computed to reach the destination. This comes from the Successor. FD is your benchmark.
RD: Reported Distance
Neighbor's metric
The distance a neighbor reports for the same destination (the neighbor's own FD). R1 receives this in the neighbor's EIGRP Update.
Successor
Best path, in RIB
The neighbor with the lowest total metric (link + RD). This route is installed in the routing table. Its total metric becomes R1's FD.
Feasibility Condition
RD < FD
A neighbor is a Feasible Successor only if its RD is strictly less than R1's FD. This guarantees the neighbor is loop-free.
Feasible Successor (FS)
Loop-free backup
A neighbor that passes RD < FD. Stored in the topology table. If the Successor fails, the FS is promoted instantly. R1 stays Passive.
DUAL Active
No FS, must query
If the Successor fails and no FS exists, EIGRP sends Query packets to all neighbors and goes Active until a reply restores a valid path.
Feasibility Condition in plain English: if your neighbor's own distance to the destination is already lower than your best-known distance, it cannot be routing through you, so promoting it cannot create a loop. RD < FD is sufficient proof of loop-freedom. No SPF recomputation needed.

Part C

Walk-Away Summary

The three things the exam expects you to distinguish instantly.

Successor
Best path. Lowest total metric wins. Installed in the routing table. Its total metric becomes R1's Feasible Distance (FD). One Successor per destination (unless equal-cost paths with variance 1, the default).
Feasible Successor (FS)
Pre-qualified backup in the topology table. Passes RD < FD. When the Successor fails, the FS is promoted instantly with no query. EIGRP stays Passive. This is the whole point of DUAL pre-computation.
Fails Feasibility Condition
RD greater than or equal to FD. Could be routing through R1, creating a loop. Not stored as an FS. Can still be used if no other path exists, but only after DUAL query.
DUAL Active State
Successor fails with no FS available. EIGRP sends Query packets to all neighbors and waits for Replies. Route stays Active until a valid Successor is confirmed. This adds convergence delay.
Exam footnotes: Default variance = 1 (equal-cost only). Increase variance to allow unequal-cost load balancing, but only to neighbors that meet the feasibility condition. EIGRP composite metric uses K1=1 (bandwidth) and K3=1 (delay) by default; K2, K4, K5 = 0. The composite metric formula: metric = [K1 * (10^7 / min-bandwidth) + (K3 * sum-delay/10)] * 256.
