The One Untagged VLANdrill

802.1Q trunks tag every VLAN — except one. The native VLAN rides untagged across the wire. Default = VLAN 1. Both ends MUST agree (CDP screams if they don't). Best practice: pick an unused VLAN (like 999), prune VLAN 1, and never put user hosts on the native. Build the trunk below and watch the frames flip.

What "untagged native" means

The traps

Mismatch

SW1 native VLAN 1, SW2 native VLAN 99. Untagged frames from SW1 land in VLAN 99 on SW2 and vice versa — silent VLAN-bridging. CDP catches it: %CDP-4-NATIVE_VLAN_MISMATCH discovered on FastEthernet0/1 (1), with SW2 (99). STP recalculates. Production traffic crosses VLANs without anyone meaning to.

Double-tagging attack (VLAN hopping)

Attacker on access VLAN that equals trunk's native sends a frame with TWO 802.1Q tags. SW1 strips the outer (native, untagged) and forwards. SW2 sees the inner tag and delivers to the target VLAN. Mitigation: native VLAN ≠ user VLAN, prune unused VLANs, vlan dot1q tag native globally so even native gets tagged.

Worked example — show interface trunk

! both sides agree on native VLAN 999 SW1# show interfaces gi0/1 trunk Port Mode Encapsulation Status Native vlan Gi0/1 on 802.1q trunking 999 Port Vlans allowed on trunk Gi0/1 1-4094 ! mismatch surfaces in CDP SW1# show log | include MISMATCH *Mar 1 00:08:12: %CDP-4-NATIVE_VLAN_MISMATCH: Native VLAN mismatch discovered on Gi0/1 (1), with SW2 Gi0/24 (99)

Faded prompt — your turn

Hosts in VLAN 1 on SW1 are reaching hosts in VLAN 99 on SW2. The trunk is up, both ends are in 802.1Q, no static routes connect 1↔99. What is misconfigured?

Answer: native VLAN mismatch — SW1 native is 1, SW2 native is 99. Untagged frames cross the trunk and get dropped into whichever native VLAN the receiving switch claims. Fix: align native on both sides AND move it to an unused VLAN (e.g. 999) per hardening guides.

Bonus: enable vlan dot1q tag native globally so even the native VLAN gets explicitly tagged on the wire. Defeats double-tagging at the cost of a bit of overhead.
