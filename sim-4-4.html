SNMP Polling & Traps Sim

A network with 1000 devices needs visibility. SNMP is the language: NMS asks (Get), agent answers (Response), and agent shouts when bad things happen (Trap). The data lives in a tree (the MIB). v1/v2c send the password in cleartext; v3 finally adds real auth + encryption.

NMS ⇄ Agent — animated

Versions

v1 (1988)

community string ✗ no auth ✗ no encryption · 32-bit counters · no GetBulk

v2c (1996)

community string ✗ no auth ✗ no encryption · 64-bit counters · GetBulk + Inform

v3 (2004)

user-based ✓ auth (HMAC-MD5/SHA) ✓ encryption (DES/AES) · noAuthNoPriv / authNoPriv / authPriv

MIB Tree

1 (iso) └─ 3 (org) └─ 6 (dod) └─ 1 (internet) ├─ 2 (mgmt) │ └─ 1 (mib-2) │ ├─ 1 (system) sysDescr, sysUpTime, sysName │ ├─ 2 (interfaces) ifInOctets, ifOutOctets, ifOperStatus │ ├─ 4 (ip) │ └─ 6 (tcp) └─ 4 (private) └─ 1 (enterprises) └─ 9 (cisco) cisco-specific OIDs
Example OID — sysUpTime: 1.3.6.1.2.1.1.3.0
